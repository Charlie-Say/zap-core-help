Вкладка История

На вкладке «История» отображается список всех запросов в том порядке, в котором они были сделаны.
Для каждого запроса вы можете увидеть:
     Индекс запроса - каждый запрос нумеруется, начиная с 1
     Метод HTML, например. ПОЛУЧИТЬ (GET) или ОТПРАВИТЬ (POST)
     Запрошенный URL
     Код HTTP-ответа
     Краткое описание того, что означает код ответа HTTP
     Время выполнения всего запроса
     Любые оповещения по запросу
     Любые примечания, которые вы добавили к запросу
     Любые теги в запросе
Выбор запроса отобразит его в Вкладка запроса и Вкладка "Ответ" выше.

Панель инструментов фильтра

Предоставляется панель инструментов фильтра, которая позволяет вам ограничивать отображаемые запросы.
Нажав на Кнопка фильтра отображает Диалоговое окно «Фильтр истории» .
Справа от кнопки отображается сводка примененного в данный момент фильтра.

Меню правой кнопки мыши

Щелкнув правой кнопкой мыши по узлу, вы откроете меню, которое позволит вам:

Атака

В меню «Атака» есть следующие подменю:

Активное Сканирование...

Это запустит Активное Сканирование диалоговое окно, позволяющее инициировать активное сканирование с отправной точкой, установленной в соответствии с выбранным вами запросом.

Паук...

Это запустит Паук диалоговое окно, позволяющее инициировать паука с отправной точкой, установленной в соответствии с выбранным вами запросом.

Включить в контекст

Это меню позволяет включить выбранные узлы и все их подчиненные узлы в указанный Контекст.
У вас также есть возможность создать новый контекст.
Контексты сеанса диалоговое окно будет отображаться для позволяют вносить любые дополнительные изменения.

Исключить из контекста

Это меню позволяет исключить выбранные узлы и все их подчиненные узлы из указанного списка. Контекст.
Контексты сеанса диалоговое окно будет отображаться для позволяют вносить любые дополнительные изменения.

Отметить как контекст

Это меню имеет следующие подменю для каждого из Контексты вы определили:

Имя контекста Запрос авторизации на основе формы

Это идентифицирует указанный узел как запрос входа в систему для указанного контекста.
У вас может быть только один узел, идентифицированный как таковой в любом одном контексте.
Аутентификация контекста сеанса экран будет отображаться на позволяют вносить любые дополнительные изменения.

Имя контекста Запрос аутентификации на основе JSON

Это идентифицирует указанный узел как запрос входа в систему для указанного контекста.
У вас может быть только один узел, идентифицированный как таковой в любом одном контексте.
Аутентификация контекста сеанса экран будет отображаться на позволяют вносить любые дополнительные изменения.

Имя контекста Узел, управляемый данными

Это идентифицирует указанный узел как Контент, управляемый данными для указанного контекста.
Структура контекста сеанса экран будет отображаться на позволяют вносить любые дополнительные изменения.

Исключить из

Это меню имеет следующие подменю:

Прокси-сервер

Это исключит выбранные узлы из прокси. Они по-прежнему будут проксироваться через ZAP, но не будут отображаться в любой из вкладок.
Это можно использовать для игнорирования URL-адресов, которые, как вы знаете, не имеют отношения к тестируемой системе.
Узлы могут быть снова включены через Свойства сессии диалог

Сканер

Это предотвратит активное сканирование выбранных узлов.
Узлы могут быть снова включены через Свойства сессии диалог

Паук

Это предотвратит сканирование выбранных узлов.
Узлы могут быть снова включены через Свойства сессии диалог

Управление тегами истории...

Это вызовет Диалоговое окно «Управление тегами истории» который позволяет изменить в теги связанные с запросом.

Примечание...

Это вызовет Диалоговое окно добавления заметки который позволяет записывать заметки связанный с запросом.

Удалить

Это удалит узел и все его дочерние элементы из ZAP.
Однако их можно добавить обратно, чтобы предотвратить это, используя меню «Исключить из».

Прерывание...

Это вызовет Диалог добавления точки останова который позволяет вам установить точку останова на этом URL-адресе.

Оповещения для этого узла

Если выбранный URL имеет оповещения связанные с ним тогда они будут перечислены в этом меню.
Выбор одного из предупреждений приведет к его отображению.

Открыть/повторно отправить с помощью редактора запросов...

Это вызовет Диалоговое окно редактора запросов вручную что позволяет вам повторно отправьте запрос после внесения в него любых изменений, которые вы хотите.

Новое оповещение...

Это вызовет Диалоговое окно «Добавить предупреждение» который позволяет вручную записывать новый оповещение против этого запроса.

Показать на вкладке "Сайты"

Это покажет выбранное сообщение в Вкладка Сайты.

Открыть URL в браузере

Это откроет URL-адрес выбранного узла в браузере по умолчанию.

Создать тестовую форму против CSRF

Это откроет URL-адрес, который даст вам сгенерированную форму для тестирования на наличие проблем CSRF.
Он будет включен только для запросов POST, если API включен и если Java поддерживает открытие URL-адресов. в браузере на вашей платформе.
