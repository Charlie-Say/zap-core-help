Вкладка поиска

Вкладка «Поиск» позволяет искать регулярные выражения во всех URL-адреса, запросы, ответы, заголовки и другие функции, предоставляемые надстройками.

Введите регулярное выражение, которое вы хотите найти в поиске поле и либо нажмите клавишу возврата, либо нажмите кнопку поиска:

Выпадающее меню позволяет выбрать, следует ли выполнять поиск по URL-адресам, запросы, ответы или все.

Все URL-адреса, запросы или ответы, содержащие шаблон поиска, будут отображаться на вкладке.
Возможно, вам будет полезно добавить .* к вашему поисковому запросу - это даст вам больше контекста, например. пароль.*

Вы можете использовать Далее и Предыдущий кнопки для просмотра найденных условий поиска в Запросе и Вкладки ответов.

Вы также можете перейти непосредственно к конкретному экземпляру, щелкнув соответствующий строку в списке результатов.

Меню правой кнопки мыши

Щелкнув правой кнопкой мыши по узлу, вы откроете меню, которое позволит вам:

Атака

В меню «Атака» есть следующие подменю:

Активное Сканирование...

Это запустит Диалоговое окно активного сканирования который позволяет инициировать активное сканирование с отправной точкой, установленной в соответствии с выбранным вами запросом.

Паук...

This will launch the Spider dialog which allows you to initiate the spider with the starting point set to the request you selected.

Включить в контекст

This menu allows you to include the selected nodes and all of their subordinates in the specified context.
You also have the option to create a new context.
The Session Contexts dialog will be displayed to allow you to make any additional changes.

Исключить из контекста

This menu allows you to exclude the selected nodes and all of their subordinates from the specified Контекст.
The Контексты сеанса dialog will be displayed to allow you to make any additional changes.

Отметить как контекст

This menu has the following submenus for each of the contexts you have defined:

Имя контекста Запрос авторизации на основе формы

This identifies the specified node as a login request for the specified context.
You may only have one node identified as such in any one context.
The Session Context Authentication screen will be displayed to allow you to make any additional changes.

Имя контекста Запрос аутентификации на основе JSON

This identifies the specified node as a login request for the specified context.
You may only have one node identified as such in any one context.
The Session Context Authentication screen will be displayed to allow you to make any additional changes.

Имя контекста Узел, управляемый данными

This identifies the specified node as Data driven content for the specified context.
The Session Context Structure screen will be displayed to allow you to make any additional changes.

Исключить из

This menu has the following submenus:

Прокси-сервер

This will exclude the selected nodes from the proxy. They will still be proxied via ZAP but will not be shown in any of the tabs.
This can be used to ignore URLs that you know are not relevant to the system you are currently testing.
The nodes can be included again via the Session Properties dialog

Сканер

This will prevent the selected nodes from being actively scanned.
The nodes can be included again via the Session Properties dialog

Паук

This will prevent the selected nodes from being spidered.
The nodes can be included again via the Свойства сеанса dialog

New Alert...

This will bring up the Add Alert dialog which allows you to manually record a new alert against this request.

Show in History tab

This will show the selected node in the History tab.

Show in Sites tab

This will show the selected message in the Sites tab.

Open URL in Browser

This will open the URL of the selected node in your default browser.
