Диалог свойств сеанса

Это позволяет вам установить свойства сеанса и состоит из следующих экранов:

Общий

Это позволяет вам установить имя и описание сеанса.

Исключить из прокси

Это позволяет вам управлять URL-адресами, которые будут игнорироваться локальными прокси.

Исключить из сканера

Это позволяет вам управлять URL-адресами, которые будут игнорироваться сканером.

Исключить из паука

Это позволяет вам управлять URL-адресами, которые будут игнорироваться пауками (стандартными и AJAX).

Регулярные выражения для URL-адресов

В Exclude from * (Исключить из * ) диалоговых окнах вы можете вводить регулярные выражения для определения исключенных URL-адресов. Хотя вы можете экранировать одиночный метасимвол с помощью обратной косой черты, вы также можете использовать \Q...\E escape-последовательность. Все символы между \Q и \E интерпретируются как буквальные символы. Например. \ Q * \ d + * \ E соответствует буквальному тексту * \ d + *. Эта escape-последовательность используется в ZAP, когда вы исключаете URL-адреса через некоторые контекстные меню.
Примечание: Если ваш URL-адрес содержит "\ E", вам необходимо выполнить следующие шаги при использовании \Q...\E escape-последовательность:
  • Откройте escape-последовательность
  • Закройте escape-последовательность перед "символом" \ E
  • Escape the backslash
  • Открыть после "\ E" другую escape-последовательность;
  • Закройте escape-последовательность, как обычно.

Пример: subdomain.example.com/path?a= \E&moredata=2 should appear as \Qsubdomain.example.com/path?a=\E\\E\Q&moredata=2\E
