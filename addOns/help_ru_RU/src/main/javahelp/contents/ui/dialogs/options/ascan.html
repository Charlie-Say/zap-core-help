Параметры экрана активного сканирования

Этот экран позволяет вам настроить активное сканирование опции

Количество хостов, сканированных одновременно

Максимальное количество хостов, которые будут сканироваться одновременно. Увеличение этого параметра может вызвать дополнительную нагрузку на компьютер, на котором работает ZAP.

Параллельные потоки сканирования на хост

Количество потоков, которые сканер будет использовать на хост.
Увеличение количества потоков ускорит сканирование, но может вызвать дополнительную нагрузку на компьютер, на котором запущен ZAP, и на целевой хост.

Максимальное количество результатов в списке

Количество результатов, которые будут отображаться на вкладке «Активное сканирование».
Отображение большого количества результатов может значительно увеличить время сканирования.

Максимальная продолжительность правила (минуты; 0 - неограниченно)

Максимальное время, в течение которого может работать любое отдельное правило, в минутах. Ноль означает отсутствие ограничений. Это можно использовать для предотвращения правил, на которые уходит слишком много времени.

Максимальная продолжительность сканирования (минут; 0 - неограниченно)

Максимальное время, в течение которого может выполняться сканирование, в минутах. Ноль означает отсутствие ограничений. Это можно использовать, чтобы гарантировать, что сканирование будет завершено примерно в установленное время.

Задержка при сканировании (в миллисекундах)

Задержка в миллисекундах между каждым запросом.
Установка ненулевого значения увеличит время активного сканирования, но вызовет меньшую нагрузку. на целевом хосте.

Вставить идентификатор плагина в заголовок для всех активных запросов сканирования.

Если выбрана эта опция, активный сканер вставит заголовок запроса. X-ZAP-Scan-ID с идентификатором сканер, отправляющий HTTP-запросы.

Обработка токенов анти-CSRF.

Если выбран этот параметр, активный сканер попытается автоматически запросить anti CSRF токены при необходимости.
Раньше это заставляло бы сканер использовать только один поток, но теперь это не так. Настоятельно рекомендуется проверить, правильно ли генерируются токены анти-CSRF, если используется более одного потока, например, используя индивидуальный теги для проверки шаблонов успеха / неудачи в ответе.

В режиме атаки запрашивать повторное сканирование узлов при изменении области видимости.

Если выбрана эта опция, то при выборе Атака режимы Вам будет предложено выбрать, следует ли повторно сканировать узлы в области.
Если этот параметр не выбран, то следующий параметр будет определять, будут ли узлы сканироваться повторно.

В режиме атаки всегда повторно проверяйте узлы при изменении области видимости.

Если выбрана эта опция, то при запуске в Атаке режимы все узлы в области будут проверены повторно, если область изменится.
Это не рекомендуется для больших сайтов, так как это может занять много времени.

Политика активного сканирования по умолчанию

Политика сканирования который используется по умолчанию при запуске активного сканирования.

Режим атаки Политика сканирования:

Политика сканирования который используется для сканирования в Атака режиме.

График максимального прогресса в минутах

Максимальное время в минутах, в течение которого коды ответов будут отображаться в Диалог выполнения сканирования .
Чтобы отключить график, необходимо установить опцию на ноль минут.
