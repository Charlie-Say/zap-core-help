Окно «Сертификат клиента» параметров

Экран «Сертификат клиента» позволяет добавить сертификат клиента для использования ZAP при тестировании приложений. защищены с помощью взаимного SSL. В настоящее время поддерживаются следующие способы использования сертификатов: PKCS12 и PKCS11.

Использование PKCS12

Сертификат клиента можно установить из командной строки с помощью следующих переключателей: 

-config certificate.use=true
-config certificate.pkcs12.path=/path/to/file.p12
-config certificate.pkcs12.password=WhateverThePasswordIs
-config certificate.pkcs12.index=1
-config certificate.persist=true

-config certificate.pkcs12.index необходим только в том случае, если файл содержит несколько сертификатов. и вы хотите использовать один, отличный от первого. Значения индекса начинаются с нуля (0), что является первым сертификатом. в файле.

-config certificate.persist=true необходим только в том случае, если вы хотите, чтобы настройки сохранялись в обычном файл конфигурации (чтобы они применялись для последующего использования ZAP).

Использование PKCS11

PKCS11 — это один из стандартов криптографии с открытым ключом. Обычно PKCS11 используется с аппаратными токенами (в основном в стиле USB) которые хранят криптографический материал безопасным способом. Таким образом, обычно требуется драйвер. Драйвера чаще всего предоставляются эмитентом сертификата, хранящегося на токене, или поставщиком токена. Некоторые токены (особенно при запуске под Linux) поддерживаются комбинацией opensc/libccid.

Сертификат клиента можно установить из командной строки с помощью следующих переключателей: 

-config certificate.use=true
-config certificate.experimentalSlotListIndex=true

На данный момент установка драйвера PKCS11, PIN-кода и активного сертификата из командной строки невозможна. С использованием certificate.experimentalSlotListIndex=true требуется только для некоторых карт. Если без не работает попробуйте добавить его.
