Экран «Параметры локальных прокси»

Этот экран позволяет настроить адреса и порты. на котором ZAP принимает входящие соединения.

Локальный прокси-сервер

По умолчанию ZAP будет прослушивать один локальный адрес и порт, и это адрес и порт, которые вы должны настроить в своем браузере для использования в качестве прокси.

Адрес

Локальный адрес, который будет использовать ZAP. Все доступные адреса определяются автоматически.

Порт

Порт, на котором ZAP будет слушать.

За NAT

Указывает, что локальный прокси (ZAP) находится за NAT. При выборе ZAP попытается определить общедоступный IP-адрес, чтобы правильно обнаруживать и обрабатывать запросы с общедоступным IP-адрес (например, для обслуживания ZAP API).

Примечание: Этот параметр поддерживается только в том случае, если ZAP работает в экземпляре AWS EC2. ZAP получит общедоступный IP-адрес от AWS EC2 метаданные экземпляра .
ZAP следует запускать с включенной этой опцией, если требуется доступ к API через общедоступный IP-адрес:

zap.sh -daemon -port 8080 -host 0.0.0.0 -config proxy.behindnat=true
Так же API необходимо настроить для приема внешних IP-адресов (т. адрес, с которого осуществляется доступ к ZAP).

Удалить неподдерживаемые кодировки

Позволяет прокси-серверу удалять неподдерживаемые кодировки из поля заголовка запроса «Accept-Encoding», поэтому никакие (неподдерживаемые) преобразования кодирования не выполняются для ответа.
Эта опция должна быть всегда включена, если только вы не тестируете преобразования кодирования.
Сообщения, закодированные с помощью неподдерживаемых кодировок, не будут корректно сканироваться (как пассивными, так и активными сканерами).

Протоколы безопасности

Позволяет выбрать разрешенные версии SSL/TLS для входящих соединений (например, из браузеров). Как минимум одна версия должна быть включена, версии, не поддерживаемые JRE, будут отменены и отключены.
Параметр SSLv2Hello должен быть выбран в сочетании хотя бы с одной версией SSL/TLS.
Параметры безопасности применяются как к основному, так и ко всем дополнительным прокси.

Дополнительные прокси

Вы можете добавить столько дополнительных адресов и портов для прослушивания ZAP, сколько захотите.
Это может быть полезно при тестировании мобильных приложений — вы можете одновременно проксировать мобильное приложение через интерфейс Wi-Fi. как использование внутреннего сайта напрямую с использованием прокси-сервера браузера через ZAP, эмулирующего мобильный пользовательский агент.

Перехватывающий/прозрачный прокси

Локальные прокси могут использоваться как перехватывающие/прозрачные прокси как для HTTP, так и для HTTPS. Для HTTPS клиентские приложения (например, браузер) необходимо использовать расширение TLS Индикация имени сервера . Это позволяет настроить тестовые локальные сети (или виртуальные машины), в которых весь HTTP- и HTTPS-трафик передается через прокси независимо от настроек программного обеспечения.

Например, если у вас есть компьютер с Linux, который вы используете для тестирования, вы можете сделать что-то вроде следующего, чтобы перенаправить все HTTP и HTTPS-трафик на локальный прокси-сервер, прослушивающий 192.168.0.14:8080: 

iptables -t nat -A OUTPUT -p tcp --dport 443 -j DNAT --to-destination 192.168.0.14:8080
iptables -t nat -A OUTPUT -p tcp --dport 80 -j DNAT --to-destination 192.168.0.14:8080
