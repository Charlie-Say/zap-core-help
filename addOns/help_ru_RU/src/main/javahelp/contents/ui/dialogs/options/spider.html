Опции Паук (Spider) экран

На этом экране можно настроить параметры паука.

Следует отметить, что изменение большинства этих параметров также влияет на бегущий Паук.

Максимальная глубина ползания

Параметр определяет максимальную глубину в процессе сканирования, на которой страница должна быть найдена, чтобы ее можно было обработать. Ресурсы найдены глубже этого уровня не извлекаются и не анализируются пауком. Значение ноль означает неограниченную глубину.

Глубина рассчитывается, начиная с семян, поэтому, если сканирование паука начинается только с одного URL-адреса (например, URL указан вручную), глубина рассчитывается из этого. Однако, если сканирование начинается с нескольких начальных значений (например. recurse (повторное просматривание) и узел дерева Sites с дочерними элементами), ресурс обрабатывается, если его глубина относительно любого начального числа меньше заданного.

Количество используемых потоков

Паук (Spider) является многопоточным, и это число, которое определяет максимальное количество рабочих потоков, используемых в процессе сканирования. Изменение этого параметр не оказывает никакого влияния на выполняемое сканирование.

Максимальная продолжительность

Максимальное время, в течение которого паук должен работать, измеряется в минутах. Ноль (значение по умолчанию) означает, что паук будет работать до тех пор, пока не найдет все возможные ссылки.

Максимальное количество детей для ползания

Этот параметр ограничивает количество дочерних элементов, которые будут просматриваться в каждом узле дерева.
Это полезно для приложений, управляемых данными, которые имеют большое количество «страниц», которые на самом деле являются одним и тем же кодом, но содержащие разные данные, например из базы данных.
По умолчанию установлено нулевое значение, что означает отсутствие ограничений на количество просканированных дочерних узлов.

Максимальный размер разбора

Определяет максимальный размер в байтах, который может потребоваться для анализа ответа. Это позволяет паук, чтобы пропустить большие ответы/файлы.

Домены всегда в зоне действия

Позволяет управлять доменами, строковыми литералами или регулярными выражениями, находящимися в размах паука. Нормальное поведение паука - переходить только по ссылкам на ресурсы находится в том же домене, что и страница, с которой началось сканирование. Тем не мение, эта опция позволяет вам определить дополнительные домены, которые считаются "в рамках" в процессе сканирования. Страницы на этих доменах обрабатывается во время сканирования.

Обработка параметров запроса

При сканировании Паук имеет внутренний механизм, который отмечает, какие страницы уже были посещены, поэтому повторно не обрабатываются. Когда эта проверка сделана, способ обработки параметров URI задается с помощью этой опции. Есть три доступных варианта:
  • Полное игнорирование параметров — при посещении www.example.org/?bar=456 тогда www.example.org/?foo=123 не будет посещен
  • Учитывать только имя параметра (игнорировать значение параметра) - если www.example.org/?foo=123 посещается, тогда www.example.org/?foo=456 не посещается, но www.example.org/?bar=789 или www.example.org/?foo=456&bar=123 будут посещены
  • Учитывайте как имя параметра, так и значение – если посещается www.example.org/?123, любой другой отличающийся uri (включая, например, www.example.org/?foo=456 или www.example.org/?bar=abc) будут посещены

Отправить заголовок «Referer»

Если запросы паука должны быть отправлены с заголовком «Referer».

Принять файлы cookie

Если сканирование пауков должно принимать файлы cookie во время сканирования. Если включено, Паук будет должным образом обрабатывать любые файлы cookie, полученные с сервера, и соответствующим образом отправлять их обратно. Если опция отключена, Паук не будет отправлять файлы cookie в своих запросах. Формы процесса:

В процессе обхода поведение паука, когда он встречается с HTML-формами, определяется этой опцией. Если отключено, HTML формы не будут обрабатываться вообще. Если включено, HTML формируется с метод, определенный как HTTP GET, будет отправлен с некоторыми сгенерированными ценности. Поведение при встрече с формами с определенным методом поскольку HTTP POST настраивается следующей опцией.

POST-формы:

Как кратко описано в предыдущем абзаце (Формы процесса), это опция настраивает поведение паука, когда Формы процесса: включен, и при встрече с HTML-формами, которые должны быть размещены.

Разобрать HTML-комментарии

Этот параметр определяет, должен ли паук также сканировать HTML-код. комментарии ищут ссылки на ресурсы. Только ресурсы, найденные в закомментированные допустимые теги HTML будут обработаны.

Разобрать файлы robots.txt

Эта опция определяет, должен ли паук также сканировать Файлы robots.txt найдены на сайтах, поиск ссылок на ресурсы. Эта опция не определяет, должен ли паук следовать правилам накладывается файлом robots.txt.

Обработка параметров OData

Этот параметр определяет, должен ли паук пытаться обнаруживать специфичные для OData параметры (т.е. идентификаторы ресурсов) для правильной их обработки по правилу, заданному опцией "Обработка параметров запроса".
