Экран подключения параметров

Экран «Параметры подключения» позволяет настроить параметры подключения ZAP:

Время ожидания (в секундах)

Это упрощает тестирование медленных приложений.

Пользовательский агент по умолчанию:

Пользовательский агент, который ZAP должен использовать при создании HTTP-сообщений (например, спайдер-сообщений или запросов CONNECT к исходящим прокси).

Заголовок запроса одного файла cookie

Определяет, должны ли быть установлены управляемые файлы cookie ZAP (включить (глобальное) состояние HTTP). в одном заголовке запроса «Cookie» или нескольких заголовках запроса «Cookie» при отправке HTTP-запроса на сервер.

Включить (глобальное) состояние HTTP

Это позволяет отслеживать детали сеанса, хранящиеся в файлах cookie.
Этот параметр необходимо выбрать, чтобы установить флажок «Использовать текущий сеанс отслеживания» в Диалоговые окна редактора ручных запросов .
Отслеживание сеанса гарантирует, что любые запросы отправляются с последними сведениями о сеансе.
Например, вы можете записать сеанс при входе в систему как один пользователь, а затем выйти и войти в систему как другой пользователь.
Если вы повторно отправите запрос из первого сеанса без отслеживания сеанса, он будет использовать файлы cookie из первая сессия.
Если вы повторно отправите тот же запрос с отслеживанием сеанса, он будет использовать файлы cookie из второго сеанса.

DNS

Успешные запросы TTL

Определяет, как долго должны кэшироваться успешные DNS-запросы:
  • Отрицательное число, кешировать навсегда;
  • Zero, отключает кеширование;
  • Положительное число, количество секунд, в течение которых запросы будут кэшироваться.
Примечание: Изменения применяются после перезагрузки.

Параметр также можно установить с помощью аргумента командной строки -config с ключом connection.dnsTtlSuccessfulQueries.

Протоколы безопасности

Позволяет выбрать разрешенные версии SSL/TLS для исходящих соединений (например, к серверам). Как минимум одна версия должна быть включена, версии, не поддерживаемые JRE, будут отменены и отключены.
Опция SSLv2Hello должна быть выбрана в сочетании хотя бы с одной версией SSL/TLS.

Это часто требуется в корпоративной среде.

Прокси-аутентификация

Этот раздел позволяет настроить исходящую аутентификацию прокси.

SOKS-прокси

Этот раздел позволяет настроить исходящий прокси-сервер SOCKS, по умолчанию конфигурация прокси-сервера SOCKS применяется ко всем подключениям, выполняемым ZAP.

Системные свойства прокси-сервера SOCKS (например, socksProxyHost, socksProxyPort) имеют приоритет над сохраненными конфигурациями. для совместимости со старыми версиями ZAP. ZAP будет использовать и отображать значения свойств системы, когда они определены при запуске, конфигурации все еще можно изменить на этой панели.

Примечание: Loopback-адреса (например, localhost, 127.0.01, ::1) не проксируются через SOCKS-прокси, соединения будут выполняться напрямую.

Включено

Если ZAP должен использовать настроенный SOCKS-прокси.
По умолчанию невыбранный , если не определены свойства прокси-системы SOCKS, в этом случае выбрано.

Хост

Имя хоста или адрес прокси-сервера SOCKS.
По умолчанию localhost.

Порт

Порт SOCKS-прокси.
По умолчанию 1080.

Версия

Версия прокси SOCKS.
По умолчанию 5.

Используйте SOCKS-DNS

Если ZAP должен позволить прокси-серверу SOCKS разрешать имена хостов. Требуется прокси-сервер SOCKS версии 5.
Это может привести к сбоям соединения, если прокси-сервер SOCKS не сможет разрешить имя хоста (например, использование имен, определенных в хостах файл).
По умолчанию выбрано.
