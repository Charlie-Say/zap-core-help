Экран аутентификации контекста сеанса

Это один из экранов контекста сеанса который позволяет вам управлять способом выполнения аутентификации для контекста.

Метод аутентификации

После выбора типа Метод аутентификации параметры, которые необходимо настроить, зависят от метода аутентификации.

Примечание: Изменение метода аутентификации после определения пользователей может привести к сбросу его учетных данных, поскольку тип учетных данных пользователя должен соответствовать схеме аутентификации. Когда это произойдет, появится диалоговое окно подтверждения.

Аутентификация на основе форм

Чтобы настроить этот метод аутентификации, вам необходимо предоставить login url, к которому логин запрос выполняется, тело запроса (request body) (Данные POST), если необходимо, и укажите Параметры используется для снабжения Если тело запроса не предоставлено, запрос на вход выполняется как HTTP GET, в противном случае используется HTTP POST. Сами верительные грамоты настроены в Пользователи таб. страница авторизации (login page) также могут быть предоставлены, чтобы указать, откуда получить новый сеанс (например, файлы cookie) и возродить Анти-CSRF токены присутствует в тело запроса (request body). Если страница авторизации (login page) не поставляется, используется URL входа в систему (login url).

Аутентификация на основе JSON

Чтобы настроить этот метод аутентификации, вам необходимо предоставить URL входа в систему (login url), к которому логин запрос выполняется, Объект JSON (JSON object) (Данные POST, приложение / json ) и идентифицировать в Параметры используется для предоставления «имени пользователя» и «пароля». Сами верительные грамоты настроены в Пользователи таб. страница авторизации (login page) также могут быть предоставлены, чтобы указать, откуда получить новый сеанс (например, файлы cookie). Если страница авторизации ( login page ) не поставляется, используется URL входа в систему ( login url ).

Аутентификация на основе сценария

Чтобы использовать этот метод аутентификации, вы первый (first) нужно написать (и сохранить) Скрипт аутентификации (Authentication Script) с помощью Сценарии вкладка (см. предоставленные примеры и шаблоны для этого типа сценария на вкладке "Сценарии"). Затем вам нужно указать имя скрипта в раскрывающемся списке. После выбора скрипта нужно нажать в Нагрузка кнопка, загружающая все требования скрипта. Любые параметры, которые вы указали как обязательный или по желанию в сценарии будет отображаться в интерфейсе, который будет определен. Их значения доступны для использования в скрипте во время аутентификации, как показано в предоставленных примерах сценариев аутентификации. Убедитесь, что после внесения любых изменений в параметры требуется для сценария аутентификации, вы повторно загружаете сценарий. В противном случае параметры, отображаемые в интерфейсе, могут не быть теми, которые использовались во время аутентификации, в противном случае могут возникнуть ошибки. Учетные данные используется для каждого пользователя во время аутентификации обрабатываемая, также может быть указана в сценарии аутентификации и настроена в Пользователи вкладки. Проверка подлинности

Стратегия проверки

Выберите Стратегия проверки подлинности обязательный.

Шаблон регулярного выражения, используемый для идентификации сообщений, вошедших в систему

Если этот шаблон регулярного выражения указан и соответствует любой строке в указанном сообщении, ZAP будет рассматривать это сообщение как зарегистрированное. Например, шаблон может использоваться для сопоставления ссылки http://example.com/logout или наличия сообщения «Добро пожаловать, пользователь X».

Шаблон регулярного выражения, используемый для идентификации сообщений, вышедших из системы

Если этот шаблон регулярного выражения указан и соответствует любой строке в указанном сообщении, ZAP будет рассматривать это сообщение как вышедшее из системы. Например, шаблон может использоваться для сопоставления ссылки http://example.com/login или наличия сообщения «Добро пожаловать, Гость».

Частота опроса

Это контролирует, как часто ZAP будет опрашивать, аутентифицирован ли пользователь. Целое число будет представлять либо количество запросов, которые будут сделаны до того, как ZAP выполнит повторный опрос, либо количество секунд до этого. он будет опрашивать снова, в зависимости от выбранного раскрывающегося списка.
Опция актуальна только в том случае, если была выбрана стратегия проверки «Опрос указанного URL».

URL для опроса для проверки

Это URL-адрес, который ZAP будет опрашивать, чтобы узнать, аутентифицирован ли пользователь.
Опция актуальна только в том случае, если была выбрана стратегия проверки «Опрос указанного URL».

Данные POST запроса опроса

Если предоставлено, то оно будет отправлено как POST в запросе на опрос. Если он пуст, ZAP сделает запрос GET.
Опция актуальна только в том случае, если была выбрана стратегия проверки «Опрос указанного URL».

Дополнительные заголовки запроса на опрос

Необязательный набор заголовков, который будет отправлен с запросом на опрос. Каждая строка должна состоять из одного заголовка и значения, разделенного двоеточием, например 
Тип содержимого: приложение / json
X-Requested-с: XMLHttpRequest
The option is only relevant if the "Poll the Specified URL" Verification Strategy has been selected.
