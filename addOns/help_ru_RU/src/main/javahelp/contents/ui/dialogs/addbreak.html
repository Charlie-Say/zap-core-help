Диалог добавления / редактирования точки останова

Этот диалог позволяет вам добавлять и редактировать точки останова HTTP .

Точка останова определяется следующими полями:

  • Location - где String проверяется в HTTP-сообщение: URL, Request Header, Request Body, Response Header, or Response Body.
  • Match - как интерпретируется String , Regex или Содержит для регулярного выражения или точного совпадения, соответственно, в Location . Регулярное выражение не обязательно должно соответствовать всему содержимому Location .
  • String - строка, запускающая точку останова.
  • Inverse - если результат Match должен быть обратное.
  • Игнорировать регистр - если регистр String должен быть игнорируется.
If you proxy a HTTP message that matches a breakpoint then ZAP will intercept it and allow you to change either the request or the response.

Примечание: ZAP предупредит и предотвратит добавление точек останова с помощью компонента идентификатора фрагмента ( #), если точка останова соответствует Содержит и расположение URL-адрес. Такая точка останова не будет работать, потому что идентификатор фрагмента не отправляется на сервер.
