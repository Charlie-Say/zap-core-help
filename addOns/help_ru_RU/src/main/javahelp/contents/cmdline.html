командная строка

Чтобы запустить ZAP через командную строку, вам нужно будет найти сценарий запуска ZAP.
Windows:
C:\Program Files (x86)\OWASP\Zed Attack Proxy\zap.bat
Примечание: Параметры командной строки не используются исполняемым файлом ( zap.exe) только bat файл.

Mac:
/Applications/OWASP\ ZAP.app/Contents/Java/zap.sh
Linux:
zap.sh будет ниже каталога, в котором был установлен ZAP.

В качестве альтернативы вы можете запустить файл JAR напрямую: 
java -jar zap.jar
Все параметры, указанные ниже, можно передать любому из них.

Параметры

ZAP поддерживает следующие параметры командной строки:
    -версияСообщает о версии ZAP
    -cmdЗапустить встроенный (завершается, когда параметры командной строки завершены)
    -daemonЗапускает ZAP в режиме демона, т.е. без пользовательского интерфейса.
    -config <kvpair>Переопределяет указанную пару ключ = значение в файле конфигурации. Параметры командной строки -config применяются в том порядке, в котором они указаны.
    -configfile <path>Заменяет пары ключ = значение на пары в указанном файле свойств
    -dir <dir>Использует указанный каталог в качестве домашнего каталога вместо каталога по умолчанию. Чтобы предотвратить (непреднамеренное) использование надстроек / переопределение файлов ядра, ZAP не запустится (и покажет ошибку), если домашний каталог и каталог установки совпадают.
    -installdir <dir>Переопределяет код, определяющий, где был установлен ZAP, в указанном каталоге.
    -hПоказывает все доступные параметры командной строки, в том числе добавленные надстройками.
    -helpТо же, что и -h
    -newsession <path>Создает новый сеанс в указанном месте
    -session <path>Открывает данную сессию после запуска ZAP
    -host <host>Переопределяет хост, используемый для проксирования, указанный в файле конфигурации
    -port <port>Переопределяет порт, используемый для проксирования, указанный в файле конфигурации
    -lowmemПо возможности используйте базу данных вместо памяти - это все еще экспериментально
    -experimentaldbИспользуйте экспериментальный общий код базы данных, который, что неудивительно, также все еще является экспериментальным.
    -nostdoutОтключает ведение журнала по умолчанию через стандартный вывод
    -silentГарантирует, что ZAP не отправляет нежелательные запросы, в том числе проверяет наличие обновлений.
    -addoninstall <addOnId>Устанавливает надстройку с указанным идентификатором из ZAP Marketplace. С идентификаторами надстроек, доступных на торговой площадке, можно ознакомиться на вкладке Marketplace в Диалог управления надстройками .
    -addoninstallallУстановите все доступные дополнения из ZAP Marketplace
    -addonuninstall <addOnId>Удаляет надстройку с указанным ID. Использовать -addonlist или проверьте вкладку "Установлено" в Диалог управления надстройками чтобы узнать идентификаторы установленных надстроек.
    -addonupdateОбновите все измененные дополнения из ZAP Marketplace
    -addonlistСписок всех установленных дополнений
    -script <script>Запустите указанный сценарий (путь к файловой системе), если командная строка / демон, или просто загрузите его, если графический интерфейс
    -suppinfoВыводит детали, относящиеся к поддержке и устранению неполадок (на консоль / стандартный выход). Например: версия ZAP, версия java, установленные надстройки и версия, информация о локали, операционная система и т. д.

Опции -session и -newsession являются взаимоисключающими. Будет показано сообщение об ошибке и ZAP завершится (если не в графическом интерфейсе), когда установлены обе опции.
Относительные пути к файлу сеанса разрешаются относительно каталога «сеанса», расположенного в домашнем каталоге ZAP (по умолчанию или указанного с помощью -dir вариант).
Ключи конфигурации следует указывать с использованием точечной нотации в зависимости от их расположения в XML файла конфигурации, например: 
<zap-script> -config api.key=12345
Обратите внимание, что надстройки могут добавлять дополнительные параметры командной строки.

Примеры:

  • Запустите ZAP в режиме демона с новым сеансом, созданным по заданному пути: 
    <zap-script> -daemon -newsession session
  • Запустите сценарий для существующего сеанса и выйдите из ZAP после завершения: 
    <zap-script> -session /full/path/to/existing/session -script /full/path/to/script.js -cmd
