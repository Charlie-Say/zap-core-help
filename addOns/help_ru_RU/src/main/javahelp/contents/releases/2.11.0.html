выпуск 2.11.0

Это исправление ошибок и усовершенствованный выпуск, посвященный 20-летию OWASP, для которого требуется как минимум Java 8. Обратите внимание, что рекомендуется как минимум Java 11, особенно для дисплеев с высоким разрешением.

Эти примечания к выпуску не включают все изменения, включенные в надстройки, обновленные с 2.10.0.

Некоторые из наиболее значительных улучшений включают:

Теги предупреждений

Оповещения теперь могут быть помечены произвольными ключами или парами ключ = значение.
Правила активного и пассивного сканирования были обновлены и теперь включают теги для OWASP Top 10 за 2021 и 2017 годы. показаны на страницах с подробностями об оповещениях веб-сайта.

Платформа автоматизации

Automation Framework - это новый способ автоматизации ZAP, который, как ожидается, станет вариантом по умолчанию для большинства случаев использования. Для получения дополнительных сведений см. страницу Automation Framework на веб-сайте.

Генерация отчетов

Новое дополнение Report Generation позволяет создавать много более гибкие отчеты с доступом к гораздо большему количеству данных. Предыдущие надстройки отчетности были удалены с рынка, так как они предоставляют меньше функциональных возможностей и больше не поддерживаются.
Новые шаблоны отчетов включают:
  • HTML-код риска и уверенности - новый отчет по умолчанию
  • Современный HTML-отчет с темами и параметрами
  • Образец отчета высокого уровня
  • Традиционный HTML-отчет с запросами и ответами
Добавлены «традиционные» шаблоны, которые соответствуют старым отчетам для всех, кто полагается на их форматы.

Поддержка OAST

Новое дополнение OAST Support позволяет вам находить и использовать внеполосные уязвимости. Это дополнение является альфа-версией на момент выпуска 2.11.0, но, как ожидается, скоро будет обновлено, поэтому проверьте страницы справки, чтобы узнать о последних функциях.

Повторное тестирование

Новое дополнение Retest позволяет повторно тестировать наличие / отсутствие ранее сгенерированных предупреждений. Это дополнение является альфа-версией на момент выпуска 2.11.0, но, как ожидается, скоро будет обновлено, поэтому проверьте страницы справки, чтобы узнать о последних функциях.

Docker

Стабильные и голые образы Docker теперь будут обновляться ежемесячно, обычно примерно в начале месяца.
Обновления будут включать все обновленные надстройки и любые изменения упакованных сканирований. В эти обновления не будут включены основные изменения.
Если вы хотите использовать определенную версию, изображения будут помечены по дате.
Пакетные отсканированные изображения переносятся для использования Automation Framework - эта миграция будет продолжаться в течение срока службы 2.11.0.

Статистика

Значительное количество статистических данных было добавлено в ядро и добавлено в дополнения. Частично это обеспечивается Automation Framework, который может напрямую использовать статистику для проверок работоспособности.
Новые основные статистические данные включают:
  • stats.api.call. <format>. <component>. <request-type>. <name>: количество вызовов данной конечной точки API
  • stats.api.error. <format>. <component>. <request-type>. <name>: сколько раз данная конечная точка API возвращала ошибку
  • stats.ascan. <rule-id> .alerts: количество предупреждений, сгенерированных данным активным правилом сканирования.
  • stats.ascan. <rule-id> .skipped: количество предупреждений, пропущенных для данного активного правила сканирования.
  • stats.ascan. <rule-id> .started: количество предупреждений, запущенных для данного активного правила сканирования.
  • stats.ascan. <rule-id> .time: совокупное количество миллисекунд, в течение которых выполнялось данное активное правило сканирования.
  • stats.ascan. <rule-id> .urls: количество URL-адресов, запрошенных данным активным правилом сканирования.
  • stats.ascan.started: количество запусков активного сканера.
  • stats.ascan.stopped: сколько раз активный сканер был остановлен (в отличие от завершения).
  • stats.ascan.time: общее количество миллисекунд, в течение которых активный сканер проработал.
  • stats.ascan.urls: количество URL-адресов, запрошенных активным сканером.
  • stats.break.drop: сколько раз запрос или ответ был отброшен через точку останова.
  • stats.break.hit: количество срабатываний точки останова.
  • stats.break.step: количество переходов через точку останова.
  • stats.pscan. <rule-id> .alerts: количество предупреждений, сгенерированных данным правилом сканирования.
  • stats.pscan. <rule-id> .time: совокупное количество миллисекунд, затраченных на выполнение данного правила сканирования.
  • stats.script.call. <engine-name>. <type>: количество вызовов данного типа скрипта.
  • stats.script.error. <engine-name>. <type>: сколько раз данный тип скрипта возвращал ошибку
  • stats.spider.started: количество запусков паука.
  • stats.spider.stopped: сколько раз паук был остановлен (в отличие от завершения)
  • stats.spider.time: общее количество миллисекунд, в течение которых паук пробежал все сканирование.
  • stats.spider.url.error: количество URL-адресов, которые паук нашел, но не смог получить доступ.
  • stats.spider.url.found: количество URL-адресов, которые паук нашел и к которым получил доступ.
Для получения подробной информации о последней статистике, включая все те, которые также поддерживаются надстройками, и ссылки на код см. страницы внутренней статистики ZAP веб-сайта.

Дополнения

Новые дополнения

Следующие надстройки включены по умолчанию в этот выпуск впервые:
  • Automation Framework - простой, гибкий и мощный способ автоматизации ZAP
  • Поддержка OAST - позволяет использовать внеполосные уязвимости.
  • Создание отчетов - новые настраиваемые отчеты
  • Retest - повторный тест на наличие / отсутствие ранее сгенерированных предупреждений

Обновить Дополнения (Add-Ons):

Все дополнения, включенные по умолчанию, были обновлены с момента последней полной версии.

Обновления Docker

В последний образ стабильного Docker включены следующие изменения:
  • Обновлено для использования Webswing 21.1.5.
  • Добавлен файл / zap / container, чтобы упростить определение того, работаем ли мы в контейнере, таком как docker.
  • Изменено для включения интеграционных тестов, включая включение AF для базовой опции -c , если перед ней используется флаг --auto .
  • Изменено использование домашнего каталога пользователя для файлов Automation Framework, поэтому он будет работать для любого пользователя.
  • Обновлен базовый план для использования Automation Framework по умолчанию для общих параметров, включая файл конфигурации / URL-адрес.
  • Alert_on_Unexpected_Content_Types.js & gt; Добавлен Content-Type text / yaml в список ожидаемых типов.
  • Проверьте, исключаются ли сообщения, анализируемые скриптами сканирования API, глобально.
  • Обеспечение большей гибкости при указании параметров командной строки ZAP при использовании Webswing
  • Python 3.5 больше не поддерживается.
  • Обновите Webswing, чтобы загрузить версию продукта, если предоставлен действительный ключ.
Полный список изменений, внесенных в образы докеров, см. В докере CHANGELOG.md .

Новые конечные точки ZAP API:

ACTION ascan / setOptionScanNullJsonValues

Устанавливает, должен ли активный сканер сканировать нулевые значения JSON.

ПРОСМОТР ascan / optionScanNullJsonValues

Сообщает, должен ли активный сканер сканировать нулевые значения JSON.
