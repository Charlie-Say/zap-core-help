выпуск 2.6.0

Этот выпуск содержит большое количество изменений, в частности, в ZAP API.
Мы добавили значительное количество новых конечных точек API, работая над нашей целью создание ZAP полностью контролируемой через API. Мы также изменили некоторые из существующих конечных точек, и во всех случаях эти изменения являются обратной совместимой.

Полный выпуск также включает в себя новый дополнительный jxbrowser, а также веб-передачи платформы, чтобы сделать его еще проще Взаимодействовать с ZAP через широкий спектр браузеров.

Изменения безопасности API Мы изменили безопасность API в ответ на вопросы, сообщаемые нам через нашу программу Bounty BUG . Детали уязвимостей приведены ниже.
Изменения безопасности являются необходимостью, не совместимыми назад, хотя у нас есть варианты отключения их, если вы используете ZAP в безопасной среде.

По умолчанию все вызовы API теперь требуют либо клавиши API или NONCE. Они могут поставляться через параметры URL, параметры публикации или заголовки. Поддерживаемые клиенты stap API (включая Java и Python) были обновлены для снабжения ключа API через заголовок. Одноразовые номера генерируются ZAP и предназначены для использования надстройками ZAP, которым необходим доступ к ZAP API. Для получения полной информации см. экран API параметров .

Есть набор новых опций API, связанных с безопасностью:

  • Пользовательский интерфейс включен - если этот параметр включен, веб-интерфейс API доступен для всех компьютеров, которые могут использовать ZAP в качестве прокси. По умолчанию это включено.
  • IP-адреса, которым разрешено использовать API - по умолчанию только компьютер, на котором запущен ZAP, может получить доступ к ZAP API. Вы можете разрешить другим машинам доступ к API, добавив подходящие шаблоны регулярных выражений. Вам следует добавлять только те IP-адреса, которым вы доверяете.
  • Не требовать ключ API для безопасных операций - если этот параметр включен, ключ API не требуется для представлений или других операций, которые считаются «безопасными», другими словами для операций, которые не вносят никаких изменений в ZAP. Однако такие операции предоставляют доступ к данным ZAP, таким как предупреждения, сообщения и пути к файловой системе. Они также могут использоваться веб-приложениями для обнаружения ZAP.
  • Сообщать об ошибках разрешений через API - если этот параметр включен, ZAP будет сообщать об ошибках разрешений через API, который может использоваться веб-приложениями для обнаружения присутствия ZAP. Это не является серьезной проблемой в безопасной среде, но если вы используете ZAP против потенциально вредоносных сайтов, вам не следует включать его.
Все параметры ZAP можно указать через командную строку при запуске ZAP - см. https://www.zaproxy.org/faq/why-is-an-api-key-required-by-default/ для получения полной информации.
Мы также добавили в API еще больше заголовков безопасности, включая строгую политику безопасности контента.

Измененные конечные точки ZAP API:

ДЕЙСТВИЕ ascan / scan

Параметр url теперь является необязательным, и был добавлен необязательный параметр contextId. Вы должны предоставить один из них.

ДЕЙСТВИЕ ascan / scanAsUser

Параметры url и contextId теперь необязательны. Вы должны предоставить один из них.

ДЕЙСТВИЕ ascan / addScanPolicy

Добавлены необязательные параметры alertThreshold и attackStrength.

Новые конечные точки ZAP API:

ПРОСМОТР сканирования / опции Макс. Продолжительность правила в минутах

Возвращает максимальное время в минутах, в течение которого может выполняться правило сканирования, ноль не ограничено.

ПРОСМОТР сканирования / опции Макс.длительность сканирования в минутах

Возвращает максимальное время в минутах, в течение которого может выполняться полное сканирование, ноль не ограничено.

ДЕЙСТВИЕ ascan / setOptionMaxRuleDurationInMins

Устанавливает максимальное время в минутах, в течение которого может выполняться правило сканирования, ноль не ограничено.

ДЕЙСТВИЕ ascan / setOptionMaxScanDurationInMins

Устанавливает максимальное время в минутах, в течение которого может выполняться полное сканирование, ноль не ограничено.

ДЕЙСТВИЕ ascan / updateScanPolicy

Обновляет указанную политику сканирования с указанным значением alertThreshold или attackStrength.

ПОСМОТРЕТЬ break / isBreakAll

Возвращает True, если ZAP нарушит и запросы, и ответы.

ПОСМОТРЕТЬ перерыв / isBreakRequest

Возвращает True, если ZAP прерывается при запросах.

ПОСМОТРЕТЬ перерыв / isBreakResponse

Возвращает True, если ZAP прерывается при ответах.

ПРОСМОТР паузы / httpMessage

Возвращает перехваченное в данный момент HTTP-сообщение (если есть).

ДЕЙСТВИЕ перерыв / перерыв

Управляет функцией глобального разрыва. Тип может быть одним из: http-all, http-request или http-response. Состояние может быть истинным (для включения прерывания для указанного типа) или ложным (для выключения прерывания). Область видимости в настоящее время не используется.

ДЕЙСТВИЕ break / setHttpMessage

Перезаписывает текущее перехваченное сообщение предоставленными данными.

ДЕЙСТВИЕ прервать / продолжить

Отправляет текущее перехваченное сообщение и сбрасывает глобальные точки останова запроса / ответа.

ДЕЙСТВИЕ перерыв / шаг

Отправляет текущее перехваченное сообщение, следующий запрос или ответ будет автоматически перехвачен.

ДЕЙСТВИЕ перерыв / падение

Удаляет текущее перехваченное сообщение.

ПРОСМОТР core / optionDnsTtlSuccessfulQueries

Получает TTL (в секундах) успешных DNS-запросов.

ДЕЙСТВИЕ ядро / sendRequest

Отправляет HTTP-запрос, при необходимости следуя перенаправлениям. Возвращает отправленный запрос и полученный ответ, а также последующие перенаправления, если таковые имеются. Режим принудительно применяется при отправке запроса (и последующих перенаправлениях), пользовательские ручные запросы не разрешены ни в «Безопасном» режиме, ни в «Защищенном» режиме, если они выходят за рамки.

RCE через тестовую форму Anti CSRF и раскрытие ключей API

Если пользователь использовал тестовую форму Anti CSRF против специально созданной HTML-страницы, то ключ API просочился на этот сайт. После этого сайт сможет получить доступ к ZAP API и выполнить любое действие, включая загрузку скриптов ZAP. Сценарии могут быть загружены только из «локальных» файловых систем, но если пользователь запускает ZAP в Windows, злоумышленник может сделать вредоносный сценарий доступным через общедоступный общий ресурс SMB. ZAP считает, что это локальный файл, поэтому сценарий загружается и может запускаться через API.
Требование ключа API или одноразового номера для всех операций API является прямым результатом этой уязвимости, так же как и изменение надстроек для использования одноразовых номеров для снижения риска утечки ключа API.

Предоставлено: Артемий Богданов (@ Abr1k0s)
Артемий был награжден премией в размере 1000 долларов за ошибку в результате этой заявки. Это первая выплаченная нами награда за устранение ошибок. Поздравляю, Артемий!

Установщик Windows уязвим для перехвата DLL

Установщик Windows ZAP для всех версий до 2.5.0 включительно уязвим для перехвата DLL в Windows 7 (и более ранних версиях). Это уязвимость в стороннем установщике InnoSetup. Установщики 2.6.0 (на всех платформах) теперь создаются с помощью Install4J.

Если по какой-то причине вам все же необходимо установить предыдущие версии ZAP в Windows 7 или более ранней версии, мы рекомендуем вам переместить установщик в чистый каталог перед его запуском.

Обратите внимание, что Burp Suite также использует Install4J, поэтому будущие уязвимости в установщиках, созданных с помощью Install4j, могут подпадать под программу вознаграждений за ошибки Burp Suite: https://hackerone.com/portswigger

Предоставлено: Джеймс Кеттл (Burp Suite).

Выполнение произвольного кода с помощью инъекции параметров приложений Invoke

Параметры HTML могут быть специально созданы, чтобы вызывать выполнение произвольного кода, если пользователь решит вызвать целевое приложение с запросом, содержащим этот параметр, из ZAP.
Надстройка Invoke Applications была обновлена для решения этой проблемы - всем пользователям ZAP следует установить эту новую версию, прежде чем продолжить использование надстройки.
