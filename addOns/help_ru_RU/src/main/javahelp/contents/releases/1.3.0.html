Выпуск 1.3.0

В этом выпуске были внесены следующие изменения:

Важные перемены:

Fuzzing

Строки в Ответе (Response) теперь могут быть заполнены, чтобы попытаться найти уязвимости.
Токены анти-CRSF могут быть обнаружены и автоматически восстановлены при фаззинге.
Эта функциональность основана на коде из проекта OWASP JBroFuzz.

Динамические SSL-сертификаты

Поддержка SSL-соединений была улучшена и упрощена.
Теперь пользователи могут создать свой собственный корневой сертификат и распространить его среди своих HTTP-клиентов.
Видеть Вариант: динамические сертификаты SSL больше подробностей.

Режим демона и API

Запуск ZAP с параметром командной строки «-daemon» приведет к его запуску в фоновом режиме в 'headless' режиме, Это означает, что пользовательский интерфейс не отображается.
Начальный API был реализован в XML, JSON и HTML.
Если ZAP работает как демон, то API включается автоматически, в противном случае API должен быть включен через экран Options API.
По API можно перемещаться, открыв в браузере http: // zap / при проксировании через ZAP.

Интеграция с BeanShell

BeanShell представляет собой интерактивную оболочку Java, которую можно использовать для выполнения сценариев BeanShell.
Эти сценарии представляют собой упрощенную форму Java, в которой используются многие элементы синтаксиса Java, но в более простом формате сценариев.
Весь код Java также является допустимым кодом BeanShell.
Интеграция BeanShell в OWASP ZAP позволяет писать сценарии с использованием функций ZAP и набора данных.
Это может быть очень мощной функцией для анализа веб-приложений.

Полная интернационализация

Вся отображаемая строка теперь полностью интернационализирована.

Локализация

Незначительные изменения:

hex (Шестнадцатеричный) вид

Запрос, Ответ вкладки теперь предоставляют опцию «Hex View», которая отображает тело запроса и ответа в шестнадцатеричном формате.

Результаты поиска

Вкладка поиска теперь отображает все экземпляры найденной строки, а не только первый экземпляр в каждом запросе или ответе.

Исключить URLы

URL-адреса могут быть явно исключены из активного сканера, паука и прокси.

Поддержка копирования

На большинстве панелей теперь есть опция меню «Копировать», которую можно щелкнуть правой кнопкой мыши, в том числе панели «Сканирование портов» и Грубая сила (Brute Force).

Поддержка Undo / Redo

Все поля ввода теперь поддерживают действия Undo / Redo с использованием ускорителей Undo / Redo по умолчанию операционной системы:
  • Windows/Linux: Ctrl+Z / Ctrl+Y
  • Mac OS X: Cmd+Z / Cmd+Shift+Z

Поддержка прокси-сервера сканера портов и возможность тайм-аута

Сканер портов теперь может использовать исходящий прокси-сервер (если настроен), также теперь можно установить тайм-аут в миллисекундах.

Кнопки разрыва запроса и ответа

Теперь есть 2 кнопки «Установить перерыв», позволяющие устанавливать перерывы для всех запросов и всех ответов независимо.

Разверните кнопки вкладок "Сайты" и "Информация"

Теперь есть 2 кнопки, с помощью которых можно переключаться между развернутыми вкладками «Сайты» и «Информация».

Значок вкладки разрыва меняет цвет при достижении точки останова

Пока вкладка "Перерыв" не используется, на ней отображается серый крестик:    .
Когда точка останова при нажатии значок вкладки изменяется на красный крест: & nbsp;  .

Регулируемый тайм-аут

Вариант: подключение экран позволяет установить тайм-аут в секундах чтобы упростить тестирование медленных приложений.

Обновления библиотеки

Большинство библиотек, используемых ZAP, обновлены до последних версий.

Известные вопросы:

Mac OS X: динамический SSL и Google Chrome

В настоящее время динамический SSL не работает при использовании Google Chrome. Это связано с нерешенной известной проблемой с Google Chrome и связкой ключей Mac OS X. При импорте корневого центра сертификации OWASP ZAP в связку ключей и запросе веб-сайта SSL отображается сообщение об ошибке «Недействительный сертификат».
