выпуск 2.7.0

Это исправление ошибок и усовершенствование выпуска, который требует минимум Java 8.

Некоторые из наиболее важных усовершенствований включают:
  • Запуск браузера включен по умолчанию - это позволяет запускать браузеры из ZAP, которые предварительно настроены на прокси через zap и игнорировать предупреждения о сертификате из-за зап корневого сертификата.
  • Разрешить ZAP прослушивать несколько адресов / портов
  • Индикация Имени Сервера
  • Обновленная реализация механизма NTLM - это устраняет случаи, когда домен проверяется, и улучшает взаимодействие с другими (серверными) реализациями NTLM.
  • Множество новых конечных точек API - подробности см. Ниже
Обратите внимание: если у вас есть какие-либо проблемы с этим выпуском, есть также новый пункт меню «Справка / информация о поддержке ...». который предоставляет важную информацию об установке ZAP, которую вы должны включать в любые возникающие у вас проблемы.

ZAP API Breaking Changes:

ACTION authentication / setAuthenticationMethod

The authentication type "formBasedAuthentication" will now require the login URL always in encoded form. The change ensures the login URL is used/sent as it was specified. Previously it would accept partially encoded URLs but they would be re-encoded when used, potentially leading to a different URL being used/sent.

VIEW context / contextList

Это изменение сломает потребителей, которые вручную разбирали / извлекали имена из строки. Структура данных возвращенный был изменен, чтобы правильно отделить каждое имя: 

{"contextList":["Context 1","Context 2"]}
and: 

<contextList type="list">
    <contextName>Context 1</contextName>
    <contextName>Context 2</contextName>
</contextList>
instead of: 

{"contextList":"[Context 1, Context 2]"}
and: 

<contextList>[Context 1, Context 2]</contextList>

VIEW core / setOptionUseProxyChain

Changed to return FAIL если исходящий прокси-сервер не был включен (так как требуемый адрес / имя хоста не был ранее установленный), прежде чем он возвратит всегда OK.

ZAP API Changed Endpoints:

VIEW core / alerts

Added optional riskId parameter to facilitate filtering the risk. Where riskId is in the range 0 for Informational and 3 for High.

VIEW core / numberOfAlerts

Added optional riskId parameter to facilitate filtering the risk. Where riskId is in the range 0 for Informational and 3 for High.

VIEW core / urls

Added optional baseurl parameter, to filter the URLs that are returned.

VIEW ascan / scans

Изменено, чтобы также возвращать общее количество предупреждений и сообщений, отправленных во время каждой проверки.

VIEW ascan / scanProgress

Изменено, чтобы также возвращать количество предупреждений, вызванных каждым сканером.

ZAP API New Endpoints:

VIEW core / alertsSummary

Новое сводное представление для оповещений, в котором отображаются счетчики оповещений на уровень риска. При необходимости, отфильтрованный по значению автоопределения. 

{"High":0,"Low":132,"Medium":39,"Informational":153}

VIEW core / messagesById

Gets the HTTP messages with the given IDs.

VIEW core / messagesHarById

Gets the HTTP messages with the given IDs, in HAR format.

VIEW core / optionAlertOverridesFilePath

Gets the path to the file with alert overrides.

VIEW core / optionMaximumAlertInstances

Возвращает Максимальное число экземпляров предупреждений для включения в отчет.

VIEW core / optionMergeRelatedAlerts

Возвращает, будут ли связанные оповещения объединяться в создаваемых отчетах.

VIEW core / zapHomePath

Gets the path to ZAP's home directory.

VIEW localProxies / additionalProxies

Возвращает все дополнительные прокси-серверы, которые были настроены.

VIEW spider / optionAcceptCookies

Получает ли процесс Spider следует принимать куки, а пауки.

ACTION core / deleteAlert

Deletes the alert with the given ID.

ACTION core / setOptionAlertOverridesFilePath

Задает (или очищает, если пусто) путь к файлу с переопределениями предупреждений.

ACTION core / setOptionMaximumAlertInstances

Задает Максимальное число экземпляров предупреждений для включения в отчет. Нулевое значение считается неограниченным.

ACTION core / setOptionMergeRelatedAlerts

Указывает, будут ли объединены связанные предупреждения в создаваемых отчетах.

ACTION ascan / importScanPolicy

Импортирует политику сканирования по заданному пути к файловой системе.

ACTION ascan / skipScanner

Пропускает сканер с помощью идентификаторов сканирования и сканера.

VIEW localProxies / addAdditionalProxy

Добавляет новый прокси, используя предоставленные сведения. Увидеть Параметры экрана локальные прокси для деталей параметров.

VIEW localProxies / removeAdditionalProxy

Удаляет дополнительный прокси с указанным адресом и портом.

ACTION spider / setOptionAcceptCookies

Устанавливает, будет ли процесс Спайдер должен принимать куки, а пауки.
