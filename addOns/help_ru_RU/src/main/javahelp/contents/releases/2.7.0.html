выпуск 2.7.0

Это исправление ошибок и усовершенствование выпуска, который требует минимум Java 8.

Некоторые из наиболее важных усовершенствований включают:
  • Запуск браузера включен по умолчанию - это позволяет запускать браузеры из ZAP, которые предварительно настроены на прокси через zap и игнорировать предупреждения о сертификате из-за зап корневого сертификата.
  • Разрешить ZAP прослушивать несколько адресов / портов
  • Индикация Имени Сервера
  • Обновленная реализация механизма NTLM - это устраняет случаи, когда домен проверяется, и улучшает взаимодействие с другими (серверными) реализациями NTLM.
  • Множество новых конечных точек API - подробности см. Ниже
Обратите внимание: если у вас есть какие-либо проблемы с этим выпуском, есть также новый пункт меню «Справка / информация о поддержке ...». который предоставляет важную информацию об установке ZAP, которую вы должны включать в любые возникающие у вас проблемы.

Критические изменения ZAP API:

ACTION аутентификация / setAuthenticationMethod

Тип аутентификации «formBasedAuthentication» теперь требует, чтобы URL-адрес входа всегда был в закодированной форме. Это изменение обеспечивает URL-адрес для входа используется / отправляется, как было указано. Раньше он принимал частично закодированные URL-адреса, но они перекодировались, когда используется, что может привести к использованию / отправке другого URL-адреса.

ПРОСМОТР context / contextList

Это изменение сломает потребителей, которые вручную разбирали / извлекали имена из строки. Структура данных возвращенный был изменен, чтобы правильно отделить каждое имя: 

{"contextList":["Context 1","Context 2"]}
и: 

<contextList type="list">
    <contextName>Context 1</contextName>
    <contextName>Context 2</contextName>
</contextList>
вместо того: 

{"contextList":"[Context 1, Context 2]"}
и: 

<contextList>[Context 1, Context 2]</contextList>

ПРОСМОТР ядра / setOptionUseProxyChain

Изменено на возврат НЕУДАЧА если исходящий прокси-сервер не был включен (так как требуемый адрес / имя хоста не был ранее установленный), прежде чем он возвратит всегда OK.

Измененные конечные точки ZAP API:

ПРОСМОТР ядра / предупреждений

Добавлен необязательный параметр riskId для облегчения фильтрации риска. Если riskId находится в диапазоне 0 для информационных и 3 для High.

ПОСМОТРЕТЬ core / numberOfAlerts

Добавлен необязательный параметр riskId для облегчения фильтрации риска. Если riskId находится в диапазоне 0 для информационных и 3 для High.

ПОСМОТРЕТЬ ядро / URL

Добавлен необязательный baseurl параметр, чтобы отфильтровать возвращаемые URL-адреса.

ПРОСМОТР ascan / scans

Изменено, чтобы также возвращать общее количество предупреждений и сообщений, отправленных во время каждой проверки.

ПРОСМОТР сканирования / выполнения сканирования

Изменено, чтобы также возвращать количество предупреждений, вызванных каждым сканером.

Новые конечные точки ZAP API:

ПРОСМОТР ядра / предупреждений

Новое сводное представление для оповещений, в котором отображаются счетчики оповещений на уровень риска. При необходимости, отфильтрованный по значению автоопределения. 

{"High":0,"Low":132,"Medium":39,"Informational":153}

ПРОСМОТР core / messagesById

Получает сообщения HTTP с заданными идентификаторами.

ПРОСМОТР core / messagesHarById

Получает сообщения HTTP с заданными идентификаторами в формате HAR.

ПРОСМОТРЕТЬ core / optionAlertOverridesFilePath

Получает путь к файлу с переопределением предупреждений.

ПРОСМОТР ядра / optionMaximumAlertInstances

Возвращает Максимальное число экземпляров предупреждений для включения в отчет.

ПРОСМОТР ядра / optionMergeRelatedAlerts

Возвращает, будут ли связанные оповещения объединяться в создаваемых отчетах.

ПОСМОТРЕТЬ core / zapHomePath

Получает путь к домашнему каталогу ZAP.

ПОСМОТРЕТЬ localProxies / additionalProxies

Получает все дополнительные настроенные прокси.

ПОСМОТРЕТЬ паука / optionAcceptCookies

Получает ли процесс Spider следует принимать куки, а пауки.

ДЕЙСТВИЕ core / deleteAlert

Удаляет оповещение с заданным идентификатором.

ДЕЙСТВИЕ core / setOptionAlertOverridesFilePath

Задает (или очищает, если пусто) путь к файлу с переопределениями предупреждений.

Ядро ACTION / setOptionMaximumAlertInstances

Задает Максимальное число экземпляров предупреждений для включения в отчет. Нулевое значение считается неограниченным.

Ядро ACTION / setOptionMergeRelatedAlerts

Указывает, будут ли объединены связанные предупреждения в создаваемых отчетах.

ДЕЙСТВИЕ ascan / importScanPolicy

Импортирует политику сканирования по заданному пути к файловой системе.

ДЕЙСТВИЕ ascan / skipScanner

Пропускает сканер с помощью идентификаторов сканирования и сканера.

ПРОСМОТРЕТЬ localProxies / addAdditionalProxy

Adds a new proxy using the details supplied.

ПРОСМОТРЕТЬ localProxies / removeAdditionalProxy

Removes the additional proxy with the specified address and port.

Паук ДЕЙСТВИЯ / setOptionAcceptCookies

Sets whether or not a spider process should accept cookies while spidering.

Windows Uninstaller уязвимы для dll угон

The ZAP Windows Uninstaller for 2.6.0 is vulnerable to DLL Hijacking on Windows. This was a vulnerability in the 3rd party installer Install4j which has now been fixed.
Note that this can only occur if a malicious DLL is already on the path.
