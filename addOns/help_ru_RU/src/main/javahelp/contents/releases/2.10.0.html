выпуск 2.10.0

Это 10-летний юбилейный выпуск с исправлением ошибок и улучшением, для которого требуется как минимум Java 8. Обратите внимание, что рекомендуется как минимум Java 11, особенно для дисплеев с высоким разрешением.

Эти примечания к выпуску не включают все изменения, включенные в надстройки, обновленные с версии 2.9.0.

Некоторые из наиболее значительных улучшений включают:

Пользовательские страницы

Пользовательские страницы могут быть определены для каждого контекста - они позволяют ZAP определять различные нестандартные условия обработки ошибок, такие как настраиваемые страницы ошибок, и обрабатывать их более эффективно.

Опрос проверки подлинности

Была представлена концепция Стратегии проверки подлинности , которая позволяет ZAP обрабатывать более широкий спектр механизмов аутентификации, включая возможность опроса указанной страницы на предмет статуса аутентификации пользователя.

Управление деревом сайта

Сценарии и надстройки теперь имеют полный доступ к тому, как узлы представлены в дереве сайтов . Как Input Vector Scripts , так и надстройки, которые включают реализации Variant класс может изменять как дерево структура и имена, используемые для новых узлов.

Для получения дополнительных сведений см. Сообщение блога Модификаторы дерева сайта .

Dynamic Look and Feel, включая темный режим

Пользовательский интерфейс рабочего стола включает новый набор внешнего вида Look and Feel с открытым исходным кодом FlatLaf , включая 2 варианта темного режима.
Вы также можете динамически переключать внешний вид с помощью кнопки на панели инструментов верхнего уровня .

Дополнительные сведения о темном режиме см. В записи блога Темный режим в еженедельном выпуске .

Заголовки аутентификации через Env Vars

Доступен новый набор переменных среды, которые позволяют легко добавлять заголовок аутентификации ко всем запросам, которые передаются через ZAP или инициируются. инструментами ZAP, включая пауков и активный сканер. Они задокументированы на странице Аутентификация .

Конфигурация прокси SOCKS

It is now possible to dynamically configure the outgoing SOCKS proxy in the Options' Connection screen. По умолчанию конфигурация прокси-сервера SOCKS применяется ко всем соединениям, выполненным ZAP.

Кешированные скрипты

Новые дополнения

Следующие надстройки включены по умолчанию в этот выпуск впервые:
  • Расширенное диалоговое окно кодирования / декодирования / хеширования - заменяет старый основной диалог кодирования / декодирования / хеширования.

Обновления Docker

В последний образ стабильного Docker включены следующие изменения:
  • Обновите Webswing до последней версии (20.2.1) для работы с более новыми версиями Java.
  • Обновите Java в стабильном образе до версии 11.
  • Добавьте поддержку проверок с проверкой подлинности.
  • Добавьте функцию zap_tune (отключите все теги и ограничьте количество предупреждений pscan до 10), обработчик zap_tuned и отключите журнал восстановления.
  • Обновите zap-api-scan.py, чтобы добавить поддержку GraphQL.
  • Сбой немедленно, если сканирование пауком не было запущено, чтобы предоставить более точное сообщение об ошибке.
  • Пакетное сканирование будет использовать предоставленный контекст при сканировании и активном сканировании.
  • Добавьте в образ контейнера переменную среды IS_CONTAINERIZED, используемую в скрипте python для проверки контейнерных сред (например, containerd), не полагаясь на файлы, специфичные для среды выполнения контейнера.
  • Сделайте podman совместимым
  • Сделайте docker стабильным, используйте ubuntu 20.04
  • Сделайте так, чтобы команда `python` использовала Python 3.
  • Удален python 2, в дальнейшем будет поддерживаться только python 3.
  • Изменены zap-full-scan.py и zap-api-scan.py, чтобы включить параметр -I для игнорирования только предупреждений, используемых zap-baseline-scan.py
Полный список изменений, внесенных в образы докеров, см. В докере CHANGELOG.md .

Библиотека ведения журналов Log4j была обновлена до новой основной версии (2.x). Предыдущая версия все еще включена для совместимости с существующим кодом, авторам надстроек и скриптов рекомендуется перейти на новый API.
Например, теперь логгер можно получить с помощью: 
import org.apache.logging.log4j.LogManager;
import org.apache.logging.log4j.Logger;


Logger logger = LogManager.getLogger(MyClass.class);

Новые конечные точки ZAP API:

ПРОСМОТРЕТЬ ascrf / optionPartialMatchingEnabled

Определите, должен ли ZAP обнаруживать токены CSRF путем поиска частичных совпадений

ДЕЙСТВИЕ ascrf / setOptionPartialMatchingEnabled

Определите, должен ли ZAP обнаруживать токены CSRF путем поиска частичных совпадений.

ДЕЙСТВИЕ оповещение / updateAlertsConfidence

Обновите достоверность предупреждений.

ACTION alert / updateAlertsRisk

Обновите риск предупреждений.

ACTION контекст / setContextCheckingStrategy

Установите стратегию проверки для контекста - это определяет, как ZAP проверяет аутентификацию запроса.

ПОСМОТРЕТЬ core / optionUseSocksProxy

Получает, следует ли использовать прокси-сервер SOCKS.

ACTION core / setOptionUseSocksProxy

Устанавливает, следует ли использовать прокси-сервер SOCKS.

ПРОСМОТР пользователей / setAuthenticationState

Получает информацию о состоянии аутентификации для пользователя, определенного контекстом и идентификаторами пользователя.

ПРОСМОТР пользователей / получение аутентифицированного сеанса

Получает информацию о сеансе аутентификации для пользователя, определенного контекстом и идентификаторами пользователя, например файлы cookie и учетные данные области.

Действие пользователей / AuthenticateAsUser

Пытается аутентифицироваться как идентифицированный пользователь, возвращая запрос аутентификации и выясняя, был ли он успешным.

Пользователи действий / pollAsUser

Пытается выполнить опрос от имени идентифицированного пользователя, возвращая запрос аутентификации и выясняя, был ли он успешным. Это будет работать, только если настроена стратегия проверки опроса.

Действие пользователей / setAuthenticationState

Устанавливает поля в состоянии аутентификации для пользователя, идентифицированного контекстом и идентификаторами пользователя.

Действие пользователей / setCookie

Устанавливает указанный файл cookie для пользователя, определенного контекстом и идентификаторами пользователя.
