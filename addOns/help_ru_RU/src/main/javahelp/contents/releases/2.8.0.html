выпуск 2.8.0

Это выпуск с исправлением ошибок и улучшением, для которого требуется как минимум Java 8. Обратите внимание, что рекомендуется как минимум Java 11, особенно для дисплеев с высоким разрешением.

Эти примечания к выпуску не включают все изменения, включенные в надстройки, обновленные с версии 2.7.0.

Некоторые из наиболее значительных улучшений включают:

Heads Up Display

Дисплей ZAP Heads Up Display (HUD) предоставляет все основные функции ZAP в вашем браузере. Он идеально подходит для людей, плохо знакомых с веб-безопасностью, а также позволяет опытным тестерам на проникновение сосредоточиться на функции приложения, предоставляя при этом ключевую информацию о безопасности и функциональные возможности.
HUD можно включить при проксировании браузера через ZAP с помощью нового экрана «Ручное исследование» и кнопки на панели инструментов верхнего уровня.

Экраны быстрого запуска

Вкладка «Быстрый старт» была переработана и теперь содержит 3 экрана:
  • Автоматическое сканирование: позволяет запускать автоматическое сканирование приложения и выбора традиционного и / или ajax-паука.
  • Обзор вручную: позволяет запускать браузеры, настроенные на прокси через ZAP, с возможностью включения HUD.
  • Узнать больше: содержит ссылки на локальные и удаленные ресурсы, которые вы можете использовать, чтобы узнать больше о ZAP.
На вкладке также есть панель новостей, которая предоставляет сводку новостей ZAP вместе со ссылкой для получения более подробной информации. Новости можно скрыть, когда они больше не нужны.

Сканирование правил продвижения

Были продвинуты следующие правила сканирования:

Правила активного сканирования - Выпуск

  • Содействие раскрытию исходного кода WEB-INF (выпуск 4448).

Правила пассивного сканирования - Выпуск

  • Продвигайте сканер несоответствия кодировки к выпуску (ошибка 4460).
  • Продвигайте ViewState Scanner к выпуску (ошибка 4453).
  • Продвигайте к выпуску небезопасный сканер ViewState JSF (ошибка 4455).
  • Продвигайте сканер небезопасной аутентификации к выпуску (ошибка 4456).
  • Продвигайте к выпуску сканер ошибок отладки для раскрытия информации (ошибка 4457).
  • Продвигайте сканер контрмер CSRF к выпуску (ошибка 4458).
  • Продвигайте к выпуску сканер файлов cookie с произвольным охватом (ошибка 4459).

Поддержка безголового браузера

Headless-браузеры теперь поддерживаются надстройкой Selenium и надстройками, которые ее используют. И Ajax Spider, и правило сканирования DOM XSS теперь по умолчанию работают в Firefox без головы.

Изменения в командной строке

-dir <dir>

Для предотвращения (непреднамеренного) использования надстроек / переопределения основных файлов ZAP больше не будет запускаться (и показывать ошибку), если домашний и каталоги установки такие же.

Обработка сертификатов ЦС

Следующие параметры позволяют установить и прочитать сертификат корневого CA: 
-certload <path>      Загружает сертификат корневого CA из указанного имени файла. 
-certpubdump <path>   Сбрасывает публичный сертификат корневого центра сертификации в указанное имя файла, это подходит для импорта в браузеры. 
-certfulldump <path>  Сбрасывает полный сертификат корневого ЦС (включая закрытый ключ) в указанное имя файла, это подходит для импорта в ZAP
Эти параметры особенно полезны при автономном запуске ZAP в Docker, поскольку они позволяют либо указать сертификат CA для использования, либо получить доступ к тому, который ZAP создает автоматически для вас.

Обработка клиентских сертификатов

Теперь можно установить сертификат клиента из командной строки с помощью следующих переключателей: 
-config certificate.use=true
-config certificate.pkcs12.path=/path/to/file.p12
-config certificate.pkcs12.password=WhateverThePasswordIs
-config certificate.pkcs12.index=1
-config certificate.persist=true

-config certificate.pkcs12.index необходим, только если файл содержит несколько сертификатов и вы хотите использовать еще один, кроме первого. Значения индекса начинаются с нуля (0), который является первым сертификатом. в файле.

-config certificate.persist = true необходим только в том случае, если вы хотите, чтобы настройки сохранялись в обычном ZAP файл конфигурации (чтобы они применялись для последующего использования ZAP).

Реструктуризация исходного кода

Все репозитории ZAP были переведены на Gradle из Ant. Стандартное форматирование исходного кода теперь также применяется для единообразия.

Удаление фильтров

Не рекомендуется, начиная с ZAP 2.4.0, функциональность фильтров, которая позволяла изменять / получать доступ к некоторым HTTP-сообщениям, отправленным / полученным через ZAP, теперь удален, то же самое и многое другое можно достичь с помощью скриптов и надстройки Replacer.

Изменения в связанных библиотеках

Следующие библиотеки больше не входят в комплект ZAP (core):
  • BrowserLauncher2 был заменен использованием класса JRE java.awt.Desktop . Надстройки должны использовать этот класс или связывать библиотеку.
  • HttpComponents Client / Core, больше не используется ядром, любые надстройки, которые требуют их, должны объединять их.
  • Среда выполнения JavaFX, теперь использующая JRE (например, OracleJDK, OpenJDK + OpenJFX).

Изменение входных векторов активного сканирования

Теперь у пользователей есть возможность добавлять или не добавлять параметр запроса к запросам GET, для которых их не было. В предыдущих версиях это поведение не контролировалось пользователем и было включено по умолчанию. С опцией нового пользователя по умолчанию был изменен на выкл.

JSON Authentication Method

Добавлен метод аутентификации, который позволяет отправлять объект JSON в теле запроса.

JSON Report Change

Для согласованности свойство site всегда будет массивом независимо от количества сайтов, содержащихся в отчете, раньше это был бы объект, если только один сайт, и массив, если бы более одного.

Критические изменения ZAP API:

ПРОСМОТРЕТЬ acsrf / optionTokensNames

Это изменение нарушит работу потребителей, которые вручную разбирали / извлекали имена из строки. Структура данных возвращено было изменено, чтобы правильно разделить каждое имя: 

{"optionTokensNames":["anticsrf","CSRFToken"]}
и: 

<optionTokensNames type="list">
    <tokenName>anticsrf</tokenName>
    <tokenName>CSRFToken</tokenName>
</optionTokensNames>
вместо того: 

{"TokensNames":"[anticsrf, CSRFToken]"}
и: 

<optionTokensNames>[anticsrf, CSRFToken]</optionTokensNames>

ПРОСМОТР аутентификации / установка метода аутентификации

Это изменение нарушит работу потребителей, которые вручную анализировали / извлекали ответ JSON (и ответ XML, для ручного Метод проверки подлинности). Структура данных была изменена, чтобы объект обертывал данные метода аутентификации, чтобы быть согласуется со всеми другими взглядами. Возвращаемые данные могут быть, например: 

{
  "method": {
    "port": "443",
    "host": "example.com",
    "methodName": "httpAuthentication",
    "realm": "example"
  }
}
вместо того: 

{
  "port": "443",
  "host": "example.com",
  "methodName": "httpAuthentication",
  "realm": "example"
}

ПРОСМОТР ядра / предупреждений

Это изменение нарушит работу потребителей, которые вручную анализировали / извлекали ответ JSON. Структура данных была изменено, чтобы объект заключал в себе сводку предупреждений, чтобы она согласовывалась со всеми другими представлениями. Возвращенные данные будут для пример: 

{
  "alertsSummary": {
    "High": 0,
    "Low": 3,
    "Medium": 1,
    "Informational": 1
  }
}
вместо того: 

{
  "High": 0,
  "Low": 3,
  "Medium": 1,
  "Informational": 1
}

ПРОСМОТР ядра / предупреждений

Теперь представление будет проверять идентификатор риска, возвращая ошибку (ILLEGAL_PARAMETER), если он недействителен.

ПОСМОТРЕТЬ core / numberOfAlerts

Теперь представление будет проверять идентификатор риска, возвращая ошибку (ILLEGAL_PARAMETER), если он недействителен.

Устаревшие конечные точки ZAP API:

Все основные конечные точки, связанные с предупреждениями, устарели, теперь они доступны через новый компонент предупреждение .

Измененные конечные точки ZAP API:

ACTION аутентификация / setAuthenticationMethod

Действие больше не будет удалять существующих пользователей, вместо этого пользователь отключается, а его учетные данные сбрасываются, если тип учетные данные устанавливаемого метода аутентификации различны.

ACTION context / excludeContextTechnologies

Действие теперь будет принимать строку с технологиями, даже если перед или после имен есть пробелы, например: 

os.linux, db.mysql
теперь будет действительным.

ACTION context / includeContextTechnologies

Действие теперь будет принимать строку с технологиями, даже если перед или после имен есть пробелы, например: 

os.linux, db.mysql
теперь будет действительным.

ACTION core / snapshotSession

Добавлены необязательные параметры name и overwrite , позволяющие указать имя и перезаписать существующие файлы.

Новые конечные точки ZAP API:

Просмотр сканирования / опции Добавить параметр запроса

Сообщает, должен ли активный сканер добавлять параметр запроса в запрос GET, у которого нет параметров для начала.

ПРОСМОТР автообновления / localAddons

Возвращает список со всеми локальными надстройками, установленными или нет.

VIEW context / urls

Перечисляет URL-адреса, к которым осуществляется доступ через ZAP, которые принадлежат контексту с заданным именем.

VIEW httpSessions / defaultSessionTokens

Получает токены сеанса по умолчанию.

ПРОСМОТРЕТЬ pscan / maxAlertsPerRule

Получает максимальное количество предупреждений, которое должно вызывать правило пассивного сканирования.

ПРОСМОТР скрипта / globalVar

Получает значение глобальной переменной с заданным ключом. Возвращает ошибку API (DOES_NOT_EXIST), если ранее не было задано значение.

ПРОСМОТР скрипта / globalVars

Получает все глобальные переменные (пары ключ / значение).

ПРОСМОТР скрипта / listTypes

Перечисляет доступные типы скриптов.

ПРОСМОТР скрипта / scriptVar

Получает значение переменной с заданным ключом для заданного скрипта. Возвращает ошибку API (DOES_NOT_EXIST), если скрипт с данным именем не существует или если ранее не было задано значение.

ПРОСМОТР script / scriptVars

Получает все переменные (пары ключ / значение) данного скрипта. Возвращает ошибку API (DOES_NOT_EXIST), если скрипт с данным именем не существует.

ДЕЙСТВИЕ ascan / setOptionAddQueryParam

Устанавливает, должен ли активный сканер добавлять параметр запроса к запросам GET, у которых нет параметров для начала.

Ядро ACTION / enablePKCS12ClientCertificate

Позволяет использовать сертификат клиента PKCS12 для сертификата с указанным путем файловой системы, паролем и необязательным индексом.

ДЕЙСТВИЕ core / disableClientCertificate

Отключает возможность использования клиентских сертификатов.

ДЕЙСТВИЕ httpSessions / addDefaultSessionToken

Добавляет токен сеанса по умолчанию с заданным именем и включенным состоянием.

ДЕЙСТВИЕ httpSessions / removeDefaultSessionToken

Удаляет токен сеанса по умолчанию с заданным именем.

ДЕЙСТВИЕ httpSessions / setDefaultSessionTokenEnabled

Устанавливает, включен ли токен сеанса по умолчанию с заданным именем.

ДЕЙСТВИЕ pscan / setMaxAlertsPerRule

Устанавливает максимальное количество предупреждений, которое должно выдавать правило пассивного сканирования.

Сценарий ДЕЙСТВИЯ / clearGlobalVar

Очищает глобальную переменную с заданным ключом.

Скрипт ДЕЙСТВИЯ / clearGlobalVars

Очищает глобальные переменные.

Сценарий ДЕЙСТВИЯ / clearScriptVar

Очищает переменную с заданным ключом данного скрипта. Возвращает ошибку API (DOES_NOT_EXIST), если скрипт с данным именем не существует.

Сценарий ДЕЙСТВИЯ / clearScriptVars

Очищает переменные данного скрипта. Возвращает ошибку API (DOES_NOT_EXIST), если скрипт с данным именем не существует.

Сценарий ДЕЙСТВИЯ / setGlobalVar

Устанавливает значение глобальной переменной с заданным ключом.

Сценарий ДЕЙСТВИЯ / setScriptVar

Устанавливает значение переменной с данным ключом данного скрипта. Возвращает ошибку API (DOES_NOT_EXIST), если скрипт с данным именем не существует.
