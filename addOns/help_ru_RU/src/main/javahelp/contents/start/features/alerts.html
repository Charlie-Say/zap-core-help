Оповещения (Alerts)

Предупреждение - это потенциальная уязвимость, связанная с конкретным запросом. <0 /> В запросе может быть несколько предупреждений.

Предупреждения отображаются в пользовательском интерфейсе с флажком, указывающим на риск:
      Высокий
      Средний
      Низкий
      Информационный
      Ложно-положительное

Оповещения могут генерироваться различными компонентами ZAP, включая, помимо прочего: активное сканирование , пассивное сканирование , скрипты, надстройки (расширения) или вручную с помощью диалогового окна добавления предупреждения (что также позволяет обновлять или изменять детали / информацию предупреждений).

Предупреждения помечаются на вкладке «История» флажком, который указывает предупреждение о самом высоком риске. <1 /> Все предупреждения перечислены на вкладке "Предупреждения" . а в нижнем колонтитуле показано общее количество предупреждений по риску. .

Предупреждение отменяет

Оповещения, создаваемые ZAP, включают как общую, так и конкретную информацию о созданных оповещениях. Конкретная информация напрямую связана с обнаруженной потенциальной проблемой, например URL-адрес и затронутый параметр. Общие информация включает в себя такие вещи, как описание и ссылки на соответствующие интернет-ресурсы.

Можно заменить или добавить к общей информации с помощью файла конфигурации "переопределение оповещений". Это позволяет включать информацию, относящуюся к Вашей компании, такую как утвержденные политики, внутренние ссылки или советы по конкретным технологиям, которые вы используете.

Файл конфигурации переопределения предупреждений-это файл свойств UTF-8, содержащий только информацию, которую вы хотите изменить. Строки, начинающиеся с '#', считаются комментариями и игнорируются.

Формат: 

<alertid>.<property> = [ + | - ] <your information>
Поддерживаются следующие свойства:
  • название
  • описание
  • решение
  • другая информация
  • reference
Формат тегов предупреждений: 
<alertid>.tag.<key> = [ + | - ] [ <value> ]
Обратите внимание, что:
  • Пробелы в ключе тега должны быть экранированы обратной косой чертой.
  • Значение тега необязательно.
Например 
# 40012 = Отраженный XSS
40012.solution = Следуйте рекомендациям нашей компании по адресу http://internet.example.com/xss.html
40012.tag.Bug \ Tracker \ URL = https://example.com/issues/1234
Если значение начинается со знака «+», оно добавляется к существующей информации.
Если он начинается с «-», то он добавляется к существующей информации.
Если он не начинается с «+» или «-», он заменяет существующую информацию.

Файл конфигурации переопределения предупреждений можно указать через API, Экран предупреждений о параметрах или используя командная строка опция 

-config alert.overridesFilename=<filename>
