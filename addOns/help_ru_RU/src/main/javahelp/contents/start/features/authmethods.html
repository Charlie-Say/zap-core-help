Методы аутентификации

ZAP обрабатывает несколько типов аутентификации (называемых аутентификацией). Методы ), которые можно использовать для веб-сайтов / веб-приложений. Каждый Контекст имеет метод аутентификации определено, что определяет способ обработки аутентификации. В аутентификация используется для создания веб-сеансов, соответствующих аутентифицированное веб-приложение Пользователи .

Методы аутентификации можно использовать в разных местах. ZAP. Некоторые из примеров включают:

  • определение пользователей и автоматический вход в систему
  • обнаружение аутентифицированных / неаутентифицированных состояний
  • выполнение автоматической повторной аутентификации

Было реализовано несколько методов аутентификации, и система поддерживает легкое добавление новых методов в соответствии с потребностями пользователя. Основные из них описаны ниже.

Ручная аутентификация

Этот метод позволяет пользователям выполнять аутентификацию вручную. (например, аутентифицироваться в браузере во время прокси-сервера через ZAP) и затем выберите соответствующий сеанс HTTP. В качестве фактического аутентификация выполняется вами, этот метод не поддерживает повторная аутентификация в случае выхода пользователя из системы. Однако, если вы все же определите стратегию проверки аутентификации тогда ZAP будет поддерживать статистику аутентификации.

При использовании этого метода аутентификации настройка пользователя для контекста требуют выбора аутентифицированного сеанса HTTP.

Аутентификация на основе форм

Этот метод используется для веб-сайтов / веб-приложений, где аутентификация выполняется путем отправки формы или выполнения запроса GET на «URL-адрес входа» с использованием пары учетных данных для аутентификации «имя пользователя / пароль». Возможна повторная аутентификация. Конфигурация может быть выполнена с помощью сеанса . Экран проверки подлинности контекстов или с помощью контекстного PopupMenu: Пометить как ... Запрос на вход с аутентификацией на основе формы .

При использовании этого метода аутентификации для настройки пользователя в контексте требуется настройка пары учетных данных имя пользователя / пароль , которые используются для формы на основе аутентификация.

Если приложение требует отправки токена анти-CSRF, представленного на странице входа, ZAP будет обрабатывать это автоматически. Убедитесь, что токен имя настраивается на экране "Параметры Anti CSRF" .

Аутентификация на основе JSON

Этот метод используется для веб-сайтов / веб-приложений, где аутентификация выполняется путем отправки объекта JSON на «URL-адрес входа» с использованием «имени пользователя / пароля» пара учетных данных для аутентификации. Возможна повторная аутентификация. Конфигурация может быть выполнена с помощью сеанса . Экран проверки подлинности контекстов или с помощью контекстного PopupMenu: Пометить как ... Запрос на вход с аутентификацией на основе JSON .

При использовании этого метода аутентификации для настройки пользователя в контексте требуется настройка пары учетных данных имя пользователя / пароль , которые используются для аутентификация.

HTTP / NTLM аутентификация

Этот метод используется для веб-сайтов / веб-приложений, где аутентификация принудительно с использованием механизмов аутентификации HTTP или NTLM с использованием заголовков сообщений HTTP. Поддерживаются три схемы аутентификации: базовая, дайджест и NTLM. Возможна повторная аутентификация, поскольку заголовки аутентификации отправляются с каждым аутентифицированным запрос. Конфигурация может быть выполнена с помощью сеанса . Экран аутентификации контекстов .

При использовании этого метода аутентификации для настройки пользователя в контексте требуется настройка пары учетных данных имя пользователя / пароль , которые используются для аутентификации HTTP / NTLM.

Аутентификация на основе сценария

Этот метод полезен для веб-сайтов / веб-приложений, где аутентификация является более сложной и имеет некоторые индивидуальные сценарии, которые обрабатывают процесс аутентификации, полезны. Чтобы использовать этот метод, вы должны сначала определить аутентификацию скрипт , который отправляет сообщения или выполняет другие действия, которые необходимы вашему веб-приложению. Затем этот сценарий выбирается для использования в данном контексте. и он вызывается всякий раз, когда выполняется аутентификация. Возможна повторная аутентификация. Конфигурацию можно выполнить с помощью экрана аутентификации контекстов сеанса .

При использовании этого метода аутентификации для настройки пользователя в контексте требуется настройка набора параметров, определенных в скрипте. Дополнительные сведения см. В предоставленных примерах сценария аутентификации.
