Структурные параметры

Структурные параметры - это разновидность Структурного модификатора которые определяют параметры, которые представляют структуру приложения, а не данные пользователя.

В «традиционных» веб-приложениях структура приложения обычно определяется путями URL. и данные содержатся в параметрах URL и данных POST.
URL-адреса вроде:

  • https://www.example.com/app/aaa?ddd=eee
  • https://www.example.com/app/aaa?ddd=fff
  • https://www.example.com/app/aaa?ddd=ggg
представлены в Дерево сайтов как один «узел» в дереве:
  • Сайты
    • https://www.example.com
      • приложение
        • GET:aaa(ddd)
Дерево сайтов очень важно, поскольку оно отражает понимание ZAP структуры приложения.
Если это не очень хорошее представление структуры, ZAP не сможет эффективно атаковать приложение.

В одностраничных приложениях параметр используется для обозначения логической «страницы»:
  • https://www.example.com/app/aaa?page=p1&ddd=eee
  • https://www.example.com/app/aaa?page=p2&ddd=fff
  • https://www.example.com/app/aaa?page=p3&ddd=ggg
эти 3 URL-адреса представляют разные логические страницы, но по умолчанию ZAP по-прежнему будет представлять их как один узел:
  • Сайты
    • https://www.example.com
      • Приложение
        • GET:aaa(ddd,page)
Это проблема, потому что теперь ZAP не атакует все функции приложения.

В терминах ZAP параметр URL «страницы» является «структурным параметром» - параметром, который определяет часть структуры приложения.
Вы можете определить структурные параметры, добавив приложение в Контекст а затем настраивая их через Экран структуры контекста сеанса .
Как только вы это сделаете, страницы будут правильно представлены как 3 узла:
  • Сайты
    • https://www.example.com
      • Приложение
        • aaa
          • GET:p1(ddd,page)
          • GET:p2(ddd,page)
          • GET:p3(ddd,page)
