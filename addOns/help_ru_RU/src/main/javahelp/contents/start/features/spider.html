Паук

Паук - это инструмент, который используется для автоматического обнаружения новых ресурсы (URL-адреса) на определенном сайте. Он начинается со списка URL-адресов посетить, называется семенами, что зависит от того, как паук началось. Затем Паук посещает эти URL-адреса, он идентифицирует все гиперссылки на странице и добавляет их в список URL-адресов для посещения и процесс продолжается рекурсивно до тех пор, пока будут найдены новые ресурсы.

Паук можно настроить и запустить с помощью диалогового окна «Паук» . .

Во время обработки URL-адреса Spider запрашивает извлекает ресурс, а затем анализирует ответ, определяя гиперссылки. В настоящее время он имеет следующее поведение при обработке типы ответов:

HTML

Обрабатывает определенные теги, определяя ссылки на новые ресурсы:
  • Base - Proper handling
  • A, Link, Area - 'href' attribute
  • Frame, IFrame, Script, Img - 'src' attribute
  • Meta - 'http-equiv' for 'location' and 'refresh'
  • Форма - правильная обработка форм с помощью GET и POST метод. Значения полей генерируются корректно, включая HTML 5.0 типы ввода .
  • Комментарии - действительные теги, найденные в комментариях, также анализируются, если указанные в параметрах Экран паука

Robots.txt file

Если установлено в Комплектация Паук экран , он также анализирует файл Robots.txt и пытается идентифицировать новые ресурсы по указанным правилам. Следует отметить, что Паук не следует правилам, указанным в 'Robots.txt' файл.

Формат OData Atom

Контент OData, использующий формат Atom, в настоящее время поддерживается. Все включенные ссылки (относительные или абсолютные) обрабатываются.

Текстовый ответ не в формате HTML

Текстовые ответы анализируются путем сканирования шаблона URL

Нетекстовый ответ

В настоящее время Паук не обрабатывает этот тип ресурсов.

Прочие аспекты

  • При проверке того, был ли URL уже посещен, поведение относительно того, как обрабатываемые параметры можно настроить на экране параметров Spider.
  • При проверке того, был ли URL уже посещен, есть несколько общих параметров. которые игнорируются: jsessionid, phpsessid, aspsessionid, utm_ *
  • Поведение паука в отношении файлов cookie зависит от того, как он запускается и какие параметры включены. Для большего подробности см. на экране параметров Spider.

Паук настраивается с помощью экрана параметров паука .
