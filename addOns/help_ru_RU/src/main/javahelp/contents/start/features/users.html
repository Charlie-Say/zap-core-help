Пользователи

Пользователи - это ZAP-представления пользователей веб-сайтов / веб-приложений. Они позволяют выполнять определенные действия с точки зрения пользователь веб-приложений. Для каждого Context , может быть определен набор пользователей, которые затем могут быть использованы в действиях связанные с контекстом. Чаще всего при различных сканированиях сообщения-запросы могут быть отправлены с точки зрения Пользователя.

Концепция пользователей тесно связана с концепциями управления сеансом и аутентификации . Когда пользователь первый используется где-то в ZAP, выполняется аутентификация (согласно метод аутентификации, определенный для контекста), а сеанс создан и настроен для этого пользователя (согласно Сессии Управление определено для контекста). После этого запросы, отправленные из точка зрения пользователя изменяются (при необходимости) и отправляются в таким образом, чтобы веб-сервер идентифицировал их как отправленные аутентифицированный пользователь веб-приложения / веб-сайта. Если в любое время Пользователь появится быть неаутентифицированным (как определено Стратегия проверки подлинности ), новый выполняется аутентификация, и Сессия обновляется соответствующим образом.

Для аутентификации пользователя на сайте / в веб-приложении метод аутентификации определяет, как аутентификация выполнено (процесс), а необходимые учетные данные (точные идентификаторы) зависят от пользователя, поэтому в ZAP они настроен в Users.
