Управление сессией

ZAP обрабатывает несколько типов управления сеансом (так называемый сеанс Методы управления ), которые можно использовать для веб-сайтов / веб-приложений. Каждый Context имеет управление сеансом Определен метод, который определяет, как хранятся сеансы.

Управление сеансом на основе файлов cookie

В случае этого метода сеанс отслеживается через печенье. В настоящее время используемые токены сеанса импортируются из расширение HTTP-сессий .

Управление сеансом HTTP-аутентификации

В случае этого метода сеанс управляется заголовком HTTP-запроса Авторизация .

Управление сеансом на основе сценария

Этот метод полезен для веб-сайтов / веб-приложений, где управление сеансом является более сложным и некоторые индивидуальные сценарии, управляющие процессом, полезны. Чтобы использовать этот метод, вы должны сначала определить сценарий управления сеансом. который анализирует сообщения или выполняет другие действия, необходимые вашему веб-приложению. Затем этот сценарий выбирается для использования в данном контексте. и он вызывается всякий раз, когда выполняются действия по управлению сеансом. Конфигурация может быть выполнена с помощью сеанса . Диалог контекстов и требует, чтобы вы установили надстройку ZAP Scripts Console из Marketplace.
