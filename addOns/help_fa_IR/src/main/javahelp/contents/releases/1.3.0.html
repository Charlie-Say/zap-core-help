تغییرات قابل توجه:

ریش ریش شده

اکنون می توانید رشته های پاسخی را برای یافتن آسیب پذیری های فازی مورد استفاده قرار دهید.
علامت های ضد CSRF می تواند شناسایی و به طور خودکار بازسازی زمانی که ریش ریش شده.
این قابلیت بر اساس کد پروژه OWASP JBroFuzz است.

گواهینامه SSL پویا

پشتیبانی از اتصالات SSL بهبود یافته و ساده شده است.
کاربران می توانند گواهینامه ریشه خود را ایجاد کنند و این را به مشتریان HTTP خود توزیع کنند.
دیدن گزینه: گواهی های SSL پویا برای جزئیات بیشتر.

حالت اهریمن و API

شروع ZAP با گزینه "-اهریمن" خط فرمان باعث می شود که در پسزمینه در حالت "بدون سر و صدا" اجرا شود به این معنی است که هیچ رابط کاربری نمایش داده نمی شود.
یک API اولیه در XML، JSON و HTML اجرا شده است.
اگر ZAP به عنوان یک اهریمن اجرا شود، API به طور خودکار فعال می شود، در غیر این صورت API باید از طریق صفحه گزینه های API فعال شود.
API را می توان با باز کردن http: //zap در مرورگر خود هنگام پروکسی از طریق ZAP هدایت کرد.

یکپارچه سازی محافظ هسته

این محافظ هسته پوسته جاوا تعاملی است که می تواند برای اجرای اسکریپت محفظ هسته استفاده شود.
این اسکریپت ها یک شکل ساده از جاوا هستند که بسیاری از عناصر ، اما در فرمت اسکریپت ساده تر جاوا استفاده میشود.
تمام کد جاوا نیز کد محافظ هسته معتبر است.
ادغام محافظ هسته در OWASP ZAP شما را قادر به نوشتن اسکریپت ها با استفاده از توابع و مجموعه داده های ZAP می کند.
این می تواند یک ویژگی بسیار قدرتمند برای تجزیه و تحلیل برنامه های وب باشد.

کاملا بین المللی

تمام رشته های نمایش در حال حاضر به طور کامل بین المللی است.

تغییرات جزئی:

مشاهده سحر و جادو

این درخواست, پاسخ زبانه ها هم اکنون حاضر به ارائه میباشد گزینه "مشاهده کد هگزا" که درخواست و پاسخ را در قالب هگزا نمایش می دهد.

نتایج جستجو

این برگه ی جستجو در حال حاضر تمام نمونه های یک رشته در بر داشت و نه تنها نمونه اول در هر درخواست یا پاسخ.

جدا سازی آدرس های اینترنتی

آدرس ها را می توان از صراحت اسکنر فعال و عنکبوتی از پروکسی جدا کرد.

پشتیبانی کپی

اکثر پانلها اکنون یک گزینه منو «کپی» راست کلیک، از جمله پانل اسکن و پورت نیروی مخلوق را فراهم می کنند.

بازگرداندن/ پشتیبانی مجدد

تمام زمینه های ورودی در حال حاضر اقدامات باطل کردن / دوباره انجام دادن را با استفاده از سیستم عامل های پیش فرض پشتیبانی می کند واگر / شتاب دهنده های مجدد:
  • ویندوز/لینوکس: Ctrl+Z / Ctrl+Y
  • مکینتاژ: Cmd+Z / Cmd+Shift+Z

پورت اسکنر پشتیبانی پروکسی و گزینه زمانبندی

پورت اسکنر هم اکنون می تواند از پروکسی خروجی استفاده کند (اگر پیکربندی شده است) و همچنین می توان زمان بندی را در میلی ثانیه نیز تنظیم کرد.

دکمه های قفل درخواست و پاسخ

اکنون 2 دکمه نقض کردن وجود دارد که اجازه می دهد بر روی تمام درخواست ها و تمام پاسخ ها به صورت جداگانه تنظیم شود.

گسترش سایت ها و دکمه های تب اطلاعات

در حال حاضر 2 دکمه وجود دارد که به شما امکان می دهد که بین صفحات وب و اطلاعات زبانه ها گسترش یابد.

Break tab icon changes colour when breakpoint hit

در حالی که زبانه نقض کردن در حال استفاده نیست، نماد آن صلیب خاکستری است: & nbsp؛;  .
وقتی یک breakpoint آیکون ضربه زده شده تغییر کرده است به یک صلیب قرمز: & nbsp؛;  .

زمان وقوع قابل تنظیم

این گزینه: اتصال صفحه نمایش به شما اجازه می دهد تا زمان بندی را در ثانیه تنظیم کنید برای تست برنامه های آهسته تست کنید.

مسائل شناخته شده:

مکینتاش: لایه سوکت های امن پویا و گوگل کروم

در حال حاضر لایه سوکتهای امن دینامیک هنگام استفاده از گوگل کروم کار نمی کند. این به خاطر مسئله معروف حل نشده با گوگل کروم و کلید حلقه مکینتاش است. هنگام وارد کردن CA Root CA OWASP ZAP به کلید حلقه و درخواست یک وب سایت لایه سوکتهای امن، یک پیام خطا "سند نامعتبر" نمایش داده می شود.
