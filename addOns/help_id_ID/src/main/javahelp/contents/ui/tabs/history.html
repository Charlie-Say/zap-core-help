Riwayat Tab

Tab Sejarah menampilkan daftar semua permintaan dalam urutan yang mereka dibuat.
Setiap permintaan Anda dapat Melihatnya:
     Indeks permintaan - setiap permintaan yang diberi nomor, dimulai dari 1
     Metode HTML, misalnya GET atau POST
     URL yang diminta
     Kode tanggapan HTTP
     Ringkasan singkat tentang apa arti dari kode tanggapan HTTP
     Lamanya seluruh waktu permintaan mengambil
     Any Alerts on the request
     Any Notes you have added to request
     Any Tags on the request
Memilih sebuah permintaan yang akan menampilkannya di Tab permintaan dan Tab tanggapan atas.

Alat penyaring bar

Alat penyaring bar yang disediakan memungkinkan anda membatasi permintaan yang ditampilkan.
Mengklik pada Tombol penyaring yang ditampilkan Sejarah dialog filter.
Ringkasan filter saat ini yang diterapkan untuk ditampilkan di sebelah kanan tombol.

Menu klik kanan

Mengklik kanan pada sebuah node akan memunculkan menu yang memungkinkan anda untuk:

Serang

Menu Attack memiliki beberapa submenu berikut:

Aktif Scan...

Ini akan meluncurkan Scan Aktif dialog yang memungkinkan anda untuk memulai sebuah aktif scan dengan titik awal sesuai dengan permintaan yang anda pilih.

Laba-laba...

Ini akan meluncurkan Laba-laba dialog yang memungkinkan anda untuk memulai laba laba dengan titik awal sesuai dengan permintaan yang anda pilih.

Sertakan dalam Konteks

Dengan menu ini anda dapat memasukkan node yang dipilih dan semua bawahannya di tempat yang ditentukan konteks.
Anda juga memiliki pilihan untuk membuat sebuah konteks yang baru.
Itu Konteks Sesi dialog akan ditampilkan untuk memungkinkan anda untuk membuat perubahan tambahan.

Kecualikan dari Konteks

Dengan menu ini anda dapat mengecualikan node yang dipilih dan semua bawahan mereka dari yang ditentukan konteks.
Itu Konteks Sesi dialog akan ditampilkan untuk memungkinkan anda untuk membuat perubahan tambahan.

Tandai sebagai konteks

Menu ini memiliki beberapa submenu untuk masing-masing konteks anda telah mendefinisikan:

Nama konteks Permintaan Login Auth Form berbasis

Ini mengidentifikasi simpul yang ditentukan sebagai permintaan masuk untuk konteks yang ditentukan.
Anda mungkin hanya memiliki satu simpul yang diidentifikasi seperti itu dalam satu konteks.
Itu Otentikasi Konteks Sesi layar akan ditampilkan untuk memungkinkan anda untuk melakukan perubahan tambahan.

Context name JSON-based Auth Login request

This identifies the specified node as a login request for the specified context.
You may only have one node identified as such in any one context.
Itu Session Context Authentication layar akan ditampilkan untuk memungkinkan anda untuk membuat setiap perubahan tambahan.

Context name Data driven node

This identifies the specified node as Data driven content for the specified context.
The Session Context Structure screen will be displayed to allow you to make any additional changes.

Keluarkan dari

This menu has the following submenus:

Proxy

This will exclude the selected nodes from the proxy. They will still be proxied via ZAP but will not be shown in any of the tabs.
This can be used to ignore URLs that you know are not relevant to the system you are currently testing.
The nodes can be included again via the Properti Sesi dialog

Scanner

This will prevent the selected nodes from being actively scanned.
The nodes can be included again via the Properti Sesi dialog

Spider

This will prevent the selected nodes from being spidered.
The nodes can be included again via the Session Properties dialog

Manage History Tags...

This will bring up the Manage History Tags dialog which allows you to change the tags associated with the request.

Catatan...

This will bring up the Add Note dialog which allows you to record notes related to the request.

Hapus

This will remove the node and all of its children from ZAP.
However they can be added back in, to prevent this use the 'Exclude from' menus.

Istirahat...

This will bring up the Add Breakpoint dialog which allows you to set a breakpoint on that URL.

Lansiran untuk simpul ini

If the URL selected has alerts associated with it then they will be listed under this menu.
Selecting one of the alerts will cause it to be displayed.

Open/Resend with Request Editor...

This will bring up the Manual Request Editor dialog which allows you to resend the request after making any changes to it that you want to.

Peringatan Baru...

This will bring up the Add Alert dialog which allows you to manually record a new alert against this request.

Tampilkan di tab situs

This will show the selected message in the Sites tab.

Buka URL di Browser

This will open the URL of the selected node in your default browser.

Buat formulir tes anti CSRF

This will open a URL which will give you a generated form for testing for CSRF issues.
It will only be enabled for POST requests, if the API is enabled and if Java supports the opening of URLs in a browser on your platform.
