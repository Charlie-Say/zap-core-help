Layar Pilihan Proxy Lokal

This screen allows you to configure the addresses and ports on which ZAP accepts incoming connections.

Local Proxy

By default ZAP will listen on one local address and port, and these are the address and port that you must configure your browser to use as a proxy.

Alamat

Alamat lokal yang akan digunakan ZAP. Semua alamat yang tersedia diidentifikasi secara otomatis.

Pelabuhan

Pelabuhan yang ZAP akan mendengarkan.

Dibelakang NAT

Menandakan bahwa Proxy Lokal (ZAP) berada di belakang NAT. Ketika ZAP yang dipilih akan mencoba untuk menentukan alamat IP publik, untuk mendeteksi secara tepat dan menangani permintaan dengan alamat IP publik (sebagai contoh, untuk dilayani oleh API ZAP).

Catatan: Pilihan ini hanya didukung ketika ZAP berjalan di sebuah Contoh AWS EC2. ZAP akan mendapatkan alamat IP publik dari AWS EC2 instance's metadata.
ZAP harus dimulai dengan opsi ini diaktifkan jika akses ke API, melalui alamat IP publik, diperlukan:

zap.sh -daemon -port 8080 -host 0.0.0.0 -config proxy.behindnat=true
Juga, API harus dikonfigurasikan untuk menerima alamat IP eksternal (yaitu alamat IP dimana ZAP diakses).

Hapus Pengkodean yang Tidak ada dukungan

Memungkinkan proxy untuk menghapus yang tidak didukung pengkodean dari "Accept-Encoding" bidang header permintaan, sehingga tidak ada (tidak didukung) pengkodean dan transformasi yang dilakukan untuk respon.
Opsi ini harus selalu diaktifkan kecuali ketika pengujian perubahan pegkodean.
Pesan yang dikodekan dengan pengkodean yang tidak didukung tidak akan dipindai secara tepat (baik itu dengan pemindai pasif dan aktif).

Protokol keamanan

Memungkinkan untuk memilih versi SSL/TLS yang diaktifkan untuk koneksi yang masuk (contonya dari peramban). Setidaknya salah satu versi mesti diaktifkan, versi yang tidak didukung oleh JRE tidak akan dipilih serta dinonaktifkan.
Pilihan SSLv2Hello harus dipilih secara bersamaan setidaknya dengan salah satu versi SSL/TLS.
Opsi keamanan berlaku untuk keduanya, proxy utama dan semua tambahan.

Proxy Tambahan

Anda bisa menambahkan sebanyak mungkin alamat dan port tambahan bagi ZAP untuk tunduk seperti keinginan anda.
Ini bisa berguna ketika menguji aplikasi mobile - anda bisa memasangkan aplikasi mobile melalui antarmuka Wi-Fi bersamaan dengan menggunakan situs backend secara langsung yang menggunakan sebuah proxy peramban melalui ZAP yang meniru alat pengguna ponsel.

Intercepting/Transparent Proxy

The local proxies can be used as intercepting/transparent proxies for both HTTP and HTTPS. For HTTPS the client applications (e.g. browser) need to use the TLS extension Server Name Indication. This allows you to set up a testing LANs (or VMs) where all HTTP and HTTPS traffic is proxied regardless of software settings.

For example, if you have a Linux machine you use for testing, you can do something like the following to forward all HTTP and HTTPS traffic to a local proxy listening on 192.168.0.14:8080: 

iptables -t nat -A OUTPUT -p tcp --dport 443 -j DNAT --to-destination 192.168.0.14:8080
iptables -t nat -A OUTPUT -p tcp --dport 80 -j DNAT --to-destination 192.168.0.14:8080
