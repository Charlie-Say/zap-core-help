Pilihan layar pindai aktif

Layar ini memungkinkan anda untuk mengkonfigurasi aktif scan pilihan:

Number of Hosts Scanned Concurrently

Jumlah maksimum host yang akan dipindai pada waktu yang bersamaan. Peningkatan ini dapat menambah tekanan pada komputer ZAP yang sedang berjalan.

Concurrent Scanning Threads per Host

Jumlah benang yang akan digunakan untuk pemindaian per host.
Meningkatkan jumlah benang dapat mempercepat pemindaian namun akan memberi tekanan ekstra pada komputer ZAP yang sedang berjalan dan host target.

Max Results to List

Jumlah hasil yang akan ditampilkan pada tab Scan aktif.
Menampilkan sejumlah besar hasil yang dapat meningkatkan pada waktu pemindaian secara signifikan.

Maximum Rule Duration (minutes; 0 is unlimited)

Waktu maksimum setiap aturan individu dapat berjalan dalam beberapa menit. Nol berarti tidak ada batas. Ini bisa digunakan untuk mencegah aturan yang menghabiskan banyak waktu.

Maximum Scan Duration (minutes; 0 is unlimited)

Waktu maksimum yang bisa dipindai secara keseluruhan dalam hitungan menit. Nol berarti tidak ada batas. Ini bisa digunakan untuk memastikan pemindaian selesai sekitar waktu yang ditentukan.

Delay When Scanning (In Milliseconds)

Penundaan pada mili detik antara setiap permintaan.
Menetapkan ini menjadi nilai nol tidak akan meningkatkan waktu pemindaian aktif yang berlangsung, namun akan mengurangi ketegangan pada host target.

Inject plugin ID in header for all active scan requests.

Jika pilihan ini dipilih maka pemindai aktif ini akan menyuntikkan permintaan header X-ZAP-Scan-ID dengan ID dari pemindai yang mengirimkan permintaan HTTP.

Handle anti-CSRF tokens.

Jika pilihan ini dipilih maka pemindai yang aktif akan secara otomatis mencoba untuk meminta anti CSRF token bila diperlukan.
Previously this would have forced the scanner to only use one thread, but that is no longer the case. You are strongly recommended to check that the anti CSRF tokens are being correctly generated if more than one thread is being used, for example using custom tags to check for success / failure patterns in the response.

In Attack Mode prompt to rescan nodes when scope changed.

If this option is selected then when you select Attack mode you will be prompted to choose whether to rescan nodes in scope.
If the option is not selected then the following option will control whether the nodes are rescanned.

In Attack Mode always rescan nodes when scope changed.

If this option is selected then when running in Attack mode all nodes in scope will be rescanned if the scope changes.
This is not recommended for large sites as it could take a long time.

Default Active Scan Policy

The Kebijakan Pindai that is used by default when you start an active scan.

Attack Mode Scan Policy

The Scan Policy that is used for scanning in Attack mode.

Max Progress Chart in Mins

The maximum time in minutes for which response codes will be charted in the Scan Progress dialog.
To disable the chart the option should be set to zero minutes.
