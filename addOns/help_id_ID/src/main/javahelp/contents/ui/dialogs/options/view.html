Pilihan Tampilan Layar

Tampilan layar memungkinkan anda untuk menyusun:

Gambar

Jika ZAP memproses gambar.

Tampilan permintaan MENGHUBUNGKAN (lokal)

Jika permintaan MENGHUBUNGKAN HTTP diterima oleh Proxy lokal harus bertahan dalam arus sidang dan ditampilkan dalam Tab riwayat. Kesalahan-nya adalah tidak menunjukkan permintaan tersebut, mereka biasanya terjadi (dalam persiapan untuk TLS/SSL, WebSocket... sambungan) dan, kecuali memeriksa perilaku aplikasi lanngganan, tidak banyak menarik (biasanya).

Tampilan

Tata letak 3 panel utama.
Pilihan berikut tersedia:
  • Maksimalkan tab kiri (situs) - panel 'pohon' yang berisi tab situs meluas untuk sepanjang sisi kiri. Ini akan mengurangi jumlah ruang yang tersedia untuk panel 'informasi'.
  • Maksimalkan tab bawah (Riwayat dll)- panel 'informasi' meluas untuk panjang penuh dari bawah. Ini akan mengurangi jumlah ruang yang tersedia untuk panel 'pohon.
  • Tata letak penuh - tab yang dipilih memakan layar penuh. ini berguna ketika menggunakan ZAP pada layar kecil.

Posisi panel tanggapan

Memungkinkan untuk mengkonfigurasi posisi tab tanggapan sehubungan dengan tab permintaan.
Pilihan tersebut tersedia:
  • Tab bedampingan - tab permintaan dan tanggapan yang berdampingan. Hal ini meningkatkan informasi yang dapat ditampilkan tetapi berarti anda tidak dapat melihat tentang keduanya permintaan dan tanggapan pada waktu yang sama.
  • Side by Side in Tab - The Request and Response panels are shown side by side in the same tab. This decreases the information that can be displayed but means you can see both the request and response at the same time.
  • Panels Side by Side - The Request panel is shown to the left of the Response panel. Hal ini mengurangi informasi yang dapat ditampilkan tapi berarti anda dapat melihat kedua permintaan dan tanggapan pada waktu yang sama.
  • Request Shown Above Response - The Request panel is shown above the Response panel. This decreases the information that can be displayed but means you can see both the request and response at the same time.
Pilihan posisi panel tanggapan tidak berlaku bila pilihan tampilan diatur ke tata letak penuh.

Tampilkan tombol istirahat

Lokasi tombol istirahat.

Permintaan besar melihat ukuran min

Ukuran minimum dari permintaan badan dalam byte di mana sebuah titik 'besar permintaan badan' pesan akan ditampilkan bukan dari badan yang sebenarnya.
Hal ini untuk mencegah tubuh yang sangat besar agar tidak memperlambat UI.
Menetapkan nilai untuk -1 ini akan menghasilkan permintaan yang selalu ditampilkan tidak peduli seberapa besar itu.

Tanggapan besar melihat ukuran min

Ukuran minimum tubuh tanggapan dalam satuan byte di mana pesan 'respon tubuh besar' akan ditampilkan daripada tubuh yang sebenarnya.
Hal ini untuk mencegah tubuh yang sangat besar agar tidak memperlambat UI.
Menetapkan nilai untuk -1 ini akan menghasilkan tanggapan yang selalu ditampilkan tidak peduli seberapa besar itu.

Pilihan tab hasil cap waktu

Setelah anda telah diaktifkan cap waktu pada tab output anda dapat mengkonfigurasi format yang anda ingin agar cap waktu tersebut itu muncul. Pilih format yang telah ditentukan sebelumnya dari daftar drop-down atau masukkan salah satu pilihan Anda sendiri. Format berbasis java FormatTanggalSederhana. Setelah memilih atau memasukkan format cap waktu jika Anda menekan enter contoh di sebelah kanan akan diperbarui untuk mencerminkan pilihan Anda. Jika ZAP tidak dapat menggunakan format yang telah Anda masukkan maka contoh ditampilkan berdasarkan pada format Default.
Panjang & Default tttt-BB-hh JJ:mm:dd
ISO8601 tttt-BB-hh'T'JJ:mm:ddZ
Hanya waktu JJ:mm:dd

Font Controls

The General Font section controls default text font and size used for the ZAP display, while the Work Panels Font section controls the font used for the top right section of ZAP's main window (Ex: Request/Response, Edit and Resend tool, etc.). If you set this to -1 then the system default size will be used.
Bidang 'Contoh Font' akan menunjukkan seberapa besar teks default akan muncul.
Note: This setting will only take effect when ZAP is restarted.

Look and Feel

Allows to choose the look and feel of ZAP. If empty the default is used, Nimbus in Windows and Linux, the native system look and feel in macOS.
The system property swing.defaultlaf takes precedence over this option.
