Pengguna

Pengguna adalah representasi ZAP dari pengguna situs web/aplikasi web. Mereka mengizinkan tindakan tertentu untuk dilakukan dari sudut pandang pengguna aplikasi web. Untuk setiap Konteks, satu set Pengguna dapat didefinisikan, yang kemudian dapat digunakan dalam tindakan terkait dengan konteks. Paling umum, selama berbagai pemindaian pesan permintaan dapat dikirim dari sudut pandang Pengguna.

Konsep Pengguna sangat terkait dengan konsep Manajemen Sesi dan Otentikasi. Ketika Pengguna pertama digunakan di suatu tempat di ZAP, otentikasi dilakukan (menurut Metode Otentikasi yang ditetapkan untuk Konteks) dan Sesi dibuat dan dikonfigurasi untuk pengguna ini (menurut Sesi Manajemen didefinisikan untuk Konteks). Setelah itu, permintaan yang dikirim dari sudut pandang Pengguna yang dimodifikasi (jika perlu) dan dikirim sedemikian rupa sehingga server web mengidentifikasi mereka sebagai kiriman dari pengguna aplikasi web/situs web yang terotentifikasi. If anytime a User appears to be unauthenticated (as determined by the Authentication Verification Strategy), a new authentication is performed and the Session is updated accordingly.

Untuk melakukan otentikasi pengguna pada situs web / pada aplikasi web, Metode Otentikasi menentukan bagaimana otentikasi dilakukan (proses), sedangkan kredensial yang diperlukan (pengidentifikasi yang tepat) bergantung pada pengguna, sehingga di ZAP, mereka dikonfigurasikan dalam Pengguna.
