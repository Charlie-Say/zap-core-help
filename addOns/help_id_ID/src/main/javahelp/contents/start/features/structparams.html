Parameter Struktural

Parameter struktural adalah jenis dari Struktural Pengubah yang mengidentifikasi parameter yang mewakili struktur aplikasi ketimbang data pengguna.

Dalam aplikasi web 'tradisional', struktur aplikasi didefinisikan secara khusus oleh jalur URL dan datanya ada dalam parameter URL serta data POST.
URL seperti:

  • https://www.example.com/app/aaa?ddd=eee
  • https://www.example.com/app/aaa?ddd=fff
  • https://www.example.com/app/aaa?ddd=ggg
diwakili di dalam Sites Tree sebagai satu 'simpul' di pohon:
  • Situs
    • https://www.example.com
      • aplikasi
        • GET:aaa(ddd)
Pohon Situs sangat penting sebagai cerminan pemahaman ZAP tentang struktur aplikasi.
Jika bukan sebuah representasi dari struktur yang bagus maka ZAP tidak dapat menyerang aplikasi secara efektif.

Pada halaman aplikasi 'tunggal' sebuah parameter digunakan untuk menandakan 'halaman' logis:
  • https://www.example.com/app/aaa?page=p1&ddd=eee
  • https://www.example.com/app/aaa?page=p2&ddd=fff
  • https://www.example.com/app/aaa?page=p3&ddd=ggg
3 URL ini mewakili halaman logis yang berbeda, namun secara default ZAP masih akan mewakili mereka sebagai satu node:
  • Situs
    • https://www.example.com
      • aplikasi
        • MENDAPATKAN:aaa(ddd,halaman)
Ini adalah sebuah masalah karena ZAP sekarang tidak akan menyerang semua fungsionalitas aplikasi.

Dalam istilah ZAP, 'halaman' parameter URL adalah sebuah 'parameter struktural' - sebuah parameter yang mendefinisikan bagian-bagian dari struktur aplikasi.
Anda dapat menentukan parameter struktural dengan menambahkan aplikasi ke Konteks dan kemudian mengkonfigurasi mereka melalui Sesi Konteks Struktur layar.
Setelah anda melakukan ini halaman akan benar digambarkan sebagai 3 node:
  • Situs
    • https://www.example.com
      • aplikasi
        • aaa
          • MENDAPATKAN:p1(ddd,halaman)
          • MENDAPATKAN:p2(ddd,halaman)
          • MENDAPATKAN:p3(ddd,halaman)
