Estatísticas

ZAP mantém estatísticas que podem ajudá-lo a entender o que realmente está acontecendo, ao interagir com aplicativos grandes.

As estatísticas estão disponíveis através da API e também podem ser enviadas para um servidor de Statsd quando configurado através da tela de opções de estatísticas.

Estatísticas baseadas em site

Estatísticas mantidas com base em sites incluem:
  • códigos de resposta, por exemplo:
    • stats.Code.200
    • stats.Code.200
  • tempos de resposta em ms (usar uma escala logarítmica), por exemplo:
    • stats.responseTime.1
    • stats.responseTime.2
    • stats.responseTime.4
    • stats.responseTime.8
    • stats.responseTime.16
  • tipos de conteúdo, por exemplo:
    • stats.contentType.text/css
    • stats.contentType.text/html;charset=utf-8
  • tags, por exemplo:
    • stats.tag.Password
    • stats.tag.Hidden
  • anticsrf tokens gerados:
    • stats.acsrf.anticsrf
  • Informações de autenticação:
    • stats.auth.Success (número de sucessos de autenticação)
    • stats.auth.Failure (número de falhas de autenticação)
    • stats.auth.state.assumedin (number of messages between successful polls that are assumed to be logged in)
    • stats.auth.state.loggedin (number of messages that appear to be logged in)
    • stats.auth.state.loggedout (number of messages that appear to be logged out)
    • stats.auth.state.noindicator (number of messages where no logged in or out indicators have been set)
    • stats.auth.state.unknown (number of messages which don't contain either logged in or out indicators)
