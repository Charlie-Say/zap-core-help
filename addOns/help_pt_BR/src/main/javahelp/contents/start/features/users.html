Usuários

Os usuários são as representações de ZAP para os usuários dos sites/webapps. Eles permitem que determinadas ações sejam executada a partir do ponto de vista de um usuário da aplicação. Para cada contexto, um conjunto de usuários pode ser definido, podendo ser usado em ações relacionadas ao contexto. Normalmente, durante várias varreduras as mensagens de solicitação podem ser enviadas como sendo do ponto de vista de um usuário.

O conceito de usuário é bem ligado aos conceitos de autenticação e Gerenciamento de sessão. Quando um usuário é usado pela primeira vez em algum lugar no ZAP, uma autenticação é realizada (de acordo com o método de autenticação definido para o contexto) e uma sessão é criada e configurada para este usuário (de acordo com o gerenciamento de sessão definido para o contexto). Depois disso, as solicitações enviadas a partir do ponto de vista de um usuário são modificadas (se necessário) e enviadas de forma a que o servidor web os identifica como sendo de um usuário autenticado no webapp/Web site. If anytime a User appears to be unauthenticated (as determined by the Authentication Verification Strategy), a new authentication is performed and the Session is updated accordingly.

Para fazer a autenticação de um usuário num Web site ou em uma webapp, o método de autenticação define como a autenticação é feita (o processo), enquanto as credenciais necessárias (os identificadores exatos) são dependentes do usuário, de modo que no ZAP eles são configurados em 'Usuários'.
