Alertas

Um alerta é uma vulnerabilidade em potencial e está associada a um request específico
Um request pode ter mais de um alerta.

Os alertas são exibidos com uma bandeira indicando o tipo de risco:
      Alto
      Médio
      Baixo
      Informacional
      Falso Positivo

Alertas podem ser gerados por vários componentes do ZAP, incluindo (mas não limitados a): varredura ativa , varredura passiva, scripts, extensões, ou manualmente usando a caixa de diálogo Adicionar alerta (que também permite a você atualizar ou alterar as informações/detalhes de alerta).

Alertas são indicados na aba Histórico com uma bandeira que indica o mais alto nível de alerta.
Todos os alertas são listados na Aba de alertas e a contagem total do número de alertas por risco é exibida no rodapé.

Substituições de alerta

Os alertas gerados pelo ZAP incluem informações genéricas e específicas sobre os alertas levantados. As informações específicas estão diretamente relacionadas ao problema potencial encontrado, como o URL e o parâmetro afetado. A informação genérica inclui coisas como uma descrição e links para recursos on-line relacionados. A informação genérica inclui coisas como uma descrição e links para recursos on-line relacionados.

Você pode substituir ou adicionar as informações genéricas usando um arquivo de configuração de "substituição de alerta". Isso permite que você inclua informações específicas para sua empresa, como políticas obrigatórias, links internos ou aconselhamento para tecnologias específicas que você usa.

Um arquivo de configuração de substituição de alerta é um arquivo UTF-8 contendo apenas as informações que você gostaria de alterar. Linhas começando com '#' são tratadas como comentários e ignoradas.

O formato é: 

.  = [+ | -]
As propriedades a seguir são suportadas:
  • name
  • description
  • solution
  • otherInfo
  • reference
Por exemplo 
# 40012 = XSS refletido 40012.solution = siga nossas orientações específicas da empresa em http://internet.example.com/xss.html
Se o valor começa com um '+', então ele é acrescentado à informação existente.
Se começa com um '-' Então é anexado para as informações existentes.
Se não começa com um '+' ou '-' Então ele substitui a informação existente.

Arquivo de configuração substituir alerta pode ser especificado por meio do API, Tela de opções de alertas ou usando a linha de comando : 

-config alert.overridesFilename=<filename>
