Opções, tela de Varredura Ativa

Esta tela permite a você configurar as opções de Varredura Ativa .

Number of Hosts Scanned Concurrently

É o número máximo de hosts que podem ser escaneados ao mesmo tempo. Aumentá-lo vai acrescentar carga ao computados onde o ZAP está rodando.

Concurrent Scanning Threads per Host

Número de subdivisões de varreduras (threads) que o ZAP vai fazer em cada host.
Aumentar o número de subdivisões vai acelerar o escaneamento mas pode sobrecarregar tanto o computador onde o ZAP está rodando quando o host alvo.

Max Results to List

Número de resultados que será exibido na aba Varredura Ativa.
Exibir um grande número de resultados pode aumentar bastante o tempo que a varredura vai levar.

Maximum Rule Duration (minutes; 0 is unlimited)

O tempo máximo que qualquer regra individual pode rodar, em minutos. Zero significa sem limite. Isto pode ser usado para evitar as regras que estão tomando uma quantidade excessiva de tempo.

Maximum Scan Duration (minutes; 0 is unlimited)

O tempo máximo que a varredura inteira pode rodar, em minutos. Zero significa sem limite. Isso pode ser usado para garantir que uma verificação seja concluída em torno de um tempo definido.

Delay When Scanning (In Milliseconds)

O intervalo em milissegundos entre dois requests sucessivos.
Deixar esta variável com um valor diferente de zero vai elevar o tempo da varredura ativa mas aliviará a carga no host-alvo.

Inject plugin ID in header for all active scan requests.

Se esta opção for selecionada o scanner ativo vai injetar o cabeçalho de solicitação X-ZAP-Scan-ID com o ID do scanner que está enviando as solicitações HTTP.

Handle anti-CSRF tokens (experimental functionality).

Se esta opção estiver selecionada, a varredura ativa tentará automaticamente solicitar tokens anti CSRF .
Observe que isto é uma funcionalidade experimental e vai desacelerar o processo de varredura porque ele não será subdividido em threads para garantir que os tokens anti CSRF não se percam.

In Attack Mode prompt to rescan nodes when scope changed.

Se esta opção estiver selecionada, quando você utilizar o modo de ataque você será convidado a decidir se vai reescanear nós no nesse escopo.
Se a opção não estiver selecionada então essa decisão (se os nós serão reescaneados) fica para a próxima opção.

In Attack Mode always rescan nodes when scope changed.

Se esta opção estiver selecionada, então quando for selecionado o modo de ataque todos os nós no escopo serão reescaneados se o escopo mudar.
Isso não é recomendável para sites grandes porque pode tomar muito tempo.

Default Active Scan Policy

A política de varredura utilizada por padrão quando você inicia uma varredura ativa.

Attack Mode Scan Policy

A Política de Varredura utilizada para escaneamento no modo de Ataque.

Max Progress Chart in Mins

O tempo máximo em minutos, para o qual os códigos de resposta vão ser grafados no Diálogo de progresso da Varredura.
Para desabilitar o gráfico a opção deve ser definida como zero minutos.
