Tela de opções de Display

A tela de Display permite que você configure:

Imagens

Se o ZAP processa imagens.

Visualizar solicitações de conexão (locais)

Se as solicitações HTTP CONNECT recebidas pelo Local Proxies devem persistir na atual sessão e exibidas na aba de Histórico. O padrão é não mostrar esses pedidos. Eles acontecem com frequência (em preparação para conexões TLS/SSL, WebSockets... ) e, a menos que se inspecione o comportamento do aplicativo cliente, não são (normalmente) de muito interesse.

Tela

O layout das três janelas principais.
As seguintes opções estão disponíveis:
  • Maximizar a aba esquerda (Sites) - O painel 'árvore' contendo a aba de Sites estende-se por toda a extensão do lado esquerdo. Isto irá reduzir a quantidade de espaço disponível para o painel 'informações'.
  • Maximizar as guias inferiores (história etc) - o painel 'informações' estende-se por todo o comprimento da parte inferior. Isto irá reduzir a quantidade de espaço disponível para o painel 'árvore'.
  • Layout completo - a aba selecionada ocupa a tela inteira. Isso é útil quando usando o ZAP em telas pequenas.

Posição do painel de resposta

Permite configurar a posição da aba de Resposta em relação à de solicitação/request.
As seguintes opções estão disponíveis:
  • Abas lado-a-lado - As abas de Request e Response ficam lado-a-lado. Isto aumenta o volume de informação que pode ser exibida, mas significa que você não pode ver o Request e a Resposta ao mesmo tempo.
  • Side by Side in Tab - The Request and Response panels are shown side by side in the same tab. This decreases the information that can be displayed but means you can see both the request and response at the same time.
  • Panels Side by Side - The Request panel is shown to the left of the Response panel. Isso diminui as informações que podem ser exibidas, mas significa que você pode ver a solicitação e a resposta ao mesmo tempo.
  • Request Shown Above Response - The Request panel is shown above the Response panel. This decreases the information that can be displayed but means you can see both the request and response at the same time.
A opção de posições para o painel de Resposta não se aplica quando a opção de exibição é definida como Layout Completo.

Mostrar botões de interrupção

A localização dos botões de interrupção.

Visualização de Request grande e tamanho mínimo

O tamanho mínimo em bytes de um corpo/body de Request, a partir do qual será exibida a mensagem 'corpo da Request muito grande' em vez do próprio corpo.
Isso é para evitar que grandes corpos tornem lenta a interface do usuário.
Definir esse valor como -1 resultará na exibição do request, não importando o seu tamanho.

Visualização de Response grande e tamanho mínimo

O tamanho mínimo, em bytes, de um corpo de resposta, a partir do qual será exibida uma mensagem de 'Corpo de resposta muito grande' em vez do próprio corpo.
Isso é para evitar que grandes corpos tornem lenta a interface do usuário.
Definir esse valor como -1 resultará na exibição da reposta, não importando o seu tamanho.

Opções de time stamp da Aba de saída

Ao habilitar time stamps na aba de saída, você pode configurar o formato em que gostaria de utilizá-los. Selecione um formato predefinido na lista drop down ou digite um de sua preferência. O formato é baseado em Java SimpleDateFormat. Depois de escolher ou digitar um formato de time stamp, se você pressionar enter o exemplo à direita será atualizado para refletir a sua escolha. Se o ZAP for incapaz de usar um formato que você digitou, o exemplo exibido será baseado no formato padrão.
Longo e padrão yyyy-MM-dd HH:mm:ss
ISO8601 yyyy-MM-dd'T'HH:mm:ssZ
Apenas Hora HH:mm:ss

Font Controls

The General Font section controls default text font and size used for the ZAP display, while the Work Panels Font section controls the font used for the top right section of ZAP's main window (Ex: Request/Response, Edit and Resend tool, etc.). If you set this to -1 then the system default size will be used.
O campo 'Exemplo de fonte' irá mostrar-lhe em que tamanho o texto padrão será exibido.
Note: This setting will only take effect when ZAP is restarted.

Look and Feel

Allows to choose the look and feel of ZAP. If empty the default is used, Nimbus in Windows and Linux, the native system look and feel in macOS.
The system property swing.defaultlaf takes precedence over this option.
