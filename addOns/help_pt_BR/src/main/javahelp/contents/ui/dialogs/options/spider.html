Tela de opções do spider

This screen allows you to configure the spider options.

Note-se que modificar a maioria dessas opções também afeta a execução do spider.

Profundidade máxima de busca

O parâmetro define a profundidade máxima no processo de busca, no qual uma página deva ser encontrada para ser processada. Recursos encontrados mais abaixo deste nível não são buscados nem analisados pelo spider. The value zero means unlimited depth.

A profundidade é calculada a partir de 'sementes' (pontos de partida), então, se uma varredura do spider começa com apenas uma única URL (por exemplo, URL manually specified), the depth is calculated from this one. No entanto, se o exame começa com várias sementes (pontos de partida) recurse and Sites tree node with children), a resource is processed if it's depth relative to any of the seeds is less than the defined one.

Número de segmentos (threads) usados

O spider é multi-threaded e este é o número que define o número máximo de threads de trabalho utilizados no processo de rastreamento. Alterar este parâmetro não tem efeito sobre qualquer rastreamento já em andamento.

Duração máxima

A extensão máxima de tempo que o spider deve ser executado, medido em minutos. Zero (o padrão) significa que o spider irá rodar até encontrar todos os links que conseguir.

Número máximo de nós "filhos" a percorrer

Este parâmetro limita o número de "filhos" que serão rastreados em cada nó na árvore.
Isso é útil para aplicativos baseados em dados, que têm um grande número de 'páginas' contendo exatamente o mesmo código, mas com dados diferentes - por exemplo, de um banco de dados.
Por padrão, o número é definido como zero, o que significa que não há limites aplicados ao número de "filhos" rastreados.

Tamanho máximo a ser analisado

Define o tamanho máximo, em bytes, que uma resposta pode ter para ser analisada. Isso permite que o spider evite respostas/arquivos grandes.

Domínios Sempre no Escopo

Permite gerenciar os domínios, strings literais ou expressões regulares, que estarão no escopo do spider. O comportamento normal do spider é seguir apenas os links para recursos encontrados no mesmo domínio da página onde começou a navegação. No entanto, esta opção permite que você defina domínios adicionais considerados "dentro do escopo" durante o processo de varredura. Páginas desses domínios são processadas durante a verificação.

Manipulação de parâmetros de query

Durante a navegação, o Spider tem um mecanismo interno que marca as páginas visitadas, para não serem processadas novamente. Quando essa verificação é feita, a maneira que os parâmetros de URIs são manipulados é definida usando essa opção. Há três opções disponíveis:
  • Ignorar parâmetros completamente - se www.example.org/? barra = 456 é visitado, em seguida www.example.org/? foo = 123 não será visitado
  • Considerar apenas o nome do parâmetro (ignorar o valor do parâmetro) - se www.example.org/? foo = 123 é visitado, em seguida www.example.org/? foo = 456 não ser visitado, mas www.example.org/? barra = 789 ou www.example.org/? foo = 456 & barra = 123 serão visitados
  • Considerar tanto o nome quanto o valor do parâmetro - se www.example.org/? 123 é visitada, algum outra URI diferente (incluindo, por exemplo, www.example.org/? foo = 456 ou www.example.org/? barra = abc) serão visitados

Enviar o cabeçalho "Referer"

Se os requests do spider devem ser enviados com o cabeçalho "Referer".

Aceitar Cookies

Define se a varredura do spider deve aceitar cookies enquanto estiver rodando. Se essa opção estiver habilitada, o spider manipulará corretamente os cookies recebidos do servidor e irá enviá-los adequadamete. Se a opção estiver desabilitada, o spider não enviará quaisquer cookies em suas solicitações. Por exemplo, isto pode controlar quando, ou não, o spider usa a mesma sessão durante a varredura do spider.
Quando os cookies são aceitos, eles não são compartilhados entre varreduras do spider, cada varredura tem o seu repositório de cookies.
Esta opção tem uma prioridade baixa, o Spider respeitará outras opções (globais) relacionadas ao estado do HTTP. Está opção é ignorada se, por exemplo, a opção Enable (Global) HTTP State está ativada, quando o spider está rodando como um Usuário ou quando a Sessão HTTP está ativa.

Processar formulários

Durante o processo de rastreamento, o comportamento do spider quando encontra formulários HTML é definido por esta opção. Se desabilitada, os formulários HTML não serão processados de modo algum. Se habilitado, os formulários HTML com o método definido como HTTP GET serão submetidos/enviados com alguns valores gerados. O comportamento ao encontrar formulários com o método definido como HTTP POST é configurado pela próxima opção.

Formulários POST

Como brevemente descrito no parágrafo anterior (processamento de formulários), esta opção configura o comportamento do spider quando Processar formulários está habilitado e quando encontrar formulários HTML que tem que ser postado.

Analisar os comentários do HTML

Esta opção define se o spider deve também processar os comentários HTML à procura de links. Apenas os recursos encontrados em tags HTML comentadas e válidas serão processados.

Analisar os arquivos 'robots.txt'

Esta opção define se o spider deve também percorrer os arquivos robots.txt encontrados em sites, à procura de links para navegar. Esta opção não define se o spider deve seguir as regras impostas pelo arquivo robots.txt.

Lidar com parâmetros específicos de OData

Essa opção define se o spider deve tentar detectar parâmetros específicos de OData (ou seja, identificadores de recursos) para processá-los adequadamente de acordo com a regra definida pela opção "Manipulação de parâmetros de consulta".
