Release 2.0.0

Les modifications suivantes ont été apportées dans cette version :

Significant changes:

An integrated add-ons marketplace

ZAP can be extended by add-ons that have full access to all of the ZAP internals. Anyone can write add-ons and upload them to the ZAP Add-on Marketplace (OK, so its a Google code project called zap-extensions, but you get the idea).
Plus important encore, vous pouvez maintenant parcourir, télécharger et installer les modules complémentaires de l'intérieur ZAP. La plupart des add-ons peuvent être dynamiquement installé (et démonter), de sorte que vous ne même pas besoin d'un redémarrage.
Vous pouvez choisir d'être averti des mises à jour, et même être mis à jour automatiquement. Et comme l'analyse des règles sont à présent mises en œuvre comme des add-ons que vous pouvez obtenir les dernières règles dès qu'ils sont publiés.

A replacement for the 'standard' Spider

Les "anciens" Spider a son âge, de sorte que son été complètement réécrit, et est beaucoup plus rapide et plus complet que l'ancien. C'est encore un "traditionnel" de l'araignée, qui analyse le code HTML pour les liens qu'il peut trouver.

A new 'Ajax' spider

En plus de la traditionnelle 'l'araignée nous avons ajouté un Ajax d'araignée qui est plus efficace avec des applications qui font une utilisation intensive du JavaScript. Il utilise la Crawljax projet qui entraîne un navigateur (en utilisant le Sélénium) et peut donc découvrir tous les liens d'une application génère, même ceux générés côté client.

Web Socket support

ZAP prend désormais en charge les WebSockets, de sorte ZAP pouvez maintenant voir tous les WebSocket les messages envoyés à partir de votre navigateur. Comme avec HTTP messages, ZAP peut aussi intercepter les messages WebSocket et vous permet de changer à la volée.
Vous pouvez également fuzz WebSockets messages à l'aide de tous les fuzzing charges inclus dans le ZAP de projets tels que le JBroFuzz et fuzzdb. Et bien sûr, vous pouvez facilement ajouter vos propres fuzzing fichiers.

Quick Start tab

Le premier onglet vous allez voir est un Démarrage Rapide de l'onglet qui vous permet de taper une URL et de les scanner avec un seul clic.
C'est un point de départ idéal pour les personnes nouvelles à la sécurité de l'application, mais les experts peuvent facilement l'enlever si ils trouvent qu'il est distrayant.

Session awareness

ZAP is now session awareness, so that ZAP can recognise and keep track of multiple sessions. Il vous permet de créer de nouvelles sessions, basculer entre eux, et s'applique à toutes les autres composantes, comme l'Araignée et Active Scanner.

User defined Contexts

Vous pouvez maintenant définir un nombre quelconque de "contextes" - ensembles de l'Url qui constituent une application. Vous pouvez alors la cible de toutes les Url dans un contexte, par exemple à l'aide de l'Araignée ou Active Scanner. Vous pouvez également ajouter les contextes à la portée, et d'associer d'autres informations, telles que les informations d'authentification.

Session scope

La session champ vous permet de spécifier les contextes que vous êtes intéressé à tout moment. Vous pouvez limiter ce que vous voyez dans les différents onglets de simplement l'Url dans le champ d'application, et éviter accidentellement attaquer Url pas dans le champ d'application, en utilisant le mode Protégé.

Different modes

ZAP now supports 3 modes:
  • Coffre-fort, dans lequel aucune des opérations potentiellement dangereuses autorisées
  • Protégé, dans lequel vous pouvez effectuer toutes les actions sur les Url dans le champ d'application
  • Standard, dans laquelle vous pouvez faire quelque chose pour toutes les UrLs

Authentication handling

Vous pouvez maintenant associer les informations d'authentification avec n'importe quel contexte, ce qui permet de ZAPPER de faire des choses comme détecter si et quand vous êtes connecté et de vous connecter automatiquement. Ceci est particulièrement utile lorsqu'il est utilisé via l'API de sécurité, des tests de régression.

More API support

The REST API has been significantly extended, giving you much more access to the functionality ZAP provides.

Fine grained scanning controls

L'analyse active règles peuvent maintenant être à l'écoute afin d'adapter leur puissance (le nombre d'attaques qu'ils effectuent) et le seuil à partir duquel ils signalent les problèmes potentiels.

New and improved active and passive scanning rules

We have uploaded the results from running ZAP 2.0.0 against wavsep (the most comprehensive open source evaluation project we are aware of) to the ZAP wiki: https://github.com/zaproxy/zaproxy/wiki/Testing TODO ;)
