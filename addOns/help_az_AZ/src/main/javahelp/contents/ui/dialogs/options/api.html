Options API screen

This screen allows you to configure the API options:

Yandırılıb

If enabled then the API is available to all machines that are able to access ZAP's proxies that expose the API.

Web UI Enabled

If enabled then the API Web UI is available to all machines that are able to access ZAP's proxies that expose the API. To access the API Web UI point your browser to the host and port that ZAP is listening on.

Secure Only

Etkinleştirilirse, API yalnız https üzərindən istifadə edilə bilər. Əks təqdirdə http və https üzərindən istifadə edilə bilər.

API Key

A key that must be specified on all API 'actions' and some 'other' operations.
The API key is used to prevent malicious sites from accessing the ZAP API.
Tamamilə səpilmiş bir mühitdə zap istifadə edirsinizsə, bir düyməyə təyin etmək qətiliklə təklif edilər.

Addresses permitted to use the API

By default only the machine ZAP is running on is able to access the ZAP API.
You can allow other machines access to the API by adding suitable regex patterns.
You should only add IP addresses that you trust.
Bundan başqa, Zap API, indi icazə verilən ünvanlar olmalıdır bir Mövzu qovşağı, belə ki, qeyd yoxlayır

Disable the API Key

Selecting this option disables the API key.
Zap API 'sına daxil olmaq üçün pis məqsədli saytların icazə verdiyindən, tamamilə izolyasiya bir mühitdə zap istifadə etmədikdə bu tövsiyə olunmur

Do not require an API key for safe operations

Əgər daxil edilir, sonra API açarı və ya təqdim edildiyi "təhlükəsiz" hesab tələb olunmur digər əməliyyatlar üçün, başqa sözlə edir ki, heç bir dəyişiklik zap əməliyyatlar. Bu cür əməliyyatlar ancaq xəbərdarlıq, mesajlar və fayl sistemi yolları kimi zap məlumatlarına giriş verir They can also be used by web applications to detect the presence of ZAP.

Report permission errors via API

Əgər bu xüsusiyyət, Zap aşkar varlığını, üçün istifadə edilə bilər ki, web apps API, qoy vasitəsi ilə səhv məlumat verir. Bu etibarlı bir mühitdə ciddi bir problem deyil, amma əgər potensial pis niyyətli saytlara qarşı zap istifadə edirsinizsə o zaman bunu etkinleştirmeniz lazımdır.

Report error details via API

If this option is selected then more error details are returned via the API.
Bu səhv mesajları, pis məqsədli saytlara məlumat görür kimi ayıklama məqsədləri xaricində təklif edilməz.
Note that the full error details are always written to the ZAP log file.

Autofill API key in the API UI

If this option is selected then the API key is automatically included in the API UI.
Zap istifadə etməyiniz tövsiyə edilmir, çünki zərərli saytlar zap DOOR düyməsinə daxil olmağa imkan verir.

Enable JSONP

Selecting this option enables the JSONP format.
Bu bəzi tətbiqlər üçün faydalı ola bilər, ancaq ZAP hücum səthi sahəsi artdıqca, pis niyyətli bir site sui ola bilər xüsusiyyətləri yəni ümumiyyətlə təklif edilməz.
If JSONP is enabled then all API operations using JSONP (including views) will require the API key to prevent malicious sites from accessing sensitive information maintained by ZAP, such as session keys.
