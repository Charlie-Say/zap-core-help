Ang mga alerto

Ang isang alerto ay isang potential na kahinaan at ay nag-uugnay sa isang tiyak na kahilingan.
Ang isang kahilingan ay maaring magkaroon ng higit sa isang alerto.

Ipinapakita ng mga alerto sa mga UI kasama ang isang bandila na nagsasaad ng panganib:
      Ang mataas
      Ang katamtaman
      Ang mababa
      Ang pagbibigay ng impormasyon
      Hindi tunay na positibo

Ang mga alerto ay maaaring tumaas sa pamamagitan ng ibat ibang klase na mga bahagi ng ZAP, kasama pero hindi limitado sa: Aktibong pag-iskan, pasibong pag-iskan, ng mga iskrip, sa pamamagitan ng mga add-on mga (ekstensiyon), o manu-manong gamitin ang pagdagdag ng alerto ng diyalogo (kung saan ay pinahihintulutan karin para i-update o palitan ang alerto ng mga detalye/impormasyon).

Ang mga alerto ay naka-flag sa mga kasaysayan ng tab na mayroong isang flag nakung saan ipinapahiwatig ang pinakamataas na panganib ng alerto.
ang lahat ng mga alerto ay nakatala sa mga mga alerto ng tab at ang isang bilang ng mga kabuuang numero ng mga alerto sa pamamagitan ng panganib ay ipinakita sa mga footer.

Ang mga pag-override

Ang mga alerto na tumaas sa pamamagitan ng ZAP na kasama sa kapwa generic at partikular na impormasyon tungkol sa pagtataas ng mga alerto. Ang tinukoy na impormasyon na nauugnay ng direkta sa mga potensyal na isyu na nakita, gayon gaya ng URL at ng parametro na apektado. Ang impormasyon na generic ay kinabibilangan ng mga bagay tulad ng isang deskripsyon at mga link na may kaugnayan sa sanggunian ng online.

Maaari mong palitan o dagdagan ang mga generic na impormasyon gamit ang isang 'alerto ng pag-override' ng pagkukumpigura ng payl. Ito ay nagpapahintulot sa iyo na isama ang impormasyon na iyan ay partikular sa iyong kumpanya tulad ng iniutos na mga patakaran, ang mga panloob na link o payo para sa partikular na mga teknolohiya na iyong ginamit.

Ang isang alerto ng pag-override na kinumpigura ng payl ay isang UTF-8 na katangian ng payl na naglalaman lamang ng impormasyon na ninais mong palitan. ang guhit na nagsisimula sa '#' ay tumalakay bilang mga komento at hindi pagpansin.

Ang format ay: 

<alertid>.<property> = [ + | - ] <your information>
Ang mga sumusunod na mga katangian ay suportado:
  • ang pangalan
  • ang paglalarawan
  • ang solusyon
  • ang otherInfo
  • ang reperensiya
The format for alert tags is: 
<alertid>.tag.<key> = [ + | - ] [ <value> ]
Note that:
  • Spaces in the tag key must be escaped with a backslash.
  • The tag value is optional.
For example 
# 40012 = Reflected XSS
40012.solution = Follow our company specific guidelines at http://internet.example.com/xss.html
40012.tag.Bug\ Tracker\ URL = https://example.com/issues/1234
If the value starts with a '+' then it is appended to the existing information.
If it starts with a '-' then it is prepended to the existing information.
If it does not start with a '+' or '-' then it replaces the existing information.

Ang pag-override ng alerto ng pagkukumpigura ng payl ay maaaring tiniyak sa pamamagitan ng API, Ang mga opsyon ng alerto ng iskrin o paggamit ng command line na opsyon: 

-config alert.overridesFilename=<filename>

     Ang mga tampokibinigay sa pamamagitan ng ZAP
