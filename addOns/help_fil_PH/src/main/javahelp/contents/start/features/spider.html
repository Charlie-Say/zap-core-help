Ang Ispayder

Ang ispayder ay isng kasangkapan na ginagamit sa awtomatikong pagtuklas ng bagong pinagkukunan ng mga (URL) sa isang partikular na sayt. Ito ay nagsisimula kasama ang listahan ng mga URL na binibisita, ito ay tinatawag na ang mga seed, kung saan ito ay naka depende kung paano amg is[ayder ay sinisimulan. Ang Spider pagkatapos ay bibisita itong mga URL, ito ay kumikilala sa laht ng mga hyperlink sa mga pahina at idinadagdag ang mga ito sa mga listahan ng mga URL na bumisita at ang prosesong ito na nagpatuloy ng recursively hanggat may mga bagong pinagkukuhaan na nakita.

Maaaring makumpigura ang ispayder at magsimula gamit ang diyalogo ng ispayder.

Sa panahon ng proseso ng isang URL, ang ispayder ay gagawa ng isang kahilingan na ikuha sa pinagmumulan at pagkatapos ay siyasatin ang tugon, pagkilala sa mga hyperlink. Ito ay kasalukuyang mayroong sumusunod na pag-uugali habang nagpa-process ng mga klase ng mga tugon:

Ang HTML

Ang pag-process ng mga partikular ng mga tag, pagkilala sa mga link ng bagong mga pinagkukunan:
  • Base - ang tamang pamamahala
  • Isang, Link, lugar - 'href' katangian
  • Kuwadro, IFrame, iskrip, Img - 'src' katangian
  • Meta - 'http-equiv' para sa 'lokasyon' at 'refresh'
  • Ang anyo - tamang pag gamit ng mga anyo na may parehong GET at POST na paraan. Ang kahalagahan ng mga field ay ang wastong pagbuo, kasama ang mga klase ng HTML 5.0 na pag-input.
  • Mga komentaryo - makikita ang mga wastong tag sa mga komentaryo na sinusuri din, kung tumutukoy sa Mga Opsyon ng Ispayder ng iskrin

Ang Robots.txt na payl

Kung nakatakda sa mga Opsyon ng ispayder, ang 'robot.txt na payl ay sinusuri din at sinusubukan na kilalanin ang bagong sanggunian gamit ang tinutukoy na mga patakaran. Dapat ito ay mabanggit na ang ispayder ay hindi sumusunod sa mga patakaran na tinutukoy sa mga 'robot.txt'na payl.

Ang ayos ng OData Atom

Ang nilalaman ng OData gamit ang ayos ng Atom ay kasalukuyang sinusuportahan. kasama ang lhat ng mga link na (kaugnay o lubos) ang pag-proseso.

Ang Iba pang mga aspeto

  • Habang nagsisiyasat kung ang URL ay binisita, ang gawain tungkol sa kung paano ang mga parametro ay mahahawakan na maaaring makumpigura sa mga ispayder ng mga opsyon ng iskrin.
  • Habang nagsisiyasat kung ang isang URL aybinisita, mayroong isang madalang na karaniwang mga parametro kung saan ay hindi pinapansin: jsessionid, phpsessid, aspsessionid, utm_*
  • Ang gawain ng mga ispayder tungko sa mga cookie na nakadepende s kung paano ang ispayder ay magsisimula at kung alin sa mga opsyon ay pagaganahin. Para sa iba pang mga detalye sumangguni sa mga opsyon ng ispayder ng iskrin.

Ang ispayder ay makukumpigura gamit ang Mga Opsiyon ng Ispayder ng iskrin.
