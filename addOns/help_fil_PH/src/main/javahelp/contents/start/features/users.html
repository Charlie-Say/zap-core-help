Ang mga gumagamit

Ang mga gumagamit ay ang mga represetasyon ng ZAP ng mga websayt/mga webapp ng mga gumagamit. Pinahihintulutan nila ang ilang mga aksyon na maisagawa mula sa punto ng pagpapakita ng isang paggamit ng mga webapp. Para sa bawat konteksto, ng isang ayos ng mga gumagamit na maaring matukoy, kung saan ay maaaring magamit sa mga aksyon na may kaugnayan sa mga konteksto. Ang pinaka karaniwan, sa panahon ng ibat-ibang mga pag-iskan ng mensahe ng kahilingan na maaaring maipadala mula sa punto ng ipinakita ng isang gumagamit.

Ang konsepto ng mga gumagamit ay mahigpit na nakatali sa mga konsepto ng sesyoon ng pamamahala at pagpapatunay. Habang ang isang gumagamit ay unang gumamit saanman sa ZAP, ang isang pagpapatunay ay naisagawa ( ayon sa mga pagpapatunay ng natukoy na paraan para sa mga konteksto) at sa isang sesyon ay nabuo at nakumpigura para sa gumagamit na ito (ayon sa mga sesyon ng pamamahala na natukoy para sa mga konteksto). At pagkatapos, ang mga kahilingan na naipadala mula sa mga punto ng naipakita ng isang gumagamit ay nabago (kung kinakailngan) at naipadala sa gayong isang paraan na ang server ng web ay tinukoy nila upang maipadala sa pamamagitan ng isang pagpapatunay ng webapp/websayt ng gumagamit. If anytime a User appears to be unauthenticated (as determined by the Authentication Verification Strategy), a new authentication is performed and the Session is updated accordingly.

Upang maisagawa ang pagpapatunay para sa isang gumagamit sa isang websayt / sa isang webapp, ang paraan ng pagpapatunay ay natukoy kung paano ang pagpapatunay ay nagawa (ang proseso), habang ang kinakailangan na mga kredensyal ( ang eksktong mga pagkakakilanlan) ay depende sa mga gumagamit, kaya, sa ZAP, sila ay nakumpigura sa mga gumagamit.
