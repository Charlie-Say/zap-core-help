Ang mga opsyon ng aktibong pag-iskan ng iskrin

Ang iskrin na ito ay nagpapahintulot sa iyo na ikumpigura ang ang aktibong pag-iskan ng mga opsyon:

Number of Hosts Scanned Concurrently

Ang ang pinakamataas na bilang ng mga host na na-iskan sa magkaparehong oras. ang pagdami nito ay maaaring maglagay ng karagdagang hirap sa mga ZAP ng computer na tumatakbo.

Concurrent Scanning Threads per Host

Ang bilang ng mga hibla ng iskaner na ginamit kada host.
Ang pagdadagdag ng bilang ng mga hibla na pinabilis ng pag-iskan subalit maaaring maglagay ng karagdagang hirap sa mga ZAP ng computer na tumatakbo at ang puntirya ng host.

Max Results to List

Ang bilang ng mga resulta na ipapakita sa mga aktibong pag-iskan ng tab.
Ang pagpapakita ng isang malaking bilang ng mga resulta na maaaring lubhang pagtataas ng oras na tagal ng isang pag-iskan.

Maximum Rule Duration (minutes; 0 is unlimited)

Ang pinakamataas na oras ng sinumang indibidwal na patakaran na maaaring tumakbo para sa ilang mga minuto. ibig sabihin ng sero ay walang limit. ito ay maaaring magamit para maiwasan ng mga patakaran na makakuha ng isang malaking halaga ng oras.

Maximum Scan Duration (minutes; 0 is unlimited)

Ang pinakamataas na oras na ang buong pag-iskan ay maaring tumakbo para sa ilang minuto. ibig sabihin ng sero ay walang limit. ito ay maaring magamit para masiguro na ang isang pag-iskan ay nakumpleto sa loob ng itinakdang oras.

Delay When Scanning (In Milliseconds)

Ang pagkaantala sa mga millisecond sa pagitan ng bawat kahilingan.
Ang pagtatakda nito sa isang hindi serong halaga ay maidadagdag sa oras na nakuha sa isang aktibong pag-iskan, subalit ito ay maglalagay sa mababang hirap sa mga pinuntirya ng host.

Inject plugin ID in header for all active scan requests.

Kung ang opsyon na ito ay napili ng aktibong iskaner na ma-iniksyon sa kahilingan ng header na X-ZAP-Scan-ID sa pamamagitan ng ID ng mga iskaner na nagpapadala ng mga kahilingan ng HTTP.

Handle anti-CSRF tokens.

Kung ang opssyon na ito ay napili habang ang aktibong iskaner ay sinusubukan sa awtomatikong kahilingan ng anti CSRF ng mga token kung kinakailngan.
Previously this would have forced the scanner to only use one thread, but that is no longer the case. You are strongly recommended to check that the anti CSRF tokens are being correctly generated if more than one thread is being used, for example using custom tags to check for success / failure patterns in the response.

In Attack Mode prompt to rescan nodes when scope changed.

If this option is selected then when you select Attack moda you will be prompted to choose whether to rescan nodes in scope.
If the option is not selected then the following option will control whether the nodes are rescanned.

In Attack Mode always rescan nodes when scope changed.

If this option is selected then when running in Attack mode all nodes in scope will be rescanned if the scope changes.
This is not recommended for large sites as it could take a long time.

Default Active Scan Policy

The Patakaran ng pag-Iskan that is used by default when you start an active scan.

Attack Mode Scan Policy

The Scan Policy that is used for scanning in Attack mode.

Max Progress Chart in Mins

The maximum time in minutes for which response codes will be charted in the Scan Progress dialog.
To disable the chart the option should be set to zero minutes.
