Ang mga opsyon ng iskrin ng ispayder

This screen allows you to configure the spider options.

Ito ay dapat na maitala na ang karamihan ng pagbabago ng mga opsyon na ito ay nakakaapekto sa pagpapatakbo ng ispayder.

Ang pinakamalalim na pag gapang

Ang parametro na tumutukoy sa pinakamalalim sa mga pag gapang ng proseso kung saan ang isang pahina ay dapat na maging kita sa ayos para sa magiging proseso na ito. Ang mga pinagmumulan na natagpuan ang malalim pa sa lebel na ito ay hindi kinuha at sinuri ng mga ispayder. The value zero means unlimited depth.

Ang lalim ay kalkulado mula sa simula ng mga seed, kaya, kung ang isang ispayder ng pag-iskan ay nagsisismula sa isang solong URL lamang (hal. URL manually specified), the depth is calculated from this one. Ganun pa man, kung ang mga simula ng maramihang pag-iskan ng mga seed (hal. recurse and Sites tree node with children), a resource is processed if it's depth relative to any of the seeds is less than the defined one.

Ang bilang ng mga ginagamit na mga Thread

Ang ispayder ay maramihang mga thread at ito ay ang bilang na mga inilarawan sa pinakamataas na bilang ng mga manggagawa ng mga thread na ginamit sa mga proseso ng paggapang. Ang pagbabago ng parametrong ito na hindi nakakaapekto sa kahit na anumang proseso ng paggapang.

Pinakamataas na tagal

Ang pinakamahaba na oras para ang ispayder ay dapat na tumakbo, ay nasusukat sa mga minuto. ang sero (Ang default) ay nangangahulugan na ang ispayder ay tatakbo hanggang sa makita ang lahat ng mga link na ito ay ginawa na.

Ang pinakamataas na bata ng paggapang

Ang bilang ng parametro ay nilimitahan sa mga bata na gumagapang at ang bawat node sa mga puno.
Ito ay tulong para sa datos na pinapatakbo ng mga aplikasyon na mayroong malaking mga bilang ng mga 'pahina' ng nasa totoo talaga na mga parehas na code pero naglalaman ng ibat-ibang datos, para sa halimbawa para sa isang database.
Sa pamamagitan ng default ito ay itinakda sa sero kung saan ay nangangahulugan na doon ay walang mga limitasyon na nailapat sa mga bilang ng bata na mga node na gumapang.

Ang pinakamataas na sukat ng parse

Ang pagtukoy sa pinakamataas na sukat, sa mga byte, na ang isang tugon ay dapat na maging parse. ito pinahihintulutan ng ispayder na laktawan ang mga tugon/mga payl.

Ang mga domain ay laging nasa saklaw

At pagpapahintulot ng pamamahala ng mga domain, ang mga literala na hanay o ang mga palagiang ekspresyon, na nasa saklaw ng mga ispayder. Ang normal na pag-uugali ng mga ispayder ay ang pagsunod lamang sa mga link na makita ang mga pinagkukunan sa mga magkakaparehong domain bialng ang pahina kung saan nagsimula ang pag-iskan. Ganunpaman, ang opsyon na ito ay pinahihintulutan ka na matukoy ang mga karagdagan ng domain ay na itinuturing "sa saklaw" habang ang pag gapang ay nasa proseso. Ang mga pahina ng mga domain na ito ay pinuproseso habang nag-i-iskan.

Ang pag-uusisa sa mga hinahawakang mga parametro

Habang gumagapang, ang ispayder ay mayroong isang panloob na mekanismo na nagmamarka kung alin mga pahina ay nabisita na, kaya sila ay hindi na maproproseso ulit. Habang ginagawa ang pagsusuring ito, ang paraan ng mga URL ay maayos na nahahawakan ng mga parametro gamit ang opsyon na ito. Mayroong tatlong opsyon na magagamit:
  • ang kumpletong pag-iwas ng mga parametro -kung ang www.example.org/?bar=456 ay nabisita, pagkatapos ang www.example.org/?foo=123 ay hindi nabisita
  • Ang itinuturing lamang na pangalan ng mga parametro (ang pag-iwas sa mga halaga ng mga parametro) - habang ang www.example.org/?foo=123 ay nabisita, pagkatapos ang www.example.org/?foo=456 ay hindi nabisita, subalit ang www.example.org/?bar=789 o ang www.example.org/?foo=456&bar=123 ay nabisita
  • itinuturing ang dalawang pangalan ng mga parametro at halaga - kung ang www.example.org/?123 ay nabisita, ang anumang uri yan ay magkaiba (kasama ang, para sa halimbawa, www.example.org/?foo=456 o www.example.org/?bar=abc) ay nabisita

Ang pagpapadala sa header ng "referer"

Kung ang mga kahilingan ng ispayder ay dapat na maipadala sa mga "referer" ng header.

Ang pagtanggap sa mga cookies

Kung ang mga pag-iskan ng ispayder ay dapat na tanggapin ng mga cookie habang nag-ispayder. Kung ang ispayder ay pinagana ng maayos na paghawak sa anumang tinanggap ng mga cookie mula sa server at ipinapadala sa kanila pabalik alinsunod dito. Kung ang opsyon ay hindi pwede, ang ispayder ay hindi magpapadala ng anumang mga cookie na nsa kahilingan nito. Para sa halimbawa, maaari nitong kontrolin maging o hindi ang ispayder ang gumamit sa magkaprehas na sesyon sa lahat ng parte ng pag -iskan ng ispayder.
Kapag tinanggap ang mga cookie ang mga cooki ay hindi maibabahagi sa pagitan ng mga pag-iskan ng ispayder, bawat pag-iskan nito ay mayroong cookie jar.
Ang opsyon na ito ay mayroong mababang prayoridad, ang ispayder ay nirerespeto ang ibang (global) na mga opsyon na may kaugnayan sa mga estado ng HTTP. ang opsyon na ito ay hindi pinapansin kung, para sa halimbawa, ang opsyon Enable (Global) HTTP State ay paganahin, habang ang pag-ispayder ng tulad ng Gumagamit o habang ang isang Sesyon ng HTTP ay aktibo.

Ang mga anyo ng proseso

Habang ang proseso ng pag gapang, ang kilos ng mga ispayder habang ang pagtatagpo ng mga anyo ng HTML ay tinutukoy sa pamamagitan ng opsyon na ito. Kung di gumana, ang mga anyo ng HTML ay hindi mapoproseso sa lahat. Kung gumana, ang mga anyo ng HTML sa mga paraan na tinukoy bilang HTTP GET ay maisasailalim sa mga binuong mga halaga. Ang kilos habang nagtatagpo ang mga anyo sa mga paraan na natukoy bilang HTTP POST ay nakumpigura sa mga susunod na opsyon.

Ang mga anyo ng POST

Bilang madaling sabi ng pagtukoy ng mga nakaraang talata (ang mga anyo ng proseso), ang opsyon na ito na kinumpigura ng kilos ng mga ispayder habang Ang mga anyo ng proseso ay gumagana at habang nagtatagpo ang mga anyo ng HTML kinakailangang ma post.

Mga komento sa Parse ng HTML

Ang mga opsyon na ito na tinukoy kung ang ispayder ay dapat din sa ispayder sa HTML na mga komento ng paghahanap para sa mga link ng mga pinagmulan. Ang pinagmumulan at makikita lamang sa balidong comento ng HTML ng mga tag na naproseso.

Ang parse"robots,txt' na mga payl

Ang opsyon na ito na tinutukoy habang ang ispayder ay dapat na ispayder din sa robot.txt na mga payl na makikita sa mga website,ang paghahanap para sa mga link sa mga pinagmumulan. Ang mga opsyon na ito ay hindi matutukoy habang ang ispayder ay dapat na masundan ang mga patakaran na ipinapataw sa pamamagitan ng robot.txt na payl.

Ang paghawak ng OData-specific na mga parametro

Ang opsyon na ito na tumutukoy habang ang ispayder ay dapat na subukang malaman ang OData-specific na mga parametro (i.e. ang mga pinagmumulan ng mga pagkakakilanlan) sa kaayusan ng pagpapatakbo ng maayos sa kanila ayon sa mga patakaran ng tinukoy sa pamamagitan ng "paghawak ng mga tanong ng mga parametro" ng opsyon.
