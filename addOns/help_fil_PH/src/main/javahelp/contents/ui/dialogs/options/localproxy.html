Ang mga opsyon ng lokal na mga proxy ng iskrin

This screen allows you to configure the addresses and ports on which ZAP accepts incoming connections.

Local Proxy

By default ZAP will listen on one local address and port, and these are the address and port that you must configure your browser to use as a proxy.

Ang Address

Ang lokal na address kung saan ang ZAP ay magagamit. Ang lahat ng mga magagamit na mga address ay awtomatikong kikilalanin.

Ang Port

Ang port na kung saan ang ZAP ay makikinig.

Ang likod ng NAT

Ay magpapakita na ang lokal na proxy ng (ZAP) ay nasa likod ng NAT. Kapag napili ang ZAP ay magtatangka upang alamin ang IP address ng publiko, sa maayos na pagsisiyasat at pamamahala ng mga kahilingan sa mga publiko ng IP address (para sa halimbawa, upang maglingkod sa pamamagitan ng ZAP ng API).

Paalala: Ang opsyon na ito ay suportado lamang kapag ang ZAP sy tumatakbo sa isang AWS EC@ ng pagkakataon. ang ZAP ay magtatamo sa IP address ng publiko mula sa AWS EC2 ng mga pagkakataon ng Metadata.
Ang ZAP ay dapat na maging simula sa pagpapagana ng opsyon na ito kung pinatakbo sa mga API, sa pamamagitan ng IP address ng publiko, ay kinakailngan:

ang zap.sh-daemon-port 8080 - host 0.0.0.0 - kumpig proxy.behindnat=true
Ganundin, ang API na nangangailngan na maikumpigura upang matanggap ang panlabas na mga IP address (i.e. ang IP adress mula sa kung saan ang ZAP ay lubusang mapapatakbo).

Ang pagtanggal ng hindi suportadong mga pag-encode

Pinahihintulutan ang proxy na tanggalin ang hindi suportadong mga pag-encode mula sa "Accept-Encoding" ng request-header ng field, kaya walang (hindi suportadong) pag-encode na mga pagbabago na nagawa sa mga tugon.
Ang opsyon na ito ay dapat na laging gumagana maliban kung ang pagsubok ng pag-encode ng mga pagbabago.
Ang mga mensahe na na-encode sa hindi suporatdong pag-eencode ay hindi magiging tama sa pag-iskan (sinuman sa pasibo at aktibong mga iskaner).

Ang seguridad ng mga protokol

Ang pagpapahintulot na piliin ang SSL/TLS na mga bersyon na gumagana para sa paparating na mga koneksyon (para sa halimbawa, mula sa mga browser). Kahit na isang bersyon ang dapat na patakbuhin, ang mga hindi suportadong mga bersyon sa pamamagitan ng JRE ay hindi mapipili at hindi papaganahin.
Ang opsyon ng SSLv2Hello ay dapat mapili sa kaugnay ng kahit na isang SSL/TLS na bersyon.
Ang seguridad ng mga opsyon na ilalapat sa parehong mga pangunahin at ang lahat ng mga nadagdag na mga proxy.

Ang karagdagang mga proxy

Maaari mong idagdag tulad ng maraming nadagdag na mga address at mga port para s ZAp upang pakinggan sa kahit anung gusto mo.
Ito ay maaaring maging kapakipakinabang habang sinusubukan ang mga aplikasyon ng mobile - maaari mong i-proxy ang mobile na aplikasyon sa pamamagitan ng isang Wi-Fi na interface sa magkaparehong oras bilang paggamit ng backend na sayt na direkta gamit ang isang browser na na-proxy sa pamamagitan ng ZAP na tularan ang isang gumagamit na ahente ng mobile.

Intercepting/Transparent Proxy

The local proxies can be used as intercepting/transparent proxies for both HTTP and HTTPS. For HTTPS the client applications (e.g. browser) need to use the TLS extension Server Name Indication. This allows you to set up a testing LANs (or VMs) where all HTTP and HTTPS traffic is proxied regardless of software settings.

For example, if you have a Linux machine you use for testing, you can do something like the following to forward all HTTP and HTTPS traffic to a local proxy listening on 192.168.0.14:8080: 

iptables -t nat -A OUTPUT -p tcp --dport 443 -j DNAT --to-destination 192.168.0.14:8080
iptables -t nat -A OUTPUT -p tcp --dport 80 -j DNAT --to-destination 192.168.0.14:8080
