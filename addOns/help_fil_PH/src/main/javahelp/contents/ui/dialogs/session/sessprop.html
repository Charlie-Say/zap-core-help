Ang mga regex ng URL

Sa mga Hindi isinama mula * sa mga diyalogo, maaaring kang pumasok na regular ng mga ekspresyon upang matukoy ang hindi isinama na maga URL. habang maaari kang makatakas sa isang solong meta-karakter sa isang backslash, maaari moring gamitin ang \Q...\E ang pag-alis ng pagkakasunod-sunod. Ang lahat ng mga karakter sa pagitan ng \Q at ng \E ang mga interpreted ay bilang mga literal na karakter. E.g.\Q*\d+*\E matches the literal text *\d+*. aang pag-alis naito ng pagkakasunod-sunod ay ginamit sa ZAP habang hinihiwalay mo ang mga URL sa pamamagitan ng ilang mga konteksto ng mga menu.
Mga paalala: Kung ang URL mo ay naglalaman ng isang "\E", pagkatapos kailangan mong gawin ang mga sumusunod na mga hakbang kapag gagamitin ang \Q...\E ang pagkakasunod-sunod ng escape:
  • Buksan ang escape na pagkakasunod-sunod
  • Isarado ang pagkakasunod-sunod ng escape bago ang "character" \E
  • Ang escape ng backlash
  • Buksan pagkatapos ang "\E" ang iba pang escape ng pagkakasunod-sunod;
  • Isarado ang escape ng pagkakasunod-sunod tulad ng pang karaniwang nais.

Ang Halibawa ng: subdomain.example.com/path?a=\E&moredata=2 ay nararapat na lumitaw bilang \Qsubdomain.example.com/path?a=\E\\E\Q&moredata=2\E
