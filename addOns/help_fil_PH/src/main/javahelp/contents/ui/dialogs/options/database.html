Mga Opsyon ng Screen ng Database

Ang screen na ito ay nagpapahintulot sa iyo na magkumpigura ng mga opsyon ng database:

Compact (sa paglabas)

Nagpapahintulot na i-compact ang database kapag sarado ang ZAP, ang pagko-compact ng database ay nagtitiyak ng minimal na espasyo sa paggamit ng dish ngunit ito ay mas matagal din na makapagsara sa ZAP.

Log ng Recovery

Kumukontrol kung o hindi paganahin ang log ng recovery ng database.
Nagpapabuti sa pagsasagawa ng database kapag hindi pinagana ngunit maaaring humantong sa pagkawala ng data kapag ang ZAP ay biglang na-exit.
Paalala: ang kasalukuyang sesyon ay hindi maapektuhan, ang mga pagbabago ay magkakabisa sa bago at nakabukas na mga sesyon.

Pinakamalaking sukat ng Reqest Body

Ang pinakamalaking sukat ng request body sa bytes na pinahintulutan ng ZAP.

Pinakamalaking laki ng Response Body

Ang pinakamalaking sukat ng response body sa bytes na pinahintulutan ng ZAP.

Mag-prompt para sa mga opsyon ng persistence sa bagong sesyon

Kung naka-check, ang Diyalogo ng sesyon ng Persist ay makikita kapag ang isang bagong sesyon ay nalikha.

Default na opsyon

Ang default na opsyon para sa mga bagong sesyon - kung ang nasa taas na prompt ay pinagana, ang default na opsyon ay mapipili.
Kung ang prompt ay hindi pinagana, ito ang mga aksyon na gagawin.
