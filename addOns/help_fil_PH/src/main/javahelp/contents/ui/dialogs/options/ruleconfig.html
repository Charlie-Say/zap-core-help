Mga opsyon sa Pagsasaayos ng Rule screen

Pinapayagan ka ng screen na ito na i-configure ang pag-uugali ng mga tukoy na aktibo at passive na mga panuntunan sa pag-scan.

Piliin ang alinman sa mga nakalistang hanay upang makita ang mga detalye ng partikular na pagsasaayos at upang baguhin ang nauugnay na halaga. Maaaring i-reset ang parehong indibidwal at lahat ng mga panuntunan.

Ang built-in na mga patakaran ay kinabibilangan ng:

PangalanPagwawalang bahalaDeskripsyon
rules.common.sleep15Ang haba ng oras sa segundo na ginagamit para sa timing ng atake.
rules.cookie.ignorelistPinaghihiwalay ng isang kuwit na listahan ng mga pangalan ng cookie. Ang mga cookie na kasama sa listahan na ito ay hindi papansinin kapag nag-scan para sa mga isyu na may kinalaman sa cookie.
rules.csrf.ignorelistIsang pinaghiwalay na kuwit ng listahan ng mga identifier. Ang anumang FORM na may pangalan o ID na tumutugma sa isa sa mga tagatukoy na ito ay hindi papansinin kapag nag-scan para sa mga nawawalang mga token ng Anti-CSRF. Gamitin lamang ang feature na ito para huwag pansinin ang FORMs na malalaman mo na ligtas, halimbawa pagsusuri ng mga anyo.
rules.csrf.ignore.attnameThe name of an HTML attribute that can be used to indicate that a form does not need an anti-CSRF Token. If rules.csrf.ignore.attvalue is specified then this must also match the attribute's value. If found any related alerts will be raised at INFO level.
rules.csrf.ignore.attvalueThe value of an HTML attribute named by rules.csrf.ignore.attname that can be used to indicate that a form does not need an anti-CSRF Token. If found any related alerts will be raised at INFO level.
rules.domains.trustedA comma separated list of URL regex patterns. Any URLs that match the patterns will be considered trusted domains and the issues ignored.
