Ang mga opsyon ng display screen

Ang mga Display screen ay nagpapahintulot sa iyo na kumpigurahin ang:

Ang imahe

Kapag ang ZAP ay nagproproseso ng mga imahe.

Ipakita ang (lokal) na CONNECT ng mga pakiusap

Kung ang mga kahilingan ng HTTP CONNECT ay matatanggap sa pamamagitan ng Lokal na mga Proxies dapat lang na magpatuloy sa kasalukuyan ang sesyon at ipinapakita na sa Ang History tab. Ang default ay para lamang sa hindi pagpapakita ng mga kahilingan, ito ay nangyayaring madalas (sa paghahanda para sa TLS/SSL, Websocket... koneksyon) at, maliban pagaralan ang ugali ng kliyenteng may aplikasyon, ay hindi (gaano) nakakainteresado.

Ang Display

Ang layout ng 3 pangaunahing mga panel.
Ang mga sumusunod na mga opsyon ay magagamit sa:
  • Palakihin ng labis na kaliwang (mga Site) tab - Ang 'tatlong' panel ay naglalaman ng mga site tab na pwedeng makadagdag Mula sa buong haba ng kaliwang kamay. Mababawasan nito ang halaga ng espasyo na pwedeng gamitin para sa 'impormasyon' sa panel.
  • Palakihin ang nasa ilalim na (History atbp) mga tab- Ang 'impormasyon' ng panel ay palawigin para sa Ang buong haba ng buttom. Mababawasan nito ang halaga ng espasyo na pwedeng gamitin para sa 'tatlong' panel.
  • Ang kabuoang Pagkakaayos - Ang napiling tab ay kukuwa ng buong screen. ito ay kagamit-gamit sa tuwing gagamit ng ZAP sa maliit na mga screen.

Tumugon sa Posisyon ng Panel

Ang pagpapahintulot upang kumpigurahin ang posisyon ng tab sa pagtugon at respeto sa tab ng kahilingan.
Ang mga sumusunod na mga opsyon ay maaari ng magamit:
  • Ang mga tab side by side - Ang kahilingan at pagtugon na mga tabs ay gilid sa gilid din. Ang pagtaas na ito ang impormasyon na maaaring makita pero ibig sabihin hindi mo maaaring makita ang parehong kahilingan at katugunan sa parehong oras.
  • Side by Side in Tab - The Request and Response panels are shown side by side in the same tab. This decreases the information that can be displayed but means you can see both the request and response at the same time.
  • Panels Side by Side - The Request panel is shown to the left of the Response panel. Ang pagbaba na ito ay ang impormasyon na maaaring makita pero ibig sabihin maaari mo rin itong makita pareho ang kahilingan at pagtugon sa magkaparihong oras.
  • Request Shown Above Response - The Request panel is shown above the Response panel. This decreases the information that can be displayed but means you can see both the request and response at the same time.
Ang Response Panel Position na opsyon ay hindi ma-a-apply kapag ang opsyon ng display ay naitakda na sa kabuoang pagkakaayos.

Ipakita ang mga break button

Ang lokasyon ng mga break button.

Ang malaking kahilingan na makita ang laki ng min

Ang minimum na laki ng isang katawan ng aplikasyon sa mga byte na kung saan ang punto ng isang 'malaking katawan ng aplikasyon' ang minsahe ay maaaring makita kaysa sa aktwal na katawan.
Ito ay para maiwasan ang pinakamalaking katawan sa pagkababa ng UI.
Ang paglagay ng halaga sa -1 ay resulta sa aplikasyon ito ay laging makikita gaano man kalaki ito.

Ang malaking pagtugon na makikita ang laki ng min

Ang pinakamaliit ng laki ng nilalaman ng tugon sa bytes kung saan ang mensahe na 'malaking nilalaman ng tugon' ay makikita sa halip na ang aktwal na nilalaman.
Ito ay para maiwasan ang pagbagal ng malaking katawan ng UI.
Ang paglagay ng halaga sa -1 ay resulta sa aplikasyon ito ay laging makikita gaano man kalaki ito.

Opsyon ng Output ng mga tab ng Time Stamp

Sa oras na pinagana mo ang time stamps sa output tab maaari mo itong i-configure ang pormat na nais mong lumabas na mga time stamp. Alinman ang piliin sa isang pre-defined na pormat mula sa listahan ng drop-down o ilagay ang isa sa iyong napili. Ang pormat ay base sa Java SimpleDateFormat. Pagkatapos mong mamili o ilagay ang pormat ng time stamp kung pipindutin mo ang enter ang halimbawa sa kanan ay maga-update para magrepleka sa iyong napili. Kung ang ZAP ay di maaaring magamit para magamit ang isang pormat na iyong nilagay kung magkagayon ang halimbawa ay makikita base lamang sa pormat ng Default.
Mahaba & Default yyyy-MM-dd HH:mm:ss
ISO8601 yyyy-MM-dd'T'HH:mm:ssZ
Oras Lamang HH:mm:ss

Font Controls

The General Font section controls default text font and size used for the ZAP display, while the Work Panels Font section controls the font used for the top right section of ZAP's main window (Ex: Request/Response, Edit and Resend tool, etc.). If you set this to -1 then the system default size will be used.
Ang 'Halimbawa ng Font' ay maipapakita sayo ang larangan kung gaano kalagi ang defualt ng teksto na lumabas.
Note: This setting will only take effect when ZAP is restarted.

Look and Feel

Allows to choose the look and feel of ZAP. If empty the default is used, Nimbus in Windows and Linux, the native system look and feel in macOS.
The system property swing.defaultlaf takes precedence over this option.
