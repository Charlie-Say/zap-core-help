Ang ispayder na tab

The Spider tab shows you the set of unique URIs found by the Spider during the scans.

Ang 'bagong pag-iskan ' ng buton ay inilunsad ang ispayder na diyalogo kung saan ay pinahihintulutan ka na matukoy ng eksakto kung ano ang dapat na maging pag-iskan.
ang ispayder ay maaaring mapatakbo sa maraming sayt sa magkakahelera at ang mga resulta para sa bawat pag -iskan ay ipinakita sa pamamagitan ng pagpili ng pag-iskan sa pamamagitan ng 'progreso' ng paghatak pababa.

Ipinapakita ng toolbar ang impormasyon tungkol s isang pag-iskan at pagpaphintulot na makontrol ito. ito ay magbibigay ng isang ayos ng mga buton kung saan ay pinahihintulutan para:

  • Ang paghinto (at pagsisimula muli) ang napili ng ispayder ng pag-iskan;
  • Ang pagtigil ng napili ng ispayder ng pag-iskan;
  • Ang paglilinis ng kumpleto ng mga pag-iskan;
  • Open the Spider Options screen.
Ang progreso ng bar ay nagpapakita kung gaano kalayo ang napiling ispayder ng pag-iskan na mayroong progreso. ito ay ipinakita rin ang bilang ng aktibong ispayder ng mga pag-iskan sa mga bilang ng mga URI na nakita para sa mga napiling pag-iskan.

Para sa bawat URI nanakita mo na maaaring tingnan:

  • Ang pagproseso - kung saan ang URI ay naproseso sa pamamagitan ng ispayder o ang pag laktaw mula sa pagkuha dahil sa isang patakaran (hal. ito ay labas sa saklaw)
  • Ang paraan - ang paraan ng HTTP, hal. ang GET o POST, sa pamamagitan ng kung saan ang pinagkukunan ay dapat na mapatakbo
  • Ang URI - ang nahanap na pinagkukunan
  • Ang mga flag - ang alin man sa imposramasyon tungkol sa URI (hal. kung ito ay isang binhi o bakit ito ay hindi naproseso)

For each spider message, shown under the Messages tab, you can see details of the request sent and response received. The Naproseso column, indicates whether:

  • Successfully - the response was successfully received and parsed
  • Empty Message - the response was not parsed because it was empty
  • I/O Error - an input/output error occurred while fetching the response
  • Max Children - the response was not parsed because the corresponding parent Sites node already has more child nodes than the maximum allowed
  • Max Depth - the response was not parsed because it passed the maximum depth allowed
  • Max Size - the response was not parsed because its size is not under the maximum allowed
  • Not Text - the response was not parsed because it's not text, for example, an image
  • Spider Stopped - the response was not fetched or parsed because the spider was already stopped
