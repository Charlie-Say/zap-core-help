Ang Inilabas na 2.7.0

Ito ay pagsasaayos ng isang bug at mas papagandahin ang inilabs nito, kung saan ang Java 8 ay nangangailangan ng minimum.

Ilan sa mga lalo pang mga mahahalagang pagsasaayos kasama ang:
  • Ang browser ay inilunsad kasama ng default -pinahihintulutan ka nito na ilunsad ang mga browser mula sa ZAP na naka preconfigured sa proxy sa pamamagitan ng ZAP at hindi pansinin ang sertipiko ng mga babala dahil sa ugat ng ZAP ng sertipiko.
  • Ang pag papahintulot ng zap na pakinggan ang maraming mga address/port
  • Ang pag suporta ng pangalan ng server ng indikasyon
  • Ang pag-updated ng NTLM na makina ng pagpapatupad - ang mga kasong isinasaayos nito kung saan ang dominyo ay napatunayan at pinagbubuti ang interoperability kasama ang ibang mga (server) ng NTLM ng mga pagpapatupad
  • Karamihan sa mga bagong endpoin ng APi - tingnan sa ibaba ang mga detalye
Paalala na kung meron kang naging mga problema sa mga inilabas na ito pagkatapos ay mayroon ring isang bagong Tulong/Suporta na impormasyon...' ang aytem ng menu na nagbibigay ng mga mahahalagang impormasyon tungkol sa pag-install mo ng ZAP kung saan kailangan mo dapat na isama ang kahit anung mga isyu na iyong pinalaki.

Ang mga pagbabgo ng mga sirang ZAP API:

Ang AKSYON ng pagpapatunay / setAuthenticationMethod

Ang klase ng pagpapatunay "formBasedAuthentication" ngayon ay kinikilala ang pag-login sa URL sa naka-encode na form. Ang pagbabago ay tinitiyak ang pag- login sa URL ay ginamit/pinadala bilang ito ay tinukoy. Noong nakaraan ito ay tintanggap ng bahagyang pag-encode ng mga URL pero muli nilang ini-encode habang ginagamit, potensyal na patungo sa isang magkakaibang URL na ginamit/pinadala.

Ang konteksto ng VIEW / contextList

Ang pagbabago na ito ay makakasira sa mga mamimili na mano-mano na pag-parse/extracting ng mga pangalan mula sa hanay. ang istruktura ng mga datos na ibinabalik ay binago sa wastong paghihiwalay ng bawat pangalan: 

{"contextList":["Context 1","Context 2"]}
at: 

<contextList type="list">
    <contextName>Context 1</contextName>
    <contextName>Context 2</contextName>
</contextList>
maliban sa: 

{"contextList":"[Context 1, Context 2]"}
at: 

<contextList>[Context 1, Context 2]</contextList>

Ang core ng VIEW / setOptionUseProxyChain

Ang pagbabago sa pagbabalik Nabigo kapag ang paparating na provy ay hindi pinagana (Dahil ang kailangan na adress/hostname ay hindi naka set dati), bago ito palaging bumalik Okay.

Ang pagbabago ng mga Endpoint ng ZAP API:

Ang core ng VIEW / mga alerto

Ang pagdadagdag ng opyonal na riskld ng parametro na pagaanin ang pagsala ng peligro. Kung saan ang riskld ay nasa 0 para sa pagbibigay kaalaman at tatlo para sa mataas.

Ang core ng VIEW /BilangNgMgaAlerto

Ang pagdadagdag ng opsyonal na riskld ng parametro na pagaanin ang pagsala ng peligro. Kung saan ang riskld ay nasa 0 para sa pagbibigay kaalaman at tatlo para sa mataas.

Ang core ng VIEW / mga url

Ang pagdadagdag ng opsyonal ang baseurl ang parametro, sa pagsala ng mga URL namga bumalik.

Ang ascan ng VIEW / mga pag-iskan

Ang pagbabgo para bumalik din ang kabuuang bilang ng mga alerto na itinaas at mga mensahe na pinadala sa panahon ng bawat pag-iskan.

Ang ascan ng VIEW / scanProgress

Ang pagbabago para bumalik din ang kabuuang bilang ng mga alerto na itinaas sa pamamagitan ng bawat pag-i-iskan.

Ang mga bagong Endpoint ng ZAP API:

Ang core ng VIEW / alertsSummary

Ang isang bagong buod ay makikita para sa mga alerto kung saan ipinapakita ang mga bilang ng mga alerto kada lebel ng peligro. opsyonal, na sinasala sa pamamagitan ng halaga ng isang baseurl. 

{"High":0,"Low":132,"Medium":39,"Informational":153}

Ang core ng VIEW / messagesByld

Nakukuha ang mga mensahe ng HTTP sa pamamagitan ng ibinigay na mga id.

Ang core ng VIEW / messagesHarByld

Nakukuha ang mga mensahe ng HTTP sa pamamagitan na ibinigay na mga ID, sa ayos ng HAR.

Ang core ng VIEW /optionAlertOverridesFilePath

Ang daan ng mga payl ay nakukuha sa pamamagitan ng mga override na alerto.

Ang core ng VIEW / optionMaximumAlertInstances

Nakukuha ang pinamataas na bilang ng alerto ng mga instance para isama sa isang ulat.

Ang core ng VIEW / optionMergeRelatedAlerts

Nakukuha kahit o hindi na kaugnayan sa mga alerto na maaaring ipagsama sa kahit na anung mga ulat na binubuo.

Ang core ng VIEW / zapHomePath

Nakukuha ang daan ng mga ZAP sa tahanan ng direktoryo.

Ang localproxies ng VIEW / mga additionalProxy

Nakukuha ang lahat ng mga karagdagang mga proxy na nakumpigura na.

Ang ispayder ng VIEW / optionAcceptCookies

Nakukuha kahit o hindi na ang pagproseso ng isang ispayder ay dapat na tanggapin ng mga cookie habang nag-ispayder.

Ang aksyon ng core /deleteAlert

Tanggalin ang mga alerto sa pamamagitan ng ibinigay na ID.

Ang AKSYON ng core / setOptionAlertOverridesFilePath

Ang mga pagtatakda (o mga pagliliwanag, kung wala ng laman) ang daan sa mga payl sa pamamagitan ng alerto ng mga overrdrive.

Ang AKSYON ng core / setOptionMaximumAlertInstances

Ang mga pagtatakda ng pinakamataas na bilang ng alerto ng mga pagkakataon na ipagsama sa isang ulat. Ang halaga ng sero ay itinuturing na walang limitasyon.

Ang AKSYON ng core / setOptionMergeRelatedAlerts

Ang mga pagtatakda kahit o hindi kaugnay sa mga alerto ay maisama sa kahit na anung ulat na binubuo.

Ang AKYON ng ascan / importScanPolicy

Ang pag-angkat ng isang patakaran ng pag-iskan gamit ang ibinigay na daan ng payl ng sistema.

Ang AKSYON ng ascan / skipscanner

Ang paglalaktaw sa pag-iskan gamit ang mga ID ng mga pag-iskan at ng pag-i-iskan.

Ang localProxies ng VIEW / addAdditionalProxy

Ang pagdadagdag ng isang bagong proxy gamit ang mga ibinigay na detalye tingnan ang Mga opsyon ng lokal na mga proxy ng iskrin para sa mga detalye ng mga parametro.

Ang mga localproxy ng VIEW / removedAdditionalProxy

Tanggalin ang mga naidagdag na proxy sa pamamagitan ng tinutukoy na adress at port.

Ang AKSYON ng ispayder /mga setOptionAcceptCookie

Ang pagtatakda o hindi ang isang pagproseso ng ispayder na dapat na tanggapin ng mga cookie habang nag-ispayder.

Ang mga detalye ng mga kahinaan

Ang mga kahinaan na mga sumusunod ay inuulat sa isang dating bersyon ng ZAP.
Maraming salamat sa lahat ng mga mananaliksik na sinong mayroong etikal na ulat na mga isyu para sa atin sa pamamagitan ng ating masaganang programa ng bug.
Kung kailangan mo ng mga detalye tungkol dito sa kahinaan pagkatapos ay mangyaring makipag-ugnayan po kayo sa amin.

Ang mga uninstaller ng mga window ng kahinaan sa Hijacking ng DLL

Ang mga wingdow ng uninstaller ng ZAP para sa 2.6.0 ay mahina sa Hijacking ng DLL ng mga window. Ito ay kahinaan sa pangatlong partido ng pag-install ng install4j kung saan ngayon na naisaayos na.
Paalala na ito ay mangyayari lamang kung ang isang malisyosong DLL ay nasa daraanan na.
