Pagpapakawala 2.5.0

Ang mga sumusunod na pagbabago ay ginawa sa pagpapakawala na ito:

Ang Mga pagbabago sa API ng ZAP:

TINGNAN ang pagpapahintulot / getAuthorizationDetectionMethod

Ngayon ay muling nagbabalik ang datos na nakabalot sa bagay na tinatawag na authorizationDetectionMethod: 
{"authorizationDetectionMethod":{"statusCode":"-1"..."headerRegex":""}}
kaysa sa: 
{"statusCode":"-1"..."headerRegex":""}

SURIIN ang konteksto / konteksto + technologyList

Magkaparehas ngayon ay muling nagbabalik ang data wrapped sa bagay na tinatawag na "context": Halimbawa na lamang: 
{"context":{"id":"1", ..., "inScope":"true","loggedOutPattern":""}}
kaysa sa: 
{"id":"1", ..., "inScope":"true","loggedOutPattern":""}

AKSYON ng ispayder / scan + scanAsUser

Magkaparehas ngayon ang pagsuporta sa bagong opsyonal na 'subtreeOnly' paramiter na syang nagbibigay ng hangganan sa ispayder hanggang sa tinututkoy na substree. Ang 'url' parameter ay isa na ngayong opsyonal, hanggat ang mabisang 'context' paramiter ay natutustusan

Ang bagong 'stats' na komponent

Ang bagong 'stats' API ay nagbibigay ng pagpasok sa ang stats ngayoy patuloy na pina-natili ng ZAP.

Tandaan na ang mga nananatiling komponent ay dapat lamang na magkaroon din ng mga bagong pagpapatakbo, tingnan lang ang Ang API Web UI para sa iba pang mga detalye.
