Inilabas na 2.6.0

Ang paglalabas na ito ay naglalaman ng isang malaking bilang ng mga pagbabago, partikular na ang ZAP API.
Kami ay nagdagdag ng isang makabuluhang bilang ng mga bagong mga endpoint ng API, nagtatrabaho patungo sa ating layunin ng paggawa ng ZAP na ganap na pag kontrol sa pamamagitan ng API. Ang ilan sa mga umiiral na mga endpoint at binago rin namin at sa lahat ng mga kaso ng mga pagbabagong ito ay pabalik na magkatugma.

Ang kumpletong paglabas ay bagbilang din sa isang bagong JxBrowser na add-on kabilang din ang partikular na plataporma upang maging mas madali na makipag ugnayan sa ZAP patungo sa isang mawalak na na ibat ibang mga browser.

ang mga pagbabago ng seguridad ng API

Bilang tugon sa mga isyu binago namin ang seguridad ng API na iniulat sa atin sa pamamagitan ng bug bounty program. Ang mga detalye ng mga kahinaan ay binigay sa ibaba.
Ang mga pagbabago ng seguridad ay sa pangangailangan na hindi paurong na tugma, bagama't isinama namin ang mga opsiyon para sa hindi pagpapagana sa kanila kung gagamit ka ng ZAP sa isang ligtas na paligid.

Sa pamamagitan ng pag-default sa lahat ng tawag ng API ngayon ay kailangan na kahi alin ng API key o ng isang noncio. Maaaring ito ay magbigay sa pamamagitan ng mga URL parameter, ang POST o ang mga header. Ang kliyente na suportado ng ZAP API ay (kabilang ang java at python) na-update upang suplayan ang API key sa pamamagitan ng isang header. Sa pamamagitan ng ZAP ang mga nuncio ay binuo at at sinadya upang magamit sa pamamagitan ng mga ZAP na add-on na kinakailangan sa pag-akses ng ZAP API. Para sa buong mga detalye tingnan ang ang mga pagpipilian sa iskrin ng API.

Ang mga pagpipilian sa bagong API at mayroong bagong ayos na may kaugnayan sa seguridad:

  • UI Enabled - Kung pinagana pagkatapos ang API Web UI ay magagamit sa lahat ng mga makina na kailangan magamit ang ZAP bilang proxy. sa pamamagitan ng default ito ay pinagana.
  • Upang gamitin ang API ang mga IP address ay pinahihintulutan nito - sa pamamagitan lamang ng pag- default ng tumatakbong makina sa ZAP ay magagawang makapasok ang ZAP API. Maaari mong pahintulutan na pumasok ang ibang makina sa API sa pamamagitan ng pagdadgdag ng mga angkop na disenyo ng regex. Ang mga IP address ay dapat mong dagdagan na iyong pinagkakatiwalaan.
  • Para sa mga ligtas na operasyon ay hindi nangangailngan ng isang API key - kung pinagana at pagkatapos ay ang API key ay hindi kinakailangan para sa pagtingin o iba pang operasyon na isinasaalang-alang ang 'kaligtasan', sa madaling salita ang mga operasyon na hindi gumawa ng anumang pagbabago sa ZAP. Ang mga naturang operasyon gayun paman ay nagbibigay ng daan sa datos ng ZAP katulad ng mga alerto, ang mga mensahe at mga pinagdaanan ng sistema ng payl. Maaari din nila itong gamitin sa pamamagitan ng mga aplikasyon ng Web upang makita ang presensya ng ZAP.
  • Ang mga pahintulot ng mga mali ay iulat sa pamamagitan ng API - kung pinagana at pagkatapos ang ZAP ay mag-uulat ng mga mali sa pamamagitan ng pahintuloy ng API, na maaaring gamitin ng mga aplikasyon ng Web para tukuyin ang kinaroroonan ng presensya ng ZAP. Sa isang ligtas na kapaligiran ito ay hindi seryosong problema pero kung ikaw ay gumagamit ng ZAP laban sa mga malisyoso na mga sayt na posible kung ganoon hindi mo dapat ito ay paganahin.
All ZAP options can be specified via the command line when you start ZAP - see https://www.zaproxy.org/faq/why-is-an-api-key-required-by-default/ para sa mga buong detalye.
Kami ay magdadagdag din ng kahit maraming seguridad na mga header sa mga API kasama ang isang matibay na nilalaman ng patakaran sa seguridad.

Ang mga pagbabago sa mga Endpoint ng ZAP API:

Ang aksyon ng ascan / scan

Ang parametro ng url na ngayon ay opsyonal at ang isang opsyonal na contexttld parametro ay idinagdag. Kinakailangan mong tustusan ang isa sa mga yon.

Ang aksyon ng ascan / scanAsUser

Ang url at contextld na parametro ay opsyonal na ngayon. Kailangan mong suportahan ang isa sa mga yon.

Ang aksyon ng ascan / addScanPolicy

Ang pagdadagdag ng mga opsyon ng alertThreshold at attackStrength ng mga parametro.

Ang bagong Endpoint ng mga ZAP API:

Ang pagpapakitang ascan / optionMaxRuleDurationInMins

Ang oras sa minuto na pinakamataas ay ibinabalik na ang isang patakaran ng iskan ay maaaring mapagana, ang sero ay walang limitasyon.

Ang pagpapakita ng ascan / optionMaxScanDurationMins

Ibalik ang pinakamataas ng oras sa minuto kung saan ang buong iskan ay maaaring mapagana sapagkat, ang zero ay walang limitasyon.

Ang AKSYON ascan / setOptionMaxRuleDurationInMins

Ang pag-set ng pinakamataas na oras ng minuto na itatagal ng pagpapatakbo ng isang patakaran ng iskan, ang sero ay walang limitasyon.

Ang AKSYON ascan / setOptionMaxScanDurationInMins

Ang pag-set ng pinakamataas na oras ng minuto na itatagal ng papatakbo patakaran ng iskan, ang sero ay walang limitasyon.

Ang AKSYON ascan / updateScanPolicy

Ang tinutukoy na mga pag-update ng patakaran ng scan na may tinukoy na alertThreshold o attackStrenght.

Ang VIEW break / isBreakAll

Kung ang ZAP ay huminto magbalik sa tama sa parehong mga kahilingan at mga tugon.

Ang VIEW break / isBreakRequest

Ibalik sa tama kung ang ZAP ay babaliin ang mga kahilingan.

Ang VIEW break / isBreakResponse

Ibalik sa tama kung ang ZAP ay babaliin ang mga kahilingan.

Ang VIEW break / httpMessage

Magbalik sa HTTP ng mensahe na kasalukuyang pumipigil (kung mayroon).

Ang AKSYON break / break

Ang mga kontrol ng pag-andar ng global break. Ang klase ay maaaring isa sa: http-all, http-request o http-response. Maaaring totoo ang estado (para sa pagpihit ng break sa on ng mga tinutukoy na klase) o mali ng (para sa pagpihit ng break sa off). Ang mga saklaw ay hindi kasalukuyang ginagamit.

Ang AKSYON break / setHttpMessage

Ang muling pag sulat sa kasalukuyang mensahe ay mahadlangan kasama ang nakalaan na datos.

Ang AKSYON break / patuloy

Submits the currently intercepted message and unsets the global request/response breakpoints.

Ang AKSYON break / hakbang

Ang pag-sumite ng kasalukuyang naharang na mensahe, ang susunod na kahilingan o tugon ay automatikong haharangin.

Ang AKSYON break / pagpatak

Ang pagbagsak ng kasalukuyang mensahe na naharang.

Ang mga VIEW core / optionDnsTtlSuccessfulQueries

Ang mga pagkuha ng TTL (sa mga segundo) ng matagumpay ng mga tanong ng DNS.

Aksyon core / sendRequest

Ang mga pagpapadala ng HTTP ng Kahilingan, opsyonal na mga sumusunod na redirection. Ang pag babalik ng mga kahilingan na naipadala at tugon na natangap at nasunod sa mga redirection, kung meron man. Ang Moda ay ipinatutupad kapag nagpadala ng kahilingan ( at kasunod ng mga redirection), ang mga gawi na manwal ng kahilingan ay hindi pinapayagan sa 'ligtas' na moda kahit sa 'protektadong' moda kung labas sa nasasaklawan.

Ang AKSYON core / setOptionDnsTtlSuccessfulQueries

Ang mga ayos ng TTl (sa mga segundo) ng matagumpay na mga tanong ng DNS ( gamitin pagkatapos na ZAP ay mai restart).

Ang iba pang core / mdreport

Ang pagbuo ng isang ulat sa Markdown na paraan.

TIGNAN ang mga httpSession / mga sayt

Kunin ang lahat ng mga site na mayroong mga session.

Ang VIEW pscan / scanOnlyInScope

Sabihin kung ang posibong pag-iskan ay dapat bang gumanap o hindi sa mga mensaheng na nasasakupan.

Ang AKSYON pscan / setScanOnlyInScope

Mga pagtatakda kung ang posibong pag- iskan ay dapat bang gumanap lang sa mga mensaheng nasa-sakupan o hindi.

Ang VIEW spider / allUrls

Ang isang listahan ng mga natatanging URL ay ibinabalik mula sa talahanayan ng kasaysayan batay sa mga mensaheng idinagdag ng ispayder sa HTTP.

Ang VIEW spider / optionMaxChildren

Kunin ang pinakamataas na bilang ng mga node na bata (bawat node) na maaaring igapang, 0 ibig sabihin walang limitasyon.

Ang ACTION spider / setOptionMaxChildren

I-set ang pinakamataas na bilang ng mga batang node (bawat node) na maaaring igapang, 0 ibig sabihin ay walang limitasyon.

Mga Detalye ng kahinaan

Ang mga kahinaan na mga sumusunod ay naiulat sa nakaraan na mga bersyon ng ZAP. Ang iba pang hindi gaanong seryosong isyu ay na ayos na bilang isang kurso ng bagay.
Maraming salamat sa lahat ng mga mananaliksik na mayroong etikal na ulat ng mga isyung ito sa amin sapamamagitan ng ating ang programa sa kasaganaan ng bug. kung kailangan mo ng karagdagang mga detalye tungkol sa kahit aling kahinaan ito mangyari na makipag-ugnayan po kayo sa amin.

Ang RCE sa pamamagitan ng Anti CSRF Test Form at API Key Disclosure

Kung ang pag gamit ng gumamit ng Anti CSRF Test Form laban sa isang partikular na ginawa ng HTML na pahina at pagkatapos ang API key ay nahayag sa sayt na iyon. Ang sayt na ito na maaaring pagkatapos ay madaanan ang ZAP API na nagsasagawa ng anumang aksyon, kabilang ang pag-upload ng mga iskripto ng ZAP. Ang mga skrip na maaari lamang na ma-upload mula sa 'lokal' filesystem ngunit kung ang gumagamit ay nagpapatakbo ng ZAP sa mga window pagkatapos ang mga attacker ay maaaring gumawa ng isang malisyosong skrip na magagamit sa pamamagitan ng isang publiko na naipapamahagi ng SMB. Bilang isang lokal na payl ito ay lilitaw sa ZAP at ang iskrip ay na-upload at maaaring mapagana sa pamamagitan ng API.
Para sa API key o noncio ang kinakailangan na sa lahat ng mga operasyon ng API ay isang direktang resulta ng kahinaan na ito, gaya ng pagbabago ng mga add-on sa paggamit ng noncio upang mabawasan ang panganib ng pagtagas ng API key.

Ang mga window ng mahinang pag-iinstall sa dll hijacking

Ang mga window ng ZAP na installer para sa lahat ng mga bersyon papataas ta kasama ang 2.5.0 ay maghina sa DLL Hijacking sa mga window 7 (at ang mga naunang bersyon). Ito ay isang kahinaan sa ika tatlong partido ng installer InnoSetup. Ang 2.6.0 na mga installer (sa lahat ng mga plataporma) ay nabuo na ngayon gamit ang Install4J.

Kung sa ibang dahilan mo na kailangan mong i-install ang lumang bersyon ng ZAP sa Windows 7 o mas maaga kung ganon ay nirerekuminda namin sa iyo na ilipat mo ang installer sa isang malinis na direktoryo bago paandarin ito.

Tandaan na ang Burp Suite ay gumagamit din ng Install4J sa hinaharap na kahinaan ng Install4J-generated na mga installer maaaring karapat dapat para sa Burp Suite bug bounty program: ang https://hackerone.com/portswigger

Ang Kredito: James Kettle (Burp Suite)

Ang di makatwirang kowd ng pagpapatupad sa pamamagitan ng mga panawagan ng paplikasyon ng parametrong Injection

Ang mga HTML na parametro ay maaaring maging tiyak na ginawa upang maging sanhi ng di-makatwirang pagpapatupad ng code, kung pipiliin ng gumagamit ng panawagan para puntiryahin ang aplikasyon na may isang kahilingan na naglalaman ng parametro mula sa ZAP.
Ang mga aplikasyon na may panawagan sa Add-on ay na-update na para maayos ang isyu - lahat ng mga gumagamit ng ZAP ay kailangan i-install itong bagong bersyon bago magpatuloy sa paggamit ng add-on.

Ang Kredito: Artemy Bogdanov (@Abr1k0s)
