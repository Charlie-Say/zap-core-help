Versiunea 1.3.0

Următoarele modificări au fost aduse acestei versiuni:

Schimbări importante:

Testarea Fuzz

Șiruri dintr-un răspuns pot fi acum testate Fuzz pentru a se încerca descoperirea de vulnerabilități.
Token-uri anti CRSF pot fi acum detectate și regenerate automat în timpul testării FUZZ.
Această funcționalitate este bazată pe cod din proiectul OWASP JBroFuzz.

Certificatele SSL dinamic

Suportul pentru conexiuni SSL a fost îmbunătățit și simplificat.
Utilizatorii pot acum crea propriu certificat rădăcină iar apoi să-l distribuie în HTTP-ul clienților.
Vezi Opțiuni: Certificate SSL dinamic pentru mai multe detalii.

Modul Daemon și API

Lansarea ZAP cu comanda "-daemon" din opțiunea liniei de comenzi va determina rularea în fundal în modul 'headless', însemând că nici o interfață de utilizator nu va fi afișată.
Un API inițial a fost implementat in XML, JSON și HTML.
Dacă ZAP rulează ca daemon atunci API-ul va fi activat automat, în caz contrar API-ul trebuie activat din ecranul de Opțiuni API.
API-ul poate fi navigat prin a deschide http://zap/ în browser cand se face proxy prin intermediul ZAP.

Integrarea Beanshell

BeanShell este un shell Java interactiv care poat fi folosit pentru a executa scripturi BeanShell.
Aceste scripturi sunt o formă simplificată de Java care folosește multe elemente din sintaxa Java, dar într-un format mai simplu de scriptare.
Tot codul Java este, de asemenea, cod valid BeanShell.
Integrarea BeanShell în OWASP ZAP permite scrierea de script-uri folosind funcțiile și setul de date ZAP.
Aceasta poate fi o caracteristică foarte puternică în analiza aplicațiilor web.

Internaționalizare completă

Toate șirurile de afișare sunt acum internaționalizate complet.

Schimbări minore:

Afișare hexagonală

Tab-urile pentru Cereri, și Răspunsuri oferă acum opțiunea de afișare hexagonală, ce va afișa corpurile cererilor și răspunsurilor în format hexagonal.

Rezultatele căutării

Tab-ul de Căutare acum afișează toate instanțele de șiruri găsite, mai degrabă decât doar prima instanță a fiecărei cereri sau fiecărui răspuns.

Excludearea URL-urilor

URL-uri pot fi acum excluse explicit din scanarea activă, din indexare și din proxy.

Suport pentru copiere

Cele mai multe panouri acum oferă opțiunea de meniu 'copiere' prin click dreapta, incluzând panourile de Portare a Scanării și de Forțare Brută.

Suport Anulează/Reface

Toate câmpurile de introducere oferă acum sprijin prin acțiuni de Anulează/Reface folosind acceleratoarele implicite Anulează/Reface a sistemului de operare:
  • Windows/Linux: Ctrl+Z / Ctrl+Y
  • Mac OS X: Cmd+Z / Cmd+Shift+Z

Suportul pentru proxy-ul portării de scanare și opțiunea de pauză

Scanerul de portare poate să folosească acum proxy-ul de ieșire( în cazul în care acesta este configurat) și pauză în milisecunde poate fi acum configurată.

Butoanele pentru cerere și rupere a răspunsului

Acum există două butoane de 'Setare a Ruperii' care permit stabilirea de pauze pentru toate cererile și răspunsurile în mod independent.

Butoanele pentru a extinde tab-urile site-uri și informație

Acum există două butoane care permit comutarea între a extinde tab-urile de Site-uri sau Informație.

Probleme cunoscute:

Mac OS X: SSL dinamic și Google Chrome

SSL-ul dinamic curent nu merge când este folosit Google Chrome. Acest lucru este din cauza unei probleme știute și nerezolvată a Google Chrome și Keychain-urilor Mac OS X. În momentul importării Rădăcinii CA a OWASP ZAP în keychain și cererea unui website SSL, un mesaj de eroare este afișat "Certificat invalid”.
